## 先打个比方

请一个新人帮你「订下周去上海的高铁票」。**自动售货机**式的助手只会干一件事：你投币、它吐货，问它别的就傻眼——这是只会一问一答的程序。**流水线工人**好一点：照着写死的步骤走，第一步查车次、第二步下单、第三步发确认，可一旦中途没票了，它不会变通，照样往下走。

你真正想要的是一个**靠谱的同事**：他会自己想「先查有没有票」，动手去查，看到没票就改查别的车次，查到了再下单，全程自己拿主意，办完了回来告诉你结果。

这一章就是讲：怎么让一个只会聊天的 AI，变成这个会自己查、自己动手、自己复盘的同事。少了这一步，你的 AI 就永远停在「自动售货机」——你说一句它答一句，啥也帮你办不成。

## 第一个概念：裸 LLM 是个「只会背书的实习生」

我们天天在用的 <Term def="大语言模型（Large Language Model）：在海量文本上训练出来的 AI 模型，本质能力是「根据前面的话，预测下一个最可能的词」，一个词一个词地把回答续写出来。">大语言模型（LLM）</Term> 强得惊人，但它本质上只做一件事：**根据前面的文字，预测下一个词**。它像一个读了万卷书、却被关在小黑屋里的实习生——你给他纸条（提问），他凭记忆写满还给你（回答）。仅此而已。

这意味着裸 LLM 有三个天生的短板：

- **不能行动**：它只会输出文字，没法真的去查天气、发邮件、跑一段代码。你问它今天北京几度，它只能瞎猜或老实说「我不知道」。
- **没有记忆**：聊完这一轮，下一轮它就忘了——它不会自己记住「上次你说过你怕冷」。
- **不知道新事**：它的知识停在训练那一刻，训练之后发生的事一概不知。

记住这三条短板——接下来要讲的每一块「外挂」，都是为了补上其中一条。

## 第二个概念：智能体 = LLM 大脑 + 四件外挂

把那个「只会背书的实习生」武装起来，他就成了 <Term def="智能体（Agent）：一个以 LLM 为决策核心、能调用工具、能记住上下文、并在循环里反复自主决定下一步、直到完成任务的程序。">智能体（Agent）</Term>。一个智能体 = 一个 LLM 大脑，外加四件外挂：

<AgentAnatomyDiagram />

四件外挂分别补上了裸 LLM 的短板：

- 一是<Term def="工具（Tools）：智能体能调用的外部函数或接口，比如查天气、搜网页、跑代码、读写文件。LLM 决定「调哪个、传什么参数」，工具真正去执行并把结果返回。">工具（Tools）</Term>：让它**能行动**——查天气、搜网页、算账、发消息。LLM 负责决定「调哪个工具、传什么参数」，工具负责真去执行。
- 二是<Term def="上下文（Context）：一次任务里喂给 LLM 的全部信息——历史对话、工具返回的结果、相关资料等。它就是智能体在这一轮里的「短期记忆」。" id="context">上下文（记忆）</Term>：让它**记得住**——把聊过的话、查到的结果都装进去，下一步思考时一并看到。
- 三是<Term def="系统提示（System Prompt）：放在对话最前面的一段指令，规定智能体的身份、目标、可用工具和行为准则，相当于给它的「岗位说明书」。" id="system-prompt">系统提示（指令）</Term>：给它一份「岗位说明书」——你是谁、要达成什么、有哪些工具、有什么规矩。
- 四是<Term def="智能体循环（agentic loop）：智能体反复执行「思考下一步 → 行动（调工具或回答）→ 观察结果」的过程，由 LLM 自己决定每一轮干什么、以及何时结束。" id="agentic-loop">智能体循环（Loop）</Term>：让它**反复干活**——想一步、做一步、看一眼结果，再想下一步，直到任务完成。这是智能体的灵魂，下面单独细讲。

> 一句话记忆：**少了工具它不能动手，少了记忆它转头就忘，少了指令它不知道要干嘛，少了循环它只能干一锤子买卖。四件齐了，才叫智能体。**

## 第三个概念：智能体 vs 工作流 vs 聊天机器人

智能体有两个长得很像的「近亲」，新手最容易搞混。一张图分清：

<AgentVsWorkflowDiagram />

- 一是<Term def="单轮一问一答的程序：你问一句，它答一句，到此为止，不会自己接着去办事。是智能体最简单的近亲。" id="chatbot">聊天机器人（Chatbot）</Term>：单轮——你问一句，它答一句，到此为止。它不会自己接着去办事。
- 二是<Term def="工作流（Workflow）：把若干步骤按固定顺序串成的流水线，每一步、走哪条路都是人预先写死的，每次执行都一样、可预测。">工作流（Workflow）</Term>：固定流水线——步骤 1 → 2 → 3 是人**提前写死**的，每次都走同一条路，不会随情况变。
- **智能体（Agent）**：循环——LLM 在循环里**自己决定**下一步该干什么，路径随情况动态变化。

关键差别就一句话：**工作流的路是人写死的，智能体的路是它自己边走边选的。** 这份「自己拿主意」的能力，就是智能体最不一样的地方，也是它的全部威力与全部风险的来源。

## 第四个概念：自主性是一根可调的旋钮

「自己拿主意」听起来很酷，但它不是非黑即白的开关，而是一道连续的 <Term def="自主性（Autonomy）：程序在多大程度上「自己决定下一步」而非按人写死的逻辑走。自主性越高越灵活，但越难预测、越难控制、通常也越贵越慢。" id="autonomy">自主性</Term>光谱：

<AutonomySpectrumDiagram />

从纯代码（一切写死）到自主 Agent（一切自己决定），自主性越往右越高。**自主性不是越高越好**——越往右越灵活、能应付越开放的任务，但也越难控制、越贵、越慢。选方案的本质，就是在「灵活」和「可控 / 便宜 / 快」之间，按任务取一个合适的点。能用代码或工作流解决的事，没必要硬上自主 Agent。这正是下一节交互动画要你亲眼看清的东西。

## 用代码看循环的骨架

上面的动画，落到代码里其实就是一个 `while` 循环。下面这段精简的伪代码，是几乎所有智能体框架的共同骨架——把它读懂，你就读懂了智能体：

```python
def run_agent(user_goal):
    # 上下文 = 智能体的记忆，先放进系统提示和用户目标
    context = [system_prompt, user_goal]

    while True:                          # 这就是「智能体循环」
        # ① 思考：LLM 看着当前上下文，决定下一步
        decision = llm(context)

        # ② LLM 自己判断：是该收尾，还是该调工具？
        if decision.is_final_answer:
            return decision.answer       # 跳出循环，给最终答案

        # ③ 行动：照 LLM 选的工具和参数，真去执行
        result = call_tool(decision.tool, decision.args)

        # ④ 观察：把工具结果喂回上下文，进入下一圈
        context.append(decision)
        context.append(result)
```

逐句对照动画看：`while True` 就是那个环；`llm(context)` 是**思考**；`if decision.is_final_answer` 是 LLM **自己**决定何时结束（对应动画第⑤步「信息够了」）；`call_tool(...)` 是**行动**；`context.append(result)` 是**观察**——把结果塞回记忆，下一圈 `llm(context)` 就能看到它了。

注意两个要害：一是**循环条件不是写死跑几次**，而是由 LLM 在每一圈里自己决定要不要继续——这就是自主性的落点；二是**记忆（context）在每一圈都在变长**，工具结果不断累加进去，所以它越转越「懂」当前任务。对比一下：工作流的代码会是几个写死的函数顺序调用，没有这个 `while`，也没有「LLM 决定下一步」。

## 容易踩的坑

<Callout type="trap">
  **现象**：「我接了个能联网搜索的
  GPT，这就是智能体了吧？」做出来发现它只会搜一次、答一次，复杂任务一塌糊涂。
  **原因**：把「能调工具」当成了智能体的全部，漏掉了**自主循环**。能调一次工具的
  LLM，只是自主性光谱里偏左的「LLM +
  工具」，不会根据结果反复思考、调整、再行动。 **修法**：补上循环——让 LLM 在
  `while`
  里反复「思考→行动→观察」，由它自己决定继续还是收尾。没有这个环，就还不是智能体（回看本章解剖图与循环动画）。
</Callout>

<Callout type="trap">
  **现象**：一个明明很固定的任务（每天把固定格式的报表转成邮件发出去），非要做成自主
  Agent，结果又贵又慢，还时不时发错。
  **原因**：误以为「智能体越自主越高级」，把自主性旋钮一拧到底。任务本身是确定、可预测的，自主决策反而引入了不必要的随机性和成本。
  **修法**：按自主性光谱反向选——固定可预测的任务用代码或工作流更稳、更便宜、更快；只有任务开放、步数不定、需要随机应变时，才往右选自主
  Agent。
</Callout>

## 小结

- 裸 LLM 只会预测下一个词，天生不能行动、没记忆、不知新事
- 智能体 = LLM 大脑 + 四件外挂：工具（能动手）、记忆（记上下文）、指令（系统提示）、循环（反复想—做—看）
- 三个近亲的区别：聊天机器人单轮问答、工作流路径写死、智能体在循环里自己决定下一步
- 智能体循环是灵魂：在「思考→行动→观察」里转圈，由 LLM 自己决定何时调工具、何时结束
- 自主性是可调旋钮，不是越大越好；固定任务用代码 / 工作流，开放任务才上自主 Agent

<Exercises>

**练习 1（改 Demo / 改循环 · 思考题）** 在本章「用代码看循环的骨架」里，`while True` 让循环可能一直转下去。如果一个任务里 LLM 始终判断不出 `is_final_answer`（比如反复调工具却拿不到想要的结果），会发生什么？请改写这段循环，加一个保护机制，让它在转了太多圈之后强制停下并返回一个兜底回答。

<Answer>

会发生**死循环**——智能体无限地「思考→调工具→观察」却永远不收尾，白白烧钱、卡住不返回。修法是加一个**最大步数上限**（业界叫 max iterations / step budget）：

```python
def run_agent(user_goal, max_steps=10):
    context = [system_prompt, user_goal]
    for step in range(max_steps):        # 不再 while True，最多转 max_steps 圈
        decision = llm(context)
        if decision.is_final_answer:
            return decision.answer
        result = call_tool(decision.tool, decision.args)
        context.append(decision)
        context.append(result)
    # 转满还没收尾 → 强制兜底，别让它无限转
    return "抱歉，我尝试了多次仍没能完成这个任务。"
```

关键改动：把 `while True` 换成 `for step in range(max_steps)`，循环转满还没等到 `is_final_answer` 就跳出、返回兜底回答。这是真实智能体系统里必备的保护——自主性给了 LLM 决定权，但人必须留一道「最多转几圈」的闸。

</Answer>

**练习 2（辨析题）** 下面三个东西，分别属于聊天机器人、工作流、还是智能体？为什么？
（A）一个固定脚本：每天 9 点查汇率 → 写进表格 → 发邮件，步骤永不变。
（B）一个客服窗口：你问「营业时间？」它答「9 点到 18 点」，问完即止。
（C）一个助手：你说「帮我订一家明晚适合 4 人的餐厅」，它自己查餐厅、比价、看评分、改条件再查、最后下单。

<Answer>

（A）是**工作流**——步骤固定、路径写死、每次都一样，没有「自己决定下一步」。
（B）是**聊天机器人**——单轮一问一答，问完就停，不会自己接着办事。
（C）是**智能体**——它在循环里自己决定查什么、不满意就改条件再查、最后自主决定下单，路径随情况动态变化。判别诀窍：看它**是不是在循环里自己拿主意**，是 → 智能体；步骤写死 → 工作流；只有一来一回 → 聊天机器人。

</Answer>

**练习 3（应用题）** 你要做一个功能：把用户上传的发票图片里的金额提取出来，填进一张固定的报销单。这个任务该用代码、工作流，还是自主 Agent？说明理由。

<Answer>

优先用**代码 / 工作流**，而不是自主 Agent。理由：这个任务**固定、可预测、步数确定**——识别金额 → 校验格式 → 填进固定字段，每次流程都一样，没有需要随机应变的开放决策。按自主性光谱，它落在偏左侧：用一个写死的流水线（OCR 识别 + 规则校验 + 填表）最稳、最便宜、最快。硬上自主 Agent 只会让它更贵、更慢，还可能因为「自由发挥」而填错。只有当任务变得开放（比如「帮我把这堆乱七八糟的票据整理报销，缺信息时自己想办法补」）时，才值得往右挪向智能体。

</Answer>

</Exercises>

  <GlossaryItem term="上下文（Context）" id="context">
    一次任务里喂给 LLM
    看的全部信息：聊过的话、查到的结果、相关资料。它就是智能体在这一轮里的「短期记忆」——每转一圈都把新结果塞进去，所以它越干越懂当前任务。详见本章「第二个概念」。
  </GlossaryItem>
  <GlossaryItem term="系统提示（System Prompt）" id="system-prompt">
    放在对话最前面的一段「岗位说明书」：告诉智能体你是谁、要达成什么目标、能用哪些工具、要守什么规矩。它定调了智能体整轮的行为方式。详见本章「第二个概念」。
  </GlossaryItem>
  <GlossaryItem term="智能体循环（agentic loop）" id="agentic-loop">
    智能体反复执行「思考下一步 → 行动（调工具或回答）→ 观察结果」的过程，由 LLM
    自己决定每圈干什么、什么时候结束。这是「智能体」三个字的灵魂——它不是一锤子买卖，而是一圈一圈地转着把活干完。详见本章「第二、五个概念」。
  </GlossaryItem>
  <GlossaryItem term="工作流（Workflow）">
    把若干步骤按固定顺序串成的流水线，每一步、走哪条路都是人提前写死的，每次执行都一样、可预测。和智能体的区别：工作流的路是写死的，智能体的路是它自己边走边选的。详见本章「第三个概念」。
  </GlossaryItem>
  <GlossaryItem term="聊天机器人（Chatbot）" id="chatbot">
    单轮一问一答的程序：你问一句，它答一句，到此为止，不会自己接着去办事。是智能体最简单的近亲。详见本章「第三个概念」。
  </GlossaryItem>
  <GlossaryItem term="自主性（Autonomy）" id="autonomy">
    程序在多大程度上「自己决定下一步」而不是按人写死的逻辑走。它是一根可调的旋钮：越往右越灵活、能应付越开放的任务，但也越难控制、越贵、越慢——不是越大越好。详见本章「第四个概念」。
  </GlossaryItem>
</Glossary>
