## 给失忆天才配齐一套装备

前两章我们认识了那位「关在没窗小屋里、绝顶聪明却严重失忆」的天才，也拆开了他的脑袋，看清他天生会瞎编、转头就忘、还干不了实事。光有一颗聪明的脑袋，他还是把活干不完。

这一章像一张「装备清单」：要把这位天才改造成真能干活的帮手，得围着他添几样东西——一张说清任务的纸条、一部能打出屋外的电话、一个记事的本子、一张待办清单。每样配件补他的一块短板，凑齐了，他才成型。

弄清这套装备由哪几件组成、各干什么、一个活儿是怎样在它们之间转一圈干完的，本章的目的就达到了——而且你会顺手拿到整本书的地图：后面每一篇，其实就是手把手教你装上其中一件。

## 一个智能体，拆开看是五大件

先把整张「解剖图」摊在桌上。一个能干活的智能体，拆开看就是**五大件**：正中间一颗大脑，外加围着它的四样配件。我们一件件来对照那位失忆天才。

中心那颗大脑，就是前两章讲的大模型，这里叫它 <Term def="智能体的大脑，负责「动脑」——读懂处境、做决策、想下一步该做什么。它就是关在小屋里的失忆天才本人，天生会幻觉、记不住、动不了手，所以才需要外面四件配件来补。" id="agt-model-anatomy">模型</Term>。它是失忆天才本人，负责决策、动脑，但也带着天生的三块短板（会幻觉、记不住、动不了手）。剩下四件，就是来给他补短板的。

第一件是 <Term def="你和失忆天才唯一的沟通方式，就是从门缝塞进去的那张纸条：把任务讲清楚、控制它的风格、让它按格式回话。纸条写不好，再聪明的大脑也答非所问。">提示（Prompt）</Term>：天才出不来、你进不去，唯一的沟通方式就是从门缝塞一张纸条进去。怎么把任务在纸条上说清楚、让他按你要的格式回话，就是这件配件管的事。

第二件是 <Term def="给小屋装的一部「电话」：失忆天才出不了屋，但他能在纸条上写「请帮我做 X」，由外面替他做完、把结果塞回去。靠它，模型才能搜索、算账、读写文件——治「动不了手」。">工具（Tools）</Term>：失忆天才出不了屋，不能上网、算账、查实时数据。给屋子装一部「电话」，他就能在纸条上写「请帮我查一下票价」，由我们替他做完、把结果塞回去。

第三件是 <Term def="给失忆天才配的笔记本（短期记忆）+ 一座可检索的资料库（长期记忆 / RAG）：该记的存下来、要用时取回；答之前先去查、有依据再说——治「记不住」和「会幻觉」。">记忆（Memory）</Term>：天才转头就忘，还不懂你的私有资料。给他一个笔记本记下短期要点、再给一座可检索的资料库放长期知识，要用时取回塞回纸条，他就「记得住」、也能照资料说话。

第四件是 <Term def="给失忆天才的一张待办清单：把一个大任务拆成一步步子任务，做完还回头检查自己的作业、错了就纠——治「大任务无从下手、会犯错不自知」。">规划（Planning）</Term>：碰到要好几步的大任务，天才容易无从下手、还会犯错不自知。给他一张待办清单，把大任务拆成子任务、一步步推进，做完再回头自检。

## 一个任务，怎样在五大件之间流转

光知道有哪五件还不够，得看它们怎么**协作**。一个任务进了智能体，会在五大件之间流转一圈——这条「数据怎么一件件传过去」的路线，就叫 <Term def="一个任务在智能体五大件之间传递、处理的路线：从任务进来，依次流经提示、模型、工具、记忆、规划，没完就回到模型再转一轮，做完才输出。它把五件孤立的配件串成一个能干活的整体。">数据流（Data Flow）</Term>。

走一遍这条路：任务进来 → **提示**把它组装成一张纸条 → **模型**读纸条、决策（比如「得先查票价」）→ 判断要不要调工具 →（要）**工具**出屋查到结果、塞回来 → **记忆**把关键信息存下/取回 → **规划**对照待办清单看「还有没有下一步」 → 没完就**回到模型**继续转下一轮、做完则**输出结果**。这个「没完就回头再转」的劲儿，正是上一章 <Term def="智能体反复运转的核心循环：感知（看现状）→ 决策（想下一步）→ 行动（调工具做事）→ 观察（看结果），再回到感知，直到把任务干完。本章看到的，就是它在五大件上的具体走法。" id="agt-loop-anatomy">Agent Loop（智能体循环）</Term>在五大件上的具体走法。

## 容易踩的坑

<Callout type="trap">
  **现象**：把五大件当成可有可无的「功能列表」，挑着实现一两件就以为做出了智能体。**原因**：忽略了五件是**协作整体**——少了工具它出不了屋，少了记忆它转头就忘，少了规划它大任务抓瞎，缺哪件都会在对应短板上翻车。**修法**：照着「一个任务如何在五件间流转」那条数据流想——每一件都在流程里占一个不可替的环节，按任务实际需要补齐，而不是随手挑两件。
</Callout>

<Callout type="trap">
  **现象**：把「提示」和「记忆」混为一谈，以为「把历史都写进纸条」就等于给了它记忆。**原因**：提示是**这一次**塞进去的纸条，受窗口长度所限；记忆是把该记的**存到屋外**、要用时再取回塞进纸条——两者一个管「当下怎么说」、一个管「跨次记得住」。**修法**：分清职责——纸条装不下的、要跨多轮留存的，交给记忆（篇
  4）；当前这轮怎么把话说清楚，才是提示（篇 2）。
</Callout>

## 小结

- 一个智能体 = **五大件**：模型（大脑）+ 提示（纸条）+ 工具（电话）+ 记忆（笔记本+资料库）+ 规划（待办清单）
- 五件不是孤立零件，而是被一条**数据流**串成的整体：提示进 → 模型断 → 工具做 → 记忆记 → 规划盘，没完就回模型再转一轮，做完才输出
- 这个「没完就回头再转」的劲儿，就是上一章 **Agent Loop** 在五大件上的具体走法
- 全书地图：**提示 → 篇 2、工具 → 篇 3、记忆 → 篇 4、规划/反思 → 篇 5**（模型在篇 1）；篇 6 多智能体组队、篇 7 上生产
- 四件配件各补失忆天才一块短板：工具治「动不了手」、记忆治「记不住 + 会幻觉」、规划治「大任务抓瞎」、提示让你「把话说对」

**练习 2（流转题）** 用「一个任务如何在五大件间流转」这条线，说说一个任务从进来到输出，依次会经过哪些件？什么情况下不会一次走完、而要回头再转一轮？

<Answer>

依次是：任务进来 → **提示**组装成纸条 → **模型**读纸条决策 → 判断要不要调工具 →（要）**工具**出屋做事、结果塞回 → **记忆**存/取关键信息 → **规划**对照待办清单看还有没有下一步。如果规划发现「还没干完」（比如查到了票价但还没比价下单），就**回到模型**继续转下一轮；只有当确实没有下一步了，才**输出结果**、停。这个「没完就回头」正是 Agent Loop 的循环。

</Answer>

**练习 3（问答题）** 同事说：「我给模型写了很详细的提示，就等于给了它记忆，对吧？」用本章对「提示」和「记忆」的区分纠正他。

<Answer>

不对。**提示**是这一次塞进门缝的纸条，只在当前这轮起作用，还受上下文窗口长度限制——写得再详细，也装不下越攒越长的历史、跨不过窗口上限。**记忆**则是把该记的信息**存到屋外**（短期笔记本 + 长期资料库），要用时再取回、塞进当前纸条，从而跨多轮「记得住」、还能装下远超窗口的私有知识。一个管「当下怎么把话说清楚」（提示，篇 2），一个管「跨次记得住、补私有知识」（记忆，篇 4），不能互相替代。

</Answer>
