## 一位关在小屋里的失忆天才

想象有位绝顶聪明的顾问，被关在一间没有窗户的小屋里。你和他唯一的交流方式，是从门缝塞一张纸条进去；他读完，写一段回复，再从门缝塞出来。麻烦的是：他写完就立刻忘光一切，下次塞纸条进去，他完全不记得上次说过什么。

更要命的是，他出不了这间屋子——不能上网、不能算账、不能查资料，只会「读纸条、写纸条」。问他「今天天气怎样」，他只能凭记忆瞎猜，因为他没法真的去看一眼窗外。

这就是本章要解决的问题：**怎么把这位「只会答话、干不了实事」的天才，改造成一个能在真实世界里把活干完的帮手？** 不解决它，你手里就永远只有一台「会聊天的嘴」，碰到需要好几步、要查实时信息、要动手做事的任务，它就抓瞎。

## 三种系统：直线、固定流程、自主循环

那位失忆天才，本质就是一个 <Term def="一个绝顶聪明但严重失忆的「纸条机器」：你给它一段文字（提示），它读完写一段回复，仅此而已——不记上次、不能上网、不能动手做事。本书把它比作关在没窗小屋里的失忆天才。">大模型</Term>。只让它「读纸条、写纸条」，你得到的就是最朴素的一种系统。先把基于它的三种系统分清楚——这是整章的地基。

第一种是 <Term def="把用户的话直接丢给大模型、拿回一段回复就结束的系统，一问一答、没有下一步。像隔着门缝问失忆天才一句、他答一句。">聊天机器人（Chatbot）</Term>：你问一句，它答一句，到此为止。它是一条**直线**——问→答→结束，没有回环、不会自己往下推进。翻译一句话、解释一个概念，它够用；可一旦任务需要好几步、要查实时数据，它就只能凭记忆瞎编。

第二种是 <Term def="由人事先写死步骤、每次都按同一条固定路线执行的系统（步骤 A→B→C）。中间哪一步用到大模型都行，但走法是人定死的，程序不会临场改主意。">工作流（Workflow）</Term>：人把步骤事先排好、写死成固定的一条链，比如「先抓汇率 → 再套模板 → 最后生成日报」，每次都老老实实走同一条路。它能干多步的活，但**路线是人定死的**——遇到没预料的情况，它不会自己拐弯。

第三种才是主角：<Term def="能自己感知处境、决定下一步、调用工具去做、再观察结果的系统；靠一个循环反复推进，直到把任务干完，不需要人把每一步都写死。本书的目标就是亲手造一个。">智能体（Agent）</Term>。它不是直线、也不是死板的固定流程，而是一个**自己转圈的循环**：看一眼现状、想下一步、动手做、再看结果，然后接着来——直到任务完成。下面这张图把三者并排画出来：

## 智能体的灵魂：感知—决策—行动—观察

智能体那个「自己转圈」的循环，有个正式名字叫 <Term def="智能体反复运转的核心循环：感知（看现状）→ 决策（想下一步）→ 行动（调工具做事）→ 观察（看结果），再回到感知。它让本来只会答话的大模型，变成能自主把任务一步步推进到底的智能体。">Agent Loop（智能体循环）</Term>。它由四个环节首尾相接：

- **感知（Perceive）**：先读懂当前处境——任务是什么、上一步做出了什么结果、现在环境如何。相当于天才先把刚塞进来的纸条看明白。
- **决策（Decide）**：大脑根据感知到的信息，想清楚下一步该做什么。这一步正是大模型最擅长的「动脑」。
- **行动（Act）**：真去做事——调用工具搜索、算账、读写文件。这是失忆天才单靠自己做不到、必须给他「装一部电话」才行的部分（后面篇章会专门讲）。
- **观察（Observe）**：看看行动产生了什么结果，把它作为下一轮的输入，于是又回到感知。

为什么说这个循环是智能体的灵魂？因为正是「行动完要**回头观察**、再据此**重新决策**」这一闭环，让智能体能自我纠偏、一步步逼近目标，而不像聊天机器人那样答完就撒手。下面这张可控动画，先演一次聊天机器人的直线，再把这个循环转给你看：

## 预告：我们要亲手造的 tinyagent

整本书会带你从零搭一个麻雀虽小五脏俱全的 Agent 框架，叫 `tinyagent`。到全书结尾，调用它会简单到像这样：

```python
# 全书我们要一砖一瓦搭出来的 tinyagent（这里只是预告，本章还不写实现）
from tinyagent import Agent, tool

@tool
def search(query: str) -> str:
    ...  # 给失忆天才装的「电话」：让他能查实时信息

agent = Agent(tools=[search])     # 把大模型 + 工具组装成一个智能体
result = agent.run("订一张明天最便宜的高铁票")  # 它自己跑 Agent Loop 直到完成
```

这段代码现在看不懂没关系——`@tool`、`Agent.run` 背后的 Agent Loop，正是后面各篇要一块块教你亲手造的。本章你只要记住：`agent.run(...)` 内部转的就是那个「感知—决策—行动—观察」的圈。

## 容易踩的坑

<Callout type="trap">
  **现象**：把任何「接了大模型的程序」都叫智能体，逢人就说自己做了个
  Agent。**原因**：只把用户的话丢给模型、拿回一段话就结束，那只是聊天机器人；只是按人写死的固定流程跑几步，那是工作流——它们都没有「自己决定下一步」的循环。**修法**：用一条线检验——这个系统会不会
  **行动完回头观察、再据此重新决策**？没有这个闭环，就不是智能体，别硬套名头。
</Callout>

<Callout type="trap">
  **现象**：一上来就想给所有任务都套上智能体，结果简单的翻译、问答也搭了一堆循环，又慢又不稳还难调。**原因**：误以为「自主性越高越好」，忽略了智能体的循环是有代价的（更慢、更贵、更难预测）。**修法**：按任务选型——能一问一答就用聊天机器人，路径能写死就用工作流，**只有需要临场决定路径**时才上智能体。够用就好，不是越高级越对。
</Callout>

## 小结

- **聊天机器人**是一问一答的直线，**工作流**是人写死的固定流程，**智能体**是自己转圈的循环——分界在「谁决定下一步」
- **Agent Loop** =「感知（看现状）→ 决策（想下一步）→ 行动（调工具做事）→ 观察（看结果）→ 回到感知」
- 让智能体不停推进、能自我纠偏的，正是「行动完回头观察、再重新决策」这个**闭环**——它是智能体的灵魂
- 三种系统排在**自主性**这把尺子上，从「人全程操控」滑到「系统自主」，智能体在最自主的一端
- 选型看任务：能问答就别上工作流，能写死流程就别上智能体——**只有需要临场决定路径**才真正需要智能体

（a）把一段产品说明从英文翻成中文。
（b）每天早上 9 点抓取昨日销售数据，套同一个模板生成日报邮件。
（c）帮我规划一次三天两夜的周末出游：要查天气、比价酒店、根据查到的情况随时调整行程。

<Answer>

（a）**聊天机器人**：单步、一问一答，把文字丢给模型拿回译文即可，路径不用临场决定，也不必调外部工具。
（b）**工作流**：要多步、也要调工具（抓数据），但**步骤是固定的**——每天都走「抓数据 → 套模板 → 发邮件」同一条路，人事先写死最稳最省，不必动用智能体。
（c）**智能体**：路径**得看中间结果临场决定**（天气不好就改室内行程、某家酒店满了就换一家），没法事先写死——这正是 Agent Loop「感知→决策→行动→观察」反复循环的主场。判断依据的关键一条就是「路径要不要临场决定」。

</Answer>

**练习 2（问答型）** 同事说：「我接了大模型、让它一步步按我写好的固定流程跑了五步，这就是智能体了吧？」这话对吗？请用本章的「一条检验线」纠正他。

<Answer>

不对。按人写死的固定流程跑五步，那是**工作流**，不是智能体。检验线是：这个系统会不会**行动完回头观察、再据此重新决策下一步**（即有没有 Agent Loop 那个闭环）？固定流程里每一步都是人安排好的，程序不会看中间结果自己改主意，所以没有「自主决策的循环」，称不上智能体。智能体的标志是自己转圈——感知→决策→行动→观察→再感知，由它按情况决定下一步。

</Answer>

**练习 3（问答型）** 用「失忆天才小屋」这个比喻，说说聊天机器人和智能体到底差在哪。智能体多出来的那个「循环」解决了天才的什么缺陷？

<Answer>

聊天机器人就是「隔着门缝问失忆天才一句、他答一句」——他读完纸条写段回复就完事，既记不住上次、又出不了屋、干不了实事。智能体则给这位天才装上了一个「感知→决策→行动→观察」的循环，外加能伸出屋外做事的「电话」（工具）：他不再答完就撒手，而是动手做一步、回头看看结果、再决定下一步，反复推进直到把活干完。这个循环解决的核心缺陷是：失忆天才本来「只会答话、干不了实事、做完一步就停」，循环让他能**自主地把一个多步任务一路推进到底**。

</Answer>

<GlossaryItem term="大模型">
  一个绝顶聪明但严重失忆的「纸条机器」：你给它一段文字（提示），它读完写一段回复，仅此而已——不记上次说过什么、不能上网、不能动手做事。本书把它比作关在一间没窗小屋里、出不去也记不住的失忆天才。它是聊天机器人、工作流、智能体共同的「大脑」。详见本章「一位关在小屋里的失忆天才」一节。
</GlossaryItem>

<GlossaryItem term="聊天机器人（Chatbot）">
  把用户的话直接丢给大模型、拿回一段回复就结束的系统，一问一答、没有下一步。像隔着门缝问失忆天才一句、他答一句。能干翻译、解释这类单步活，但碰到要多步推进、查实时信息的任务就抓瞎。详见本章「三种系统」一节。
</GlossaryItem>

<GlossaryItem term="工作流（Workflow）">
  由人事先写死步骤、每次都按同一条固定路线执行的系统（步骤
  A→B→C）。中间哪一步用到大模型都行，但走法是人定死的，程序不会看中间结果临场改主意。能干多步的活，却不会自己拐弯。详见本章「三种系统」一节。
</GlossaryItem>

<GlossaryItem term="智能体（Agent）">
  能自己感知处境、决定下一步、调用工具去做、再观察结果的系统；靠一个循环反复推进，直到把任务干完，不需要人把每一步都写死。它是把「只会答话」的失忆天才升级成「能把活干完的帮手」的形态，也是本书要亲手造出来的东西。详见本章「三种系统」一节。
</GlossaryItem>

<GlossaryItem term="自主性（Autonomy）">
  一个系统在多大程度上能自己决定下一步、而不靠人把每步都安排好。聊天机器人几乎为零（人问一句它答一句），工作流是中等（人写死流程），智能体最高（自己按情况循环决策）。三种系统就排在这把尺子上，从「人全程操控」滑到「系统自主」。详见本章「三种系统」一节。
</GlossaryItem>

<GlossaryItem term="Agent Loop（智能体循环）">

  观察（看结果），再回到感知。让本来只会答话的大模型，变成能自主把任务一步步推进到底、还能自我纠偏的智能体——它是智能体的灵魂。详见本章「智能体的灵魂」一节。
