## 同一张纸条，凭什么有时它死板有时它发疯

还是那位关在小屋里的失忆天才。你塞同一张纸条进去，他读完写回话。可你发现怪事：有时他像复读机，问十遍答得几乎一字不差；有时又像喝多了，同一个问题答得天马行空、每次都不一样。明明纸条没变，他怎么时而呆板时而放飞？

秘密在于：他写每个字之前，脑子里其实是一**堆**候选词，各自带着「我有多想写这个」的分数。怎么从这堆候选里**挑一个**落笔——是永远挑分最高的，还是按分数高低「掷个骰子」碰运气——由几个旋钮控制。旋钮拧死板，他就复读；拧松了，他就放飞。

这一章要弄清的就是这几个旋钮：它们怎么把「一堆带分数的候选」捏成最终落笔的那个字，什么时候该把它拧紧（要稳）、什么时候该松开（要花样）。搞懂了，你就能精准控制这位失忆天才的「脑回路松紧」。

## 从一堆分数到一个字：先看「一次采样」的全过程

模型写每个字时，并不是直接蹦出一个词，而是先给词表里**每一个**候选 token 打一个原始分数（叫 logits，有正有负、还不是概率），再一步步把这堆分数变成「最终落笔的那一个」。下面这张可控动画，把这「一次采样」拆成五步演给你看——先整体过一遍，后面几节再逐个旋钮细讲：

## 两条路：贪心永远挑最高，采样按概率掷骰子

从一摞概率里挑一个 token，最简单的办法叫 <Term def="挑 token 时永远选概率最高的那一个，不掺任何随机。结果完全确定：同一份提示每次输出一字不差。稳，但死板、容易复读，没有花样。">贪心解码（greedy decoding）</Term>：闭着眼睛选概率最高的那个，绝不掺随机。它的好处是**确定**——同一张纸条问十遍，答得一模一样；坏处是死板，容易复读、没新意。

另一条路叫 <Term def="挑 token 时不一定选最高的，而是按各候选的概率「掷骰子」随机挑——概率高的更容易被选中，但低的也有机会。结果有随机性：同一份提示可能给出不同回答，更有花样。">采样（sampling）</Term>：按各候选的概率「掷骰子」——概率 0.6 的候选大约六成机会被选，概率 0.1 的也有一成机会冒头。它带来**多样性**，但也带来不确定。本章后面所有旋钮（温度、top-p、top-k），调的都是「掷骰子」这件事：要么改骰子各面的大小，要么限制骰子只允许落在哪几个候选上。

## 温度：把分布捏陡还是捏平

第一个、也是最重要的旋钮是 <Term def="一个缩放原始分数的数 T，控制概率分布的「陡峭度」。T 越小，分布越陡（强者愈强，趋近贪心、确定）；T 越大，分布越平（人人有份，趋近均匀、发散）。它只改「挑 token 的随机度」，不改谁本来分高。">温度（temperature）</Term>。它做的事一句话：在 softmax 之前，把每个候选的原始分数都除以 $T$，再算概率。

- $T$ **小**（比如 0.2）：分数被放大，强者愈强——分布变得很**陡**，几乎总挑那个最高的，趋近贪心、输出确定。
- $T$ **大**（比如 1.5）：分数被压扁，人人有份——分布变得很**平**，挑谁都有可能，输出发散、花样多。

温度只改「挑 token 时的随机度」，**不改谁本来分高**。具体怎么改，下一节用公式推清楚。

## top-p 与 top-k：先把骰子的候选范围圈小

光有温度还不够：高温下连那些明显不靠谱的长尾候选也有机会被抽中，偶尔就蹦出奇怪的字。于是再加一道闸——先把候选范围圈小，只在「靠谱的那几个」里掷骰子。

最常用的是 <Term def="一种把采样范围圈小的方法（也叫 nucleus 核采样）：把候选按概率从高到低累加，只保留累积概率刚够到 p 的最小候选集（这一小撮叫「核」nucleus），其余长尾直接砍掉，再在核里按概率采。p 越小，候选越少、越稳。">top-p（核采样，nucleus sampling）</Term>：把候选按概率从高到低累加，只留累积概率刚够到 $p$ 的最小一撮（这一撮就叫「核」），剩下的长尾全砍掉，再在核里采。比如 `top-p=0.9` 就是「只在累积概率前 90% 的候选里掷骰子」。它的妙处是**自适应**——分布陡时核里就一两个，分布平时核里自动放宽到好几个。

另一个常见旋钮是 <Term def="另一种圈小采样范围的方法：固定只保留概率最高的前 k 个候选，其余砍掉，再在这 k 个里采。简单直接，但不像 top-p 那样随分布陡平自适应——分布很陡时 k 个里可能混进没用的，很平时又可能砍掉本该保留的。">top-k</Term>：简单粗暴，固定只留概率最高的前 $k$ 个，其余砍掉。它不像 top-p 自适应，所以实践中 top-p 更常用，常和温度搭配一起调。

## 一句话权衡：要稳调低，要花样调高

把上面几个旋钮合起来，本质是一道 <Term def="采样参数的核心取舍：调低温度 / 收紧 top-p → 输出更确定、更稳、可复现，但死板、缺花样；调高温度 / 放宽 top-p → 输出更多样、更有创意，但更不稳、偶尔跑偏。按任务在两端之间取舍。">确定性 ↔ 多样性权衡</Term>。要事实、要格式、要可复现（如解析 JSON、做数学题），就把温度调低、top-p 收紧，让它稳；要创意、要花样（如写诗、起名、头脑风暴），就把温度调高、top-p 放宽，让它放飞。下一节先把温度的数学讲透，再回到这个权衡。

## 数学：带温度的 softmax 到底在算什么

### 符号表

本节首次出现的符号，一行一个：

- $z_i$：第 $i$ 个候选 token 的 **logit**（模型吐出的原始分数，有正有负，未归一）
- $p_i$：第 $i$ 个候选被选中的**概率**（$0 \le p_i \le 1$，且全部相加为 1）
- $T$：**温度**（一个正数，$T > 0$，控制分布陡平）
- $V$：**词表**（所有候选 token 的集合）；$j$ 遍历 $V$ 里每个候选

### 第一步：普通 softmax 把分数变概率

logits 有正有负、又不相加为 1，没法直接当概率。softmax 干的就是「转成概率」这件事：

$$
p_i = \frac{\exp(z_i)}{\sum_{j \in V} \exp(z_j)}
$$

这个式子在说：把每个候选的分数 $z_i$ 取一次指数 $\exp$（让它一定为正、且分数越高放大越狠），再除以「所有候选指数之和」做归一——这样每个 $p_i$ 都落在 0 到 1 之间、全部相加恰好等于 1，成了一份合法的概率分布。

### 第二步：除以温度，就得到带温度的 softmax

温度做的事极简单：在取指数**之前**，把每个分数都除以 $T$。把上式里的 $z_i$ 换成 $z_i / T$：

$$
p_i = \frac{\exp(z_i / T)}{\sum_{j \in V} \exp(z_j / T)}
$$

这个式子在说：和普通 softmax 一模一样，只多了一步「先把每个分数除以温度 $T$ 再算」。$T=1$ 时它就退化成普通 softmax；$T$ 改了，整份分布的陡平就跟着变——下面看三种极限。

### 三种极限：T 怎么影响分布的陡峭度

把温度推到三个极端，就能摸清它的脾气。关键看**两个候选的分数差**在除以 $T$ 后被放大还是缩小。

**极限一：$T \to 0$（温度趋零）。** 分数除以一个极小的数，差距被无限放大。设 $z_a$ 是最高分、$z_b$ 是次高分（$z_a > z_b$），看它俩概率的比值：

$$
\frac{p_a}{p_b} = \exp\!\left(\frac{z_a - z_b}{T}\right)
$$

这个式子在说：两个候选的概率之比，等于「它们分数差除以 $T$」再取指数。当 $T \to 0$，指数里的 $(z_a - z_b)/T \to +\infty$，所以比值 $\to +\infty$——最高分那个把概率几乎全占了，其余趋近 0。**分布塌成一根独大**，等价于贪心解码：永远挑最高的那个。

**极限二：$T = 1$（温度为一）。** 公式退回普通 softmax，不做任何额外缩放——这是「原汁原味」的概率分布，既不额外变陡也不额外变平。

**极限三：$T \to \infty$（温度趋大）。** 分数除以一个极大的数，所有 $z_j / T \to 0$，于是每个 $\exp(z_j / T) \to \exp(0) = 1$。代回公式，分子分母都成了「一堆 1」：

$$
p_i = \frac{\exp(z_i / T)}{\sum_{j \in V} \exp(z_j / T)} \;\longrightarrow\; \frac{1}{|V|}
$$

这个式子在说：温度推到无穷大时，每个候选的概率都趋近 $1 / |V|$（$|V|$ 是候选个数）——也就是**均匀分布**，谁的原始分数高已经无所谓了，挑谁都一样，输出最发散。

一句话收口：**$T$ 越小越陡（趋贪心、确定），$T=1$ 是原始分布，$T$ 越大越平（趋均匀、发散）**。下一节你就能亲手拖这个 $T$，看分布条形图实时变陡变平。

## 动手：给 tinyagent 加上 sample_next 和 llm()

上一章我们把「一份提示」组装成了 `messages`，但还没真把它喂给模型。这一章就补上「采样」这块砖，并把它收进 `llm()` 薄封装里。先把数学落成能跑的代码——一个纯 Python 的 `sample_next`，亲手实现带温度的 softmax：

```python
import math
import random


def softmax(logits, temperature):
    # 带温度的 softmax：每个分数先除以 T，再做 softmax（§4 那个公式）
    scaled = [z / temperature for z in logits]
    m = max(scaled)  # 减最大值防 exp 溢出（数值稳定，不改结果）
    exps = [math.exp(s - m) for s in scaled]
    total = sum(exps)
    return [e / total for e in exps]
```

`softmax` 就是 §4 公式的逐字翻译：先把每个 logit 除以 `temperature`（这一步是温度的全部魔法），再取指数、归一。减最大值 `m` 只是数值技巧——分子分母同乘一个常数，结果不变，却能避免大指数溢出。接着实现一次完整采样：

```python
def sample_next(logits, temperature=1.0, top_p=1.0, seed=None):
    probs = softmax(logits, temperature)  # ① 带温度的概率分布
    # ② top-p：按概率从高到低累加，只留累积刚够 top_p 的最小候选集
    ranked = sorted(range(len(probs)), key=lambda i: probs[i], reverse=True)
    kept, acc = [], 0.0
    for i in ranked:
        kept.append(i)
        acc += probs[i]
        if acc >= top_p:
            break
    # ③ 在存活候选里按概率重新归一，再掷骰子抽一个
    weights = [probs[i] for i in kept]
    rng = random.Random(seed)  # 传 seed 可复现，便于调试
    r = rng.random() * sum(weights)
    for i, w in zip(kept, weights):
        r -= w
        if r <= 0:
            return i
    return kept[-1]
```

这就是动画五步的代码版：①`softmax` 出带温度的分布；②`top_p` 把候选从高到低累加、砍掉长尾留下「核」；③在核里按权重「掷骰子」（`r` 从 `[0, 权重和)` 里取一个随机点，沿权重走到哪根就抽中谁）。传 `seed` 能复现同一次抽样，方便调试。跑一遍验证三件事——低温分布更陡、`top_p` 越小存活候选越少、抽样确实能出结果：

```python
logits = [3.2, 2.6, 1.8, 1.1, 0.6, 0.2, -0.4, -1.0]
p_low = softmax(logits, 0.3)   # 低温：最高概率 ≈ 0.87（很陡）
p_high = softmax(logits, 1.5)  # 高温：最高概率 ≈ 0.36（很平）
print(round(max(p_low), 2), round(max(p_high), 2))  # 0.87 0.36
idx = sample_next(logits, temperature=1.0, top_p=0.9, seed=42)
print("抽中下标:", idx)  # 抽中下标: 1
```

输出印证了公式：低温下最高概率 0.87（分布陡），高温下只剩 0.36（分布平）；`sample_next` 也确实掷出了一个候选。最后把它收进 `tinyagent` 的 `llm()` 薄封装——这层接口才是全书后面真正调用的入口，采样参数从这里透传给底层模型：

## 容易踩的坑

<Callout type="trap">
  **现象**：解析 JSON、做数学题、要可复现的结果，却把温度开到 1
  甚至更高，结果时对时错、格式飘忽。**原因**：高温让分布变平、采样随机性变大，模型在「该挑最稳那个」时反而去掷骰子，偶尔就选错字、写坏格式。**修法**：要确定性的任务把温度调到接近
  0（或直接用贪心），让它老老实实挑最高分的那个。
</Callout>

<Callout type="trap">
  **现象**：以为「调高温度 =
  模型变聪明、答得更好」，于是写代码、答事实题也一律开高温。**原因**：温度只改「挑
  token
  的随机度」，**不改谁本来分高**——它给的是花样不是正确率。高温只是让它更敢选冷门候选，并不会让它更懂。**修法**：把温度当「创意旋钮」而非「智力旋钮」：要花样才调高，要准确就调低。
</Callout>

<Callout type="trap">
  **现象**：温度和 top-p
  一起拉到很高想要「极致创意」，结果输出开始语无伦次、蹦出乱码或不相干的字。**原因**：高温已经把长尾候选的概率顶上来，top-p
  又放得很宽，等于允许在一大堆离谱候选里掷骰子，自然容易抽到垃圾。**修法**：两个旋钮别同时拉满；想要创意，常见做法是温度适度调高（如
  0.8~1.0）、top-p 收在 0.9 左右兜住长尾。
</Callout>

## 小结

- 从一堆 logits 到一个字要走五步：logits → softmax → 按温度重塑 → top-p 截断 → 掷骰子抽一个
- 挑 token 有两条路：**贪心**永远挑最高（确定、死板）、**采样**按概率掷骰子（多样、不稳）
- **温度 $T$** 捏分布陡平：$T \to 0$ 趋贪心/确定、$T=1$ 原始分布、$T \to \infty$ 趋均匀/发散
- **top-p**（核采样）只在累积概率前 $p$ 的最小候选集里采，自适应砍长尾；**top-k** 固定留前 $k$ 个
- 一句话权衡：要稳、要格式、要可复现就调低；要创意、要花样就调高——tinyagent 用 `sample_next` + `llm(messages, temperature, top_p)` 把这套落成代码

**练习 1（手推题）** 三个候选的 logits 是 $z = [2,\ 1,\ 0]$。请用带温度的 softmax 公式，分别算出 $T=1$ 和 $T \to \infty$ 时三个候选的概率（$T \to \infty$ 不必精算，说出趋向即可），并说明哪个温度下分布更平。

<Answer>

带温度 softmax：$p_i = \exp(z_i/T) \big/ \sum_j \exp(z_j/T)$。

- $T = 1$：分子是 $\exp(2),\ \exp(1),\ \exp(0) \approx 7.39,\ 2.72,\ 1.00$，和约 $11.11$。所以 $p \approx [0.665,\ 0.245,\ 0.090]$——最高的占大头。
- $T \to \infty$：每个 $z_i/T \to 0$，于是每个 $\exp(z_i/T) \to 1$，三个概率都趋近 $1/3 \approx 0.333$，即 $[0.333,\ 0.333,\ 0.333]$。

$T \to \infty$ 时分布**更平**（趋近均匀），$T=1$ 时分布更陡（最高那个明显占优）。这正是「温度越高分布越平」的手算验证。

</Answer>

**练习 2（top-p 手算题）** 某步五个候选的概率（已从高到低排好）是 $[0.55,\ 0.25,\ 0.12,\ 0.05,\ 0.03]$。若 `top-p = 0.9`，核采样会保留哪几个候选？为什么？

<Answer>

从高到低累加，直到累积概率首次 $\ge 0.9$：

- 加第 1 个：$0.55 < 0.9$（不够，继续）
- 加第 2 个：$0.55 + 0.25 = 0.80 < 0.9$（不够，继续）
- 加第 3 个：$0.80 + 0.12 = 0.92 \ge 0.9$（够了，停）

所以保留**前 3 个**（概率 0.55、0.25、0.12），后两个长尾（0.05、0.03）被砍掉。然后在这 3 个里按概率重新归一、掷骰子。这就是 top-p「只在累积前 90% 的核里采」的含义。

</Answer>

**练习 3（改 Demo / 选型题）** 你在用 tinyagent 做两个任务：(a) 把用户输入解析成固定格式的 JSON；(b) 给一篇文章起 10 个不同风格的标题。分别该把 `sample_next` 的 `temperature` 和 `top_p` 往哪个方向调？说出理由。

<Answer>

- **(a) 解析 JSON**：调**低** `temperature`（接近 0，甚至直接贪心）、`top_p` 收紧（如 0.3~0.5）。这类任务要的是确定、可复现、格式不能错——让模型老老实实挑最稳的候选，别掷骰子，否则偶尔就写坏花括号或字段。
- **(b) 起 10 个不同标题**：调**高** `temperature`（如 0.9~1.2）、`top_p` 放宽（如 0.95）。要的是多样、有花样——温度高分布变平、采样随机性大，才能十个各不相同；top-p 放宽但别拉满（兜住长尾，免得蹦出乱七八糟的字）。

一句话：要稳调低、要花样调高，这就是「确定性 ↔ 多样性权衡」落到旋钮上的样子。

</Answer>

<Glossary>

<GlossaryItem term="贪心解码（greedy decoding）">
  挑下一个 token
  时永远选概率最高的那一个，不掺任何随机。结果完全确定——同一份提示每次输出一字不差。稳，但死板、容易复读、没花样。详见本章「两条路」一节。
</GlossaryItem>

<GlossaryItem term="采样（sampling）">
  挑下一个 token
  时不一定选最高的，而是按各候选的概率「掷骰子」随机挑——概率高的更容易中，低的也有机会冒头。带来多样性，但也带来不确定。本章的温度、top-p、top-k，调的都是这件「掷骰子」的事。详见本章「两条路」一节。
</GlossaryItem>

<GlossaryItem term="温度（temperature）">
  一个正数 $T$，控制概率分布的「陡峭度」：在 softmax 前把每个分数除以 $T$。$T$
  越小分布越陡（强者愈强，趋近贪心、确定）、$T$
  越大分布越平（人人有份，趋近均匀、发散）；$T=1$ 是原始分布。它只改「挑 token
  的随机度」，不改谁本来分高。详见本章「温度」与「数学」两节。
</GlossaryItem>

<GlossaryItem term="top-p（核采样，nucleus sampling）">
  一种把采样范围圈小的方法：把候选按概率从高到低累加，只保留累积概率刚够到 $p$
  的最小候选集（这一小撮叫「核」nucleus），其余长尾砍掉，再在核里按概率采。$p$
  越小候选越少、越稳；好处是随分布陡平**自适应**——分布陡时核里就一两个，平时自动放宽。详见本章「top-p
  与 top-k」一节。
</GlossaryItem>

<GlossaryItem term="top-k">
  另一种圈小采样范围的方法：固定只保留概率最高的前 $k$ 个候选，其余砍掉，再在这
  $k$ 个里采。简单直接，但不像 top-p 那样随分布自适应，实践中常被 top-p
  替代。详见本章「top-p 与 top-k」一节。
</GlossaryItem>

<GlossaryItem term="确定性 ↔ 多样性权衡">
  采样参数的核心取舍：调低温度 / 收紧 top-p →
  输出更确定、更稳、可复现，但死板缺花样（适合解析 JSON、做数学题）；调高温度 /
  放宽 top-p →
  输出更多样、更有创意，但更不稳、偶尔跑偏（适合写诗、起名、头脑风暴）。按任务在两端取舍。详见本章「一句话权衡」一节。
</GlossaryItem>

</Glossary>
