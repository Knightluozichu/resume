## 核心概念再梳理

在往前走之前，把最关键的几个概念再串一遍——这些是面试必问、项目必用的：

### Activity 与 Fragment 生命周期
- `onCreate` → `onStart` → `onResume` → `onPause` → `onStop` → `onDestroy`
- Fragment 的生命周期与 Activity 联动，尤其是 `onAttach`/`onDetach` 和 `onCreateView`/`onDestroyView`
- 核心理解：**配置变更（旋转屏幕）不会销毁 Activity**——它会走 `onDestroy` 再 `onCreate`，但你需要用 `onSaveInstanceState` 或 ViewModel 保住数据

### RecyclerView——所有列表的基石
- `RecyclerView.Adapter` + `ViewHolder` + `LayoutManager` 三部曲
- `notifyItemChanged/Inserted/Removed` 精确更新，避免全量刷新
- ListAdapter + DiffUtil 自动计算差异（现代推荐做法）

### Room——数据库不再难
- Entity、DAO、Database 三层抽象
- `@Insert`、`@Update`、`@Delete`、`@Query` 编译时校验
- LiveData/Flow 与 Room 的响应式查询——数据一变，UI 自动更新

### MVVM——架构的骨架
- Model（数据）→ ViewModel（逻辑）→ View（界面）
- ViewModel 在配置变更时存活，不持有 View 引用
- LiveData / StateFlow 驱动 UI 更新，单向数据流

### WorkManager——可靠的后台任务
- 保证执行（即使 App 退出或设备重启）
- 约束条件（网络、电量、空闲状态）
- 链式任务编排（A 完成 → B 开始）

## 下一步学什么

这本书是"基础"——但不是"全部"。真正的 Android 开发栈，你还需要：

</Callout>

</Callout>

<Callout type="tip">
  **面试前必串的三条线**：生命周期 + 列表(RecyclerView) + 持久化(Room/ViewModel)——本书 Part1–3 已覆盖，能白板讲清楚即可应付多数初级岗。
</Callout>

### Jetpack Compose——声明式 UI 的未来
Google 已经明确 Compose 是 Android UI 的未来方向。它的核心思想是用 Kotlin 函数描述 UI——没有 XML，没有 findViewById，只有 `@Composable` 函数。学 Compose 不是"多学一个技能"，而是**换一种思维方式**——从命令式（告诉系统怎么做）到声明式（描述你想要什么）。

### Kotlin 协程与 Flow——异步编程的标准答案
书里用了 Thread、Handler、AsyncTask 来讲解后台任务——这些都是理解异步编程的基石。但现代 Android 开发的正确答案是**协程**。`launch`、`async`、`withContext`——挂起而非阻塞，结构化并发而非散养线程。`Flow` 是协程版的响应式流——比 LiveData 更灵活，更适合数据层到 UI 层的传递。

### 依赖注入——Hilt / Koin
当你的 App 有几十个 ViewModel、Repository、Service 时，手动 new 对象会变成一场噩梦。Hilt（基于 Dagger）和 Koin（纯 Kotlin DSL）让对象的创建和注入自动化——你只声明"我需要这个"，框架帮你准备好。这是中等规模以上 App 的标配。

### Clean Architecture——让代码经得起时间考验
MVVM 处理了 UI 层的架构。当 App 大到需要分层（数据层、领域层、展示层），Clean Architecture 提供完整的分层方案——依赖倒置、用例驱动、接口隔离。这不是"用不用"的问题，而是"什么时候开始需要"的问题。

### 测试——不是可选项
单元测试（JUnit + Mockito）、UI 测试（Espresso / Compose Testing）、端到端测试——测试不是"写完代码再补"，而是**开发流程的一部分**。测试覆盖的代码让你敢于重构、敢于升级依赖、敢于加新功能。

## 真正的起点

这本书的结束不是终点。它是一个起点——你有了坚实的地基，知道了怎么"造房子"，接下来就是选择方向：是做漂亮的 UI（Compose + 动画）、做稳定的架构（Clean Architecture + 测试）、还是做深入的底层（性能优化、NDK）。

Android 开发是一个持续进化的领域——每年都有新的 API、新的范式、新的工具。但底层的东西不变：对用户负责的心态、写可读代码的习惯、定位问题的能力、持续学习的动力。这些，是任何一本书都教不了，但你在这一年里已经慢慢磨练出来的东西。

继续写代码，继续造东西。最好的学习永远是——**做一个自己想用的 App**。

<Glossary>

<GlossaryItem term="MVVM">
  Model-View-ViewModel 架构模式。Model 管理数据，ViewModel 持有 UI 状态和业务逻辑（在配置变更时存活），View 观察 ViewModel 的状态变化并更新 UI。单向数据流：View → ViewModel → Model → ViewModel → View。
</GlossaryItem>

<GlossaryItem term="Jetpack Compose">
  Google 推出的声明式 UI 框架。用 Kotlin 函数描述 UI，无需 XML。核心概念：Composable 函数、状态驱动重组、Material 3 组件。是 Android UI 开发的未来方向。
</GlossaryItem>

<GlossaryItem term="Kotlin 协程 (Coroutines)">
  Kotlin 的异步编程方案。核心特点：挂起而非阻塞（suspend 关键字）、结构化并发（父协程取消则子协程全部取消）、轻量级（百万协程不是梦）。配合 Flow 实现响应式数据流。
</GlossaryItem>

<GlossaryItem term="依赖注入 (Dependency Injection)">
  对象不自己创建依赖，而是由外部"注入"。好处：解耦、可测试、可替换实现。Android 常用方案：Hilt（基于 Dagger 的官方方案）和 Koin（轻量级纯 Kotlin DSL 方案）。
</GlossaryItem>

</Glossary>
