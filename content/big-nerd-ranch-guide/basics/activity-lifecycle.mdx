## 你的 Activity 是「活的」——它有生命周期

用户切到别的 App、旋转屏幕、按 Back 键——这些操作都会触发 <Term def="系统按固定顺序调用的六个回调方法，通知 Activity 当前处于创建、可见、前台、暂停、停止或销毁状态。">Activity 生命周期</Term>。不懂生命周期，就容易内存泄漏、丢状态、莫名其妙崩溃。

用**相机**类比：onCreate 是开机，onResume 是进入拍摄，onPause 是接电话暂时放下，onStop 是盖上镜头盖，onDestroy 是关机。

## 六大生命周期回调

| 回调 | 调用时机 | 典型用途 |
|---|---|---|
| `onCreate()` | Activity 首次创建 | 加载布局、初始化变量 |
| `onStart()` | Activity 即将可见 | 注册可见时才需要的监听 |
| `onResume()` | 前台可交互 | 开启动画、恢复相机预览 |
| `onPause()` | 失去焦点 | 暂停动画、保存轻量状态 |
| `onStop()` | 完全不可见 | 释放重资源 |
| `onDestroy()` | Activity 被销毁 | 最终清理 |

### Stepper：A 启动全屏 B 时发生了什么

1. **A.onPause()** —— A 失去焦点，但仍部分可见（若 B 非全屏）
2. **B.onCreate() → onStart() → onResume()** —— B 到前台
3. **A.onStop()** —— A 完全不可见

按 Back 返回时顺序相反：B.onPause → A.onRestart/onStart/onResume → B.onStop/onDestroy。

### Stepper：旋转屏幕（配置变更）

1. onPause → onStop → **onDestroy**（旧实例销毁）
2. onCreate → onStart → onResume（新实例创建）

实例被重建——成员变量默认清空，必须靠 ViewModel 或 `onSaveInstanceState` 保状态（下一章详讲）。

<Callout type="trap">
  **onPause 是最后保证执行的**——内存不足时系统可能在 onPause 后就杀进程，onStop/onDestroy 不一定跑到。持久化关键数据应在 onPause 完成。
</Callout>

<Callout type="trap">
  每个回调**第一行必须** `super.onCreate()` 等——父类负责 Fragment 恢复、主题初始化；跳过会 `SuperNotCalledException` 或界面异常。
</Callout>

<Callout type="trap">
  **现象**：onCreate 里做网络请求 → 启动黑屏数秒。**原因**：onCreate 在主线程。**修法**：协程/ViewModel 发起请求，onCreate 只做 UI 绑定。
</Callout>

## 小结

- 六回调：Create → Start → Resume ↔ Pause/Stop → Destroy
- 启动第二个 Activity：先 Pause 当前，再 Create 新的，最后 Stop 旧的
- 旋转 = 销毁 + 重建，需状态保持机制
- 耗时操作勿放主线程生命周期回调

<Exercises>

**练习 1（画图）** 画出从启动到按 Home 再切回、最后按 Back 退出的完整回调链。

<Answer>
启动：onCreate→onStart→onResume。按 Home：onPause→onStop。切回：onRestart→onStart→onResume。按 Back：onPause→onStop→onDestroy。
</Answer>

**练习 2（场景）** 来电时你的全屏 Activity 会走到哪个状态？用户挂断后呢？

<Answer>
来电弹出系统界面：onPause（可能仍 Stopped 若被完全盖住）。挂断返回：onRestart/onStart/onResume。若进程被杀则需重新 onCreate。
</Answer>

</Exercises>

<Glossary>

<GlossaryItem term="Activity生命周期">
  Activity 从创建到销毁经历的回调序列。App 在对应回调里做初始化与资源释放。
</GlossaryItem>

<GlossaryItem term="配置变更">
  屏幕旋转、语言切换等导致 Activity 重建的系统事件。默认会走 onDestroy + onCreate。
</GlossaryItem>

</Glossary>
