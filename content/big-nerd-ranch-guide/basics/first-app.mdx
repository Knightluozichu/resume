## 你的第一个 Android 应用——从零到跑起来

你以前用的手机 App，背后都是别人写的代码。这一章，你来做那个人——从下载工具、点几下按钮，到自己的手机上弹出一个新界面，全程不超过一小时。

这件事的核心是一条**工厂流水线**：你写 Kotlin 代码和 XML 布局（原料），经过一道叫 Gradle 的工位把所有东西打包压缩，最后输出一个叫 <Term def="Android Package，Android 应用的安装包文件，本质是一个 zip 压缩包，里面装着编译好的代码、资源文件、清单文件等所有应用运行需要的东西。">APK</Term> 的成品。把这个 APK 装进手机，点一下图标，你就看到了自己写的界面。

没有这条流水线会怎样？你写的几百行 Kotlin 代码永远只是一堆纯文本文件，手机一个字也不认识。这一章的每一节都在帮你把这条流水线走通一遍。

## 概念讲解

### Android Studio：你的工作台

<Term def="Google 官方提供的 Android 应用集成开发环境（IDE），基于 IntelliJ IDEA，内置了代码编辑器、模拟器、调试器、性能分析工具等所有开发 Android 应用需要的工具。">Android Studio</Term> 是你写 Android 应用的"主战场"。它集成了代码编辑器、模拟器、调试工具和 Gradle 构建系统——你不需要单独安装任何其他东西就能完整地开发、运行、调试一个 Android 应用。

下载安装 Android Studio 后，第一次启动会让你下载 Android SDK——这是一套包含 Android 各个版本系统镜像、开发工具的命令行工具集。**从 Android Studio 官网下载最新稳定版即可**，安装向导会自动帮你把 SDK 配好。

### 项目结构：三间房各管什么事

创建新项目时选"Empty Views Activity"，你会得到一个叫 `app` 的模块。打开左侧项目面板，把视图从 `Android` 切换到 `Project`，你会看到一个标准 Gradle 项目的完整目录树。核心的三个目录：

```
app/
├── manifests/
│   └── AndroidManifest.xml    ← 应用的"身份证"
├── java/
│   └── com.example.myapp/
│       └── MainActivity.kt    ← 你的 Kotlin 代码
└── res/
    ├── layout/
    │   └── activity_main.xml  ← 界面的样子
    ├── values/
    │   ├── strings.xml        ← 文字资源
    │   └── colors.xml         ← 颜色资源
    └── drawable/              ← 图片资源
```

`manifests/` 里放的是 <Term def="每个 Android 应用必须包含的 XML 清单文件，向系统声明应用包含哪些组件（Activity、Service 等）、需要哪些权限、支持哪些设备特性。系统在安装应用前会先读这个文件。">AndroidManifest.xml</Term>——应用的"身份证"。它告诉 Android 系统：这个应用叫什么名字、图标是哪个、里面有哪些界面（Activity）、需要哪些权限（比如联网、拍照）。

`java/`（或 `kotlin/`）放你的 Kotlin 源代码。每个界面通常对应一个 <Term def="Android 四大组件之一，代表一个用户可以交互的屏幕。每个 Activity 负责一个界面，管理界面的创建、显示和用户交互。可以理解为应用中一个"页面"。"">Activity</Term> 类——`MainActivity` 就是你应用的第一个界面。

`res/` 是资源目录。`layout/` 里放布局 XML 文件，定义界面上按钮、文字摆在哪里、长什么样；`values/` 里放字符串、颜色等常量；`drawable/` 里放图片。这种把代码和资源**分开管理**的方式是 Android 的核心设计理念之一。

### Gradle：流水线的工位

你点"Run"按钮之后，背后真正干活的是 <Term def="Android 官方使用的自动化构建工具，负责把 Kotlin/Java 源码编译成字节码、再转换成 DEX 格式、打包资源文件、最终生成 APK。构建规则写在 build.gradle 文件里。">Gradle</Term>。它是把源码变成 APK 的**自动化流水线**：

1. **编译**：把 Kotlin 代码编译成 Java 字节码（`.class`），再转成 Android 专用的 DEX 格式（`.dex`）
2. **打包**：把 DEX 文件 + 编译后的资源 + AndroidManifest.xml 等打成一个未签名的 APK
3. **签名**：给 APK 加上数字签名（debug 模式下 Android Studio 自动用 debug 签名）
4. **安装**：通过 adb 把 APK 推送到模拟器或真机上安装

Gradle 的配置写在 `build.gradle` 文件里。你有两个：项目根目录的（全局配置）和 `app/` 目录下的（当前模块配置）。`app/build.gradle` 里最重要的几项：

```kotlin
android {
    namespace = "com.example.myapp"
    compileSdk = 35

    defaultConfig {
        applicationId = "com.example.myapp"
        minSdk = 24
        targetSdk = 35
        versionCode = 1
        versionName = "1.0"
    }
}

dependencies {
    implementation("androidx.core:core-ktx:1.15.0")
    // ... 你的第三方库写在这里
}
```

`applicationId` 是应用的全球唯一标识，`minSdk` 是最低支持的 Android 版本，`dependencies` 里列出你依赖的第三方库。

## 动手：走通你的第一条流水线

下面创建一个新项目并运行它。每一步都配合实际操作。

> 猜一猜：点 Run 到手机上弹出界面，中间最快的步骤是哪一步？最慢的呢？做完下面全流程再看答案。

<Stepper>
  <Step title="① 创建新项目">
    打开 Android Studio → New Project → 选 "Empty Views Activity" → Next。
    在配置页填好：Name = "MyFirstApp"，Package name = "com.example.myfirstapp"（保持默认即可），Language = Kotlin，Minimum SDK = API 24。
    点 Finish，等待 Gradle 同步完成（底部状态栏显示 "Gradle sync finished"）。
  </Step>
  <Step title="② 认识 MainActivity 和布局文件">
    打开 `app/java/.../MainActivity.kt`，你会看到：

    ```kotlin
    class MainActivity : AppCompatActivity() {
        override fun onCreate(savedInstanceState: Bundle?) {
            super.onCreate(savedInstanceState)
            setContentView(R.layout.activity_main)
        }
    }
    ```

    `onCreate` 是 Activity 的入口，`setContentView(R.layout.activity_main)` 把布局文件加载进来。

    再打开 `app/res/layout/activity_main.xml`——这是界面的定义。里面有一个 `TextView`，显示 "Hello World!"。
  </Step>
  <Step title="③ 创建模拟器">
    点顶部工具栏的 Device Manager 图标（手机形状）→ Device Manager → 点 "+" → Create Virtual Device。
    选一个设备型号（推荐 Pixel 6），选系统镜像（推荐 Tiramisu API 33 或更高），一路 Next → Finish。
  </Step>
  <Step title="④ 运行！">
    工具栏上选中你刚创建的模拟器 → 点绿色三角形 ▶ Run。
    看底部 Build 面板：Gradle 开始编译、打包、安装——和上面讲的流水线完全一致。
    模拟器启动后，你会在屏幕上看到 "Hello World!"。
  </Step>
</Stepper>

<Callout type="tip">
  如果模拟器启动很慢，可以在 `app/build.gradle` 的 `android` 块里把 `compileSdk` 和 `targetSdk` 改成你已下载的最高版本（Tools → SDK Manager 查看）。另外，**首次运行 Gradle 会下载很多依赖**，耐心等它走完；第二次就快多了。
</Callout>

<Callout type="trap">
  **现象**：点 Run 后报错 `The application could not be installed: INSTALL_FAILED_INSUFFICIENT_STORAGE`。
  **原因**：模拟器存储空间不足——模拟器的系统分区或数据分区满了。
  **修法**：在 Device Manager 里找到该模拟器 → 点右侧下拉箭头 → Wipe Data，清除模拟器数据后重试。或者直接删掉这个模拟器重新建一个。
</Callout>

<Callout type="trap">
  **现象**：代码里 `R.layout.activity_main` 飘红，提示 `Unresolved reference: R`。
  **原因**：资源文件（比如布局 XML）有语法错误，导致 R 文件生成失败——Gradle 编译资源时出错，整个 R 类就没生成。
  **修法**：检查 `res/layout/` 下的 XML 文件有没有语法错误（最常见：标签忘闭合、属性名拼错）。打开 Build → Build Output 看详细错误信息。
</Callout>

## 代码逐段拆解

### MainActivity.kt：界面的大脑

```kotlin
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
    }
}
```

逐行看：`package` 声明这个文件属于哪个包——和你在创建项目时填的包名一致。`import` 导入需要用到的类，`AppCompatActivity` 是 Android 官方兼容库提供的一个基类（兼容老版本 Android）。`class MainActivity : AppCompatActivity()` 声明你的 MainActivity **继承**自 AppCompatActivity——继承了它所有现成的功能，你只需要重写（override）你需要改的部分。

`onCreate(savedInstanceState: Bundle?)` 是 Activity 生命周期的第一个回调——系统创建你的界面时首先调它。`super.onCreate(savedInstanceState)` 先让父类把该做的初始化做了（**这条必须放第一行**），然后 `setContentView(R.layout.activity_main)` 把你写的布局 XML 加载进来，变成屏幕上可见的按钮和文字。

`R.layout.activity_main` 里的 `R` 是 Android 自动生成的一个类——你放进 `res/` 的每一个资源，Gradle 都会在 `R` 里生成一个对应的整数 ID，你的 Kotlin 代码通过这些 ID 来引用资源。

### activity_main.xml：界面的骨架

```xml
<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintBottom_toBottomOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>
```

`ConstraintLayout` 是一种布局容器——它里面可以放各种控件（按钮、文字、图片），并通过"约束"来控制它们的位置。`android:layout_width="match_parent"` 意思是宽度撑满整个父容器（这里是屏幕）。`TextView` 是显示文字的控件，`wrap_content` 意思是"内容的尺寸多大我就多大"——文字有多宽，控件就多宽。

四个 `app:layout_constraint...` 属性把 TextView 的上下左右都约束在父容器中央——效果就是文字居中显示。

## 小结

- Android Studio + Gradle 构成一条自动化流水线：源码 → 编译 → DEX 转换 → 打包 → 签名 → APK
- 项目三间房：`manifests/` 是身份证，`java/` 是代码，`res/` 是资源——三个分开管理
- `MainActivity.onCreate()` 是应用的入口，`setContentView()` 把布局加载到屏幕
- `R.layout.activity_main` 是布局文件的 ID，Gradle 自动生成的 `R` 类管理所有资源引用
- 模拟器需要单独创建；首次运行 Gradle 较慢是正常现象

<Exercises>

**练习 1（改代码型）** 打开 `activity_main.xml`，把 `TextView` 的 `android:text` 改成 `"这是我的第一个应用！"`，再把文字大小改成 `24sp`（加 `android:textSize="24sp"`）。运行看效果，解释 `sp` 是什么意思（提示：查 "sp vs dp"）。

<Answer>

```xml
<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="这是我的第一个应用！"
    android:textSize="24sp"
    ... />
```

`sp`（scale-independent pixels）是 Android 的字体大小单位。和 `dp` 一样会根据屏幕密度自动缩放，但 **sp 额外会响应用户在系统设置里调的字体大小**——如果用户在系统中把字体调大了，用 sp 单位的文字也会跟着变大。这是无障碍设计的基本要求：文字大小必须用 sp，其他尺寸用 dp。

</Answer>

**练习 2（问答型）** Gradle 在构建过程中做了四件事：编译→打包→签名→安装。如果把"签名"这一步跳过（用未签名的 APK），能安装到手机上吗？为什么？

<Answer>

不能安装。Android 系统要求所有 APK 都必须经过数字签名才能安装——即使只是在自己手机上调试。签名的作用是标识应用的作者身份，系统通过签名来判断"这次安装的更新包是不是和上次安装的同一个作者"。Android Studio 在开发模式下会自动使用一个 **debug 签名**（一个通用的调试证书），所以你不感知，但它确实签了。正式发布到应用商店时，你必须用自己的 release 签名。

</Answer>

**练习 3（问答型）** 在 Android Studio 中把项目视图从 `Android` 切到 `Project`，找到 `app/build.gradle` 文件。把 `minSdk` 的值改成 `21`，然后 Sync Now。这个改动意味着什么？如果你的应用用了一个 API 24 才引入的功能，会发生什么？

<Answer>

`minSdk = 21` 意味着你的应用最低支持 Android 5.0（API 21）的设备。如果把 minSdk 改成比实际用到的 API 最低版本更低，编译不会报错——但如果你在运行 Android 5.0 的设备上调用了 API 24 才引入的方法（比如 `java.time.LocalDate`），运行时会抛出 `NoSuchMethodError` 或 `NoClassDefFoundError` 崩溃。Google 会通过 lint 检查给出警告：`Call requires API level 24 (current min is 21)`。解决办法是用 `if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.N)` 做运行时版本判断——这个在第 7 章会详细讲。

</Answer>

</Exercises>

<Glossary>

<GlossaryItem term="APK">
  Android 应用的安装包文件，本质是一个 zip 压缩包。里面装着编译好的 .dex 字节码文件、res 目录下的所有资源、AndroidManifest.xml 清单文件、以及签名信息。把 APK 传进手机点安装，你的应用就出现在桌面上了。详见本章"你的第一个 Android 应用"一节。
</GlossaryItem>

<GlossaryItem term="Android Studio">
  Google 官方提供的 Android 集成开发环境，基于 JetBrains IntelliJ IDEA。集成了代码编辑、模拟器管理、Gradle 构建、Logcat 日志查看、Profiler 性能分析等所有开发必需的模块。免费下载使用。详见本章"Android Studio：你的工作台"一节。
</GlossaryItem>

<GlossaryItem term="AndroidManifest.xml">
  每个 Android 应用根目录下必须有的 XML 清单文件。它向 Android 系统声明：应用里有哪些组件（Activity、Service 等）、需要哪些权限（联网、相机等）、支持哪些设备特性、最低和目标 SDK 版本。系统在安装 APK 前会先解析这个文件。详见本章"项目结构"一节。
</GlossaryItem>

<GlossaryItem term="Activity">
  Android 四大应用组件之一，代表一个用户可以交互的屏幕。每一个界面（登录页、设置页、聊天列表等）都是一个 Activity。它管理界面的生命周期（创建→可见→暂停→销毁），处理用户输入。详见本章"MainActivity 与布局文件"一节。
</GlossaryItem>

<GlossaryItem term="Gradle">
  Android 官方使用的自动化构建工具。你点 Run 之后，Gradle 依次执行：编译 Kotlin → 转 DEX 字节码 → 打包资源和代码 → 签名 → 生成 APK。构建规则写在 build.gradle（Kotlin DSL）文件里。详见本章"Gradle：流水线的工位"一节。
</GlossaryItem>

</Glossary>
