## 你的 App 看起来像打了补丁

同一个 App，按钮有的蓝色有的灰色，文字大小不统一——为什么？因为每个 View 的属性都是手写硬编码的，没有统一管理。Style 和 Theme 就是解决这个问题的——定义一套"视觉规则"，所有 View 照着来。

## Style——View 的"穿搭模板"

```xml
<!-- res/values/styles.xml -->
<style name="PrimaryButton" parent="Widget.Material3.Button">
    <item name="android:textColor">@color/white</item>
    <item name="backgroundTint">@color/primary</item>
    <item name="cornerRadius">8dp</item>
    <item name="android:textSize">16sp</item>
</style>
```

然后一个按钮只需引用这个 style：

```xml
<Button style="@style/PrimaryButton" android:text="登录" />
```

改了 style 里的 `cornerRadius` 从 8dp 到 12dp → 所有引用这个 style 的按钮自动变成圆角 12dp。**统一改一处，全 App 生效**。

### Style 继承

```xml
<!-- 继承父 style（点记法） -->
<style name="PrimaryButton.Outlined">
    <item name="backgroundTint">@android:color/transparent</item>
    <item name="strokeColor">@color/primary</item>
</style>
```

`PrimaryButton.Outlined` 继承 `PrimaryButton` 的所有属性，只覆盖要改的——描边按钮只有背景变透明+加描边，文字大小和圆角沿用父 style。

## Theme——整个 App 的"皮肤"

Theme 也是 Style，但作用于 Application 或 Activity 级别：

```xml
<!-- res/values/themes.xml -->
<style name="Theme.MyApp" parent="Theme.Material3.Light.NoActionBar">
    <item name="colorPrimary">@color/blue_500</item>
    <item name="colorOnPrimary">@color/white</item>
    <item name="android:statusBarColor">@color/blue_700</item>
</style>
```

在 Manifest 中应用：

```xml
<application android:theme="@style/Theme.MyApp">
```

## 深色模式

```xml
<!-- res/values-night/themes.xml -->
<style name="Theme.MyApp" parent="Theme.Material3.Dark.NoActionBar">
    <item name="colorPrimary">@color/blue_200</item>
    <!-- 深色背景、浅色文字由父主题自动处理 -->
</style>
```

`values-night/` 目录——系统切换到深色模式时自动使用这里的资源。Material Design 3 的 Dark 主题帮你处理了大部分颜色映射，你只需要微调品牌色即可。

<Stepper>
  <Step title="① 定义 Theme">
    在 `res/values/themes.xml` 中定义 `<style name="Theme.MyApp" parent="Theme.Material3.Light.NoActionBar">`，设置 `colorPrimary`、`colorOnPrimary`、`statusBarColor` 等属性——这些颜色会渗透到所有 Material 组件中。
  </Step>
  <Step title="② Manifest 应用主题">
    在 `AndroidManifest.xml` 的 `<application>` 标签添加 `android:theme="@style/Theme.MyApp"`，全局生效。也可以单独给某个 `<activity>` 指定不同主题——但一般全局统一。
  </Step>
  <Step title="③ 提取常用 View 的 Style">
    把多个按钮/文本重复的 `android:textSize`、`backgroundTint`、`cornerRadius` 等属性提取到 `res/values/styles.xml` 中，View 上只保留 `style="@style/PrimaryButton"` 引用。用点记法（`PrimaryButton.Outlined`）实现样式继承。
  </Step>
  <Step title="④ 创建 values-night 深色主题">
    新建 `res/values-night/themes.xml`，定义同名 `Theme.MyApp` 但 parent 改为 `Theme.Material3.Dark.NoActionBar`。微调品牌色（如浅蓝 `@color/blue_200` 替代深蓝 `@color/blue_500`），让文字在暗色背景上可读。
  </Step>
</Stepper>

## 容易踩的坑

<Callout type="trap">
  **现象**：改了 style 里的某个属性，但 View 没变化。**原因**：View 上直接写了同名的 XML 属性——内联属性优先级高于 style。**修法**：把 view 上的硬编码属性移到 style 里管理，view 只保留 `style="@style/xxx"` 引用。
</Callout>

<Callout type="trap">
  **现象**：深色模式下文字看不清——白底白字。**原因**：只在 values/colors.xml 定义颜色，没有在 values-night/ 提供深色版本。**修法**：为每个需要适配的颜色提供 night 版本，或使用 `?attr/colorOnSurface` 等系统属性自动切换。
</Callout>

<Callout type="trap">
  **现象**：Material 组件颜色跟 Theme 不一致。**原因**：用了硬编码 `@color/xxx` 而非 `?attr/colorPrimary`。**修法**：主题相关色一律走 theme attribute，便于 night 与动态取色。
</Callout>

## 小结

- Style 复用 View 属性 → 消除 XML 重复、统一视觉
- Theme 是作用在 Application/Activity 级别的 Style → 整体皮肤
- Style 继承（点记法）→ 子 Style 覆盖少量属性，其余继承父
- 深色模式：values-night/ 目录提供暗色版本的资源和主题
- 内联属性优先级最高 → 统一管理时应去掉 View 上的硬编码

<Glossary>

<GlossaryItem term="Style">
  一组 View 属性的集合——定义在 res/values/styles.xml 中，通过 `style="@style/xxx"` 引用。复用 View 属性消除 XML 重复代码。
</GlossaryItem>

<GlossaryItem term="Theme">
  作用在 Application/Activity 级别的特殊 Style。定义全局视觉属性——主色、强调色、背景色、状态栏颜色等。决定整个 App 的视觉基调。
</GlossaryItem>

</Glossary>
