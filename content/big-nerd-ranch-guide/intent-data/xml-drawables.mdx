## 你的 App 塞满了 PNG 图片

按钮背景是一张 9-patch，圆角卡片是一张 PNG，选中的高亮又是一张图——你的 APK 越来越胖。XML drawable 在运行时由系统渲染，**一个 2KB 的 XML 文件能替代所有屏幕密度下的 PNG（xxhdpi/xxxhdpi 各一份，加起来几十 KB）**。

## Shape Drawable

```xml
<!-- res/drawable/rounded_background.xml -->
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <solid android:color="#FFFFFF" />
    <corners android:radius="12dp" />
    <stroke android:width="1dp" android:color="#E0E0E0" />
</shape>
```

支持的 shape：`rectangle`、`oval`、`line`、`ring`。`gradient` 子元素可实现线性/径向/扫描渐变：

```xml
<gradient
    android:startColor="#FF6B6B"
    android:endColor="#FF8E53"
    android:angle="45" />
```

## Selector（State-List）

按钮按下变暗、禁用变灰——这些状态切换用 selector 一行代码搞定：

```xml
<!-- res/drawable/button_bg.xml -->
<selector xmlns:android="http://schemas.android.com/apk/res/android">
    <!-- 按下状态 -->
    <item android:state_pressed="true">
        <shape>
            <solid android:color="#1565C0" />
            <corners android:radius="8dp" />
        </shape>
    </item>
    <!-- 禁用状态 -->
    <item android:state_enabled="false">
        <shape>
            <solid android:color="#BDBDBD" />
            <corners android:radius="8dp" />
        </shape>
    </item>
    <!-- 默认 -->
    <item>
        <shape>
            <solid android:color="#2196F3" />
            <corners android:radius="8dp" />
        </shape>
    </item>
</selector>
```

Selector 按从上到下顺序匹配——第一个匹配的 `<item>` 生效。支持的状态：`state_pressed`、`state_focused`、`state_enabled`、`state_selected`、`state_checked` 等。

## Layer-List

把多个 drawable 叠起来：

```xml
<layer-list xmlns:android="...">
    <item>
        <shape><solid android:color="#FFF"/></shape>
    </item>
    <item android:top="2dp" android:bottom="2dp">
        <shape><solid android:color="#E0E0E0"/></shape>
    </item>
</layer-list>
```

## Vector Drawable

用 SVG 路径描述图标——任意缩放不变形、一个文件通吃所有屏幕：

```xml
<vector android:width="24dp" android:height="24dp"
    android:viewportWidth="24" android:viewportHeight="24">
    <path
        android:fillColor="#000"
        android:pathData="M12,2C6.48,2 2,6.48 2,12s4.48,10 10,10..." />
</vector>
```

Android Studio 内置 Vector Asset 工具——直接从 Material Icons 导入矢量图标。

<Stepper>
  <Step title="① 创建 shape drawable">
    在 `res/drawable/` 下新建 XML 文件，根元素 `<shape>` 设置 `android:shape="rectangle"`（或 `oval`/`line`/`ring`）。系统在运行时将 XML 解析为位图，自动适配当前屏幕密度。
  </Step>
  <Step title="② 加 gradient/corners/stroke">
    `<gradient>` 定义渐变（线性/径向/扫描），设 `startColor`、`endColor`、`angle`；`<corners>` 设 `android:radius` 控制圆角；`<stroke>` 设 `android:width` 和 `android:color` 添加描边。
  </Step>
  <Step title="③ 创建 selector 切换状态">
    `<selector>` 根元素，内含多个 `<item>`。按状态优先级从高到低排列：`state_pressed="true"` → `state_enabled="false"` → 默认（无条件）。系统按从上到下顺序匹配，第一个命中的 item 生效——**默认 item 必须放最后**。
  </Step>
  <Step title="④ 替换布局中的 @drawable 引用">
    把布局中 `android:background="@color/xxx"` 或 PNG 图片引用替换为 `android:background="@drawable/button_bg"`。XML drawable 一个文件通吃所有密度，比多套 PNG（xxhdpi/xxxhdpi）节省数十 KB APK 体积。
  </Step>
</Stepper>

## 容易踩的坑

<Callout type="trap">
  **现象**：shape 设了 corners radius，但 View 上看到的是直角。**原因**：View 本身有背景色覆盖了 drawable。**修法**：确保 `android:background="@drawable/rounded_bg"` 后的代码没有重新设 background 或 backgroundTint。
</Callout>

<Callout type="trap">
  **现象**：selector 里的状态切换不起作用——按下跟默认一样。**原因**：selector 中 `<item>` 的顺序错误——第一个匹配的 item 生效。如果默认 item 放在最前面，后面的状态永远不会被匹配。**修法**：默认 item 放最后。
</Callout>

<Callout type="trap">
  **现象**：vector 在 API 21 以下崩溃。**原因**：旧系统需 AppCompat 矢量兼容或 VectorDrawableCompat。**修法**：Gradle 开启 vectorDrawables.useSupportLibrary 或提供 png fallback。
</Callout>

## 小结

- XML drawable 替代 PNG → 省空间、自适应密度、运行时渲染
- shape 画基本图形；gradient 画渐变；corners 设圆角；stroke 加描边
- selector 根据 View 状态（按下/禁用/选中）自动切换 drawable
- layer-list 叠加多层 drawable；vector 用路径描述图标 → 无限缩放
- selector 顺序：默认 item 放最后，状态 item 放前面

<Glossary>

<GlossaryItem term="Shape Drawable">
  用 XML 定义的几何图形（矩形/椭圆/线/环）。支持纯色(solid)、圆角(corners)、描边(stroke)、渐变(gradient)等属性。替代 PNG 背景图。
</GlossaryItem>

<GlossaryItem term="Selector（State-List）">
  根据 View 状态（按下/禁用/选中/聚焦）自动切换 drawable 的 XML 定义。创建按钮、列表项等交互反馈的标准方式。
</GlossaryItem>

<GlossaryItem term="Vector Drawable">
  用 XML 路径（pathData）描述的矢量图。与分辨率无关——任意缩放不失真，一个文件适用所有屏幕密度。
</GlossaryItem>

</Glossary>
