<Objectives>

- 能用 Toolbar 替代默认 ActionBar，并在布局 XML 中自定义 AppBar
- 能创建 Options Menu 资源文件，在 onCreateOptionsMenu 中加载菜单
- 能处理菜单项点击——onOptionsItemSelected 返回 true 表示已消费事件

## 顶栏不只是标题——它是命令中枢

<Term def="Material Design 顶栏，可显示标题、导航图标和操作菜单，现代 App 多用 Toolbar 实现。">App Bar</Term>（Toolbar）承载标题、返回箭头和「更多」菜单。Fragment 时代默认 <Term def="旧版框架提供的顶栏，现推荐用 Toolbar + setSupportActionBar 替代。">ActionBar</Term> 不够灵活——把 Toolbar 放进布局，才能和 CoordinatorLayout、Drawer 配合。

## 启用 Toolbar

`res/layout/activity_main.xml`：

```xml
<com.google.android.material.appbar.MaterialToolbar
    android:id="@+id/toolbar"
    android:layout_width="match_parent"
    android:layout_height="?attr/actionBarSize"
    app:title="@string/app_name" />
```

```kotlin
setSupportActionBar(findViewById(R.id.toolbar))
supportActionBar?.setDisplayHomeAsUpEnabled(true)
```

主题使用 `NoActionBar` 变体，避免双顶栏。

## Options Menu

`res/menu/crime_list_menu.xml`：

```xml
<menu xmlns:android="http://schemas.android.com/apk/res/android">
    <item
        android:id="@+id/menu_add"
        android:icon="@drawable/ic_add"
        android:title="@string/add_crime"
        app:showAsAction="ifRoom" />
    <item
        android:id="@+id/menu_settings"
        android:title="@string/settings"
        app:showAsAction="never" />
</menu>
```

```kotlin
override fun onCreateOptionsMenu(menu: Menu): Boolean {
    menuInflater.inflate(R.menu.crime_list_menu, menu)
    return true
}

override fun onOptionsItemSelected(item: MenuItem): Boolean {
    return when (item.itemId) {
        R.id.menu_add -> { viewModel.addCrime(); true }
        R.id.menu_settings -> { startActivity(Intent(this, SettingsActivity::class.java)); true }
        android.R.id.home -> { onBackPressedDispatcher.onBackPressed(); true }
        else -> super.onOptionsItemSelected(item)
    }
}
```

`showAsAction="ifRoom"` 有空间时显示图标；`never` 进 Overflow（⋮）。

<Stepper>
  <Step title="① 列表页配置 Toolbar">
    在列表 Activity 的布局中放置 `MaterialToolbar`，主题使用 `NoActionBar` 变体避免双顶栏。通过 `setSupportActionBar(toolbar)` 将 Toolbar 设为 Activity 的 App Bar——列表页的 Toolbar 通常只显示标题和菜单项（如"添加"），不显示返回箭头（home 图标）。
  </Step>
  <Step title="② 详情页启用 Up 导航">
    在详情 Activity 中调用 `supportActionBar?.setDisplayHomeAsUpEnabled(true)`——Toolbar 左边出现一个向左的箭头（←），告诉用户"点这里可以返回上一级"。这是"向上导航"（Up）而非系统"返回"（Back）——Up 保证用户回到应用内的逻辑父页面。
  </Step>
  <Step title="③ 点击 ← 触发 onOptionsItemSelected">
    用户点击返回箭头时，系统回调 `onOptionsItemSelected(item)`，`item.itemId` 为 `android.R.id.home`。在回调中判断：如果用 Jetpack Navigation，调 `navController.navigateUp()` 或 `NavigationUI.navigateUp(navController, configuration)`；如果手动管理，调 `onBackPressedDispatcher.onBackPressed()` 或 `finish()`。必须 `return true` 表示事件已被消费。
  </Step>
  <Step title="④ 返回栈弹出，回到列表页">
    详情 Activity 从返回栈弹出（或 Fragment 详情页被 NavController 回退），列表页重新进入 `onResume` 状态。如果列表页之前被系统回收，此时自动重建并恢复滚动位置——用户感觉一切如初。
  </Step>
</Stepper>

<Callout type="tip">
  **Jetpack Navigation** 与 Toolbar 绑定：`setupWithNavController(toolbar, navController)` 自动处理标题与 Up，少写重复 home 处理。
</Callout>

<Callout type="trap">
  忘记 `return true` → 系统继续处理，可能触发默认行为或重复执行。已处理的 item 必须返回 true。
</Callout>

<Callout type="trap">
  Fragment 里要菜单：在 `onCreateView` 里 `setHasOptionsMenu(true)`（旧 API）或 MenuProvider（AndroidX Activity 1.4+），否则 `onCreateOptionsMenu` 不会调到 Fragment。
</Callout>

## 小结

- NoActionBar 主题 + 布局 Toolbar + setSupportActionBar
- menu XML + inflate + onOptionsItemSelected
- Home/Up 与返回栈一致；Navigation 组件可自动绑定

<Exercises>

**练习 1（布局）** 为何主题还要设 NoActionBar？

<Answer>
系统默认再画一条 ActionBar，与布局里自定义 Toolbar 重叠，出现双顶栏或空白条。
</Answer>

**练习 2（行为）** showAsAction 设为 never 的项出现在哪？用户如何点到？

<Answer>
Overflow 菜单（通常右上角 ⋮）。点击后弹出列表，选该项触发 onOptionsItemSelected。
</Answer>

</Exercises>

<Glossary>

<GlossaryItem term="Toolbar">
  可嵌入布局的顶栏控件，功能同 ActionBar 但可定制位置与样式。
</GlossaryItem>

<GlossaryItem term="Options Menu">
  Activity/Fragment 顶栏右侧的操作项集合，由 menu 资源定义。
</GlossaryItem>

<GlossaryItem term="Up navigation">
  应用内层级返回（←），不同于系统 Back——应弹出当前 Task 内上一级 Destination。
</GlossaryItem>
