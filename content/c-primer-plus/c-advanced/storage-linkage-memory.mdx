## 为什么变量「活多久」和「谁能看见」要分开想？

写程序时，你声明一个名字，编译器要回答两件事：这块内存**放在哪**（栈？堆？静态区？），以及这个名字**谁能用**（只在当前 `{}` 里？整个文件？整个程序？）。

想象公司工位：临时工（auto）下班就清桌子；档案室（static）的文件整年都在，换班的人还能翻到上次记录；跨部门共享的台账（extern）全公司只认一份。若你借了仓库空间（malloc）却从不归还，仓库迟早堆满——那就是 **内存泄漏**。

这一章把 C 的存储类、链接规则，以及栈/堆/静态区的内存模型讲清楚。搞懂这些，多文件项目、全局状态、动态数组才会可控。

## 四种存储类：作用域与生命周期

C 用 <Term def="关键字 auto、static、extern、register，决定变量的存储位置、默认初值、作用域和链接属性。">存储类（storage class）</Term> 描述变量在内存中的「户口」。函数内不写修饰符的局部变量默认是 **auto**。

<StorageClassDiagram />

### auto：块内自动变量

```c
void demo(void) {
    int count;           /* 等价于 auto int count; */
    count = 0;
}
/* count 已销毁，内存可被复用 */
```

<Term def="进入块时分配（通常在栈上），离开块时自动释放；未显式初始化时内容不确定。">auto 变量</Term> 只在声明它的 `{}` 内可见。每次进入函数，栈上重新划一块；函数返回，这块就「还回去」。

### static：活过整个程序

```c
void counter(void) {
    static int calls = 0;
    calls++;
    printf("第 %d 次调用\n", calls);
}
```

<Term def="存储在静态区，程序启动时分配、退出时释放；块内 static 仅初始化一次，下次进入块仍保留上次值。">static 变量</Term> 若在函数内声明，作用域仍是该函数，但**生命周期是整个程序**。`calls` 不会在每次调用时归零——这就是 static 的「记忆」。

文件作用域的 static（见下文链接）还限制名字只在本 `.c` 可见。

### extern：声明「别处有定义」

```c
/* file_a.c */
int shared = 100;      /* 定义 + 外部链接 */

/* file_b.c */
extern int shared;     /* 声明：链接到 file_a 的那份 */
```

<Term def="告诉编译器该标识符的定义在别的翻译单元；本身不分配存储（除非同时是定义）。">extern</Term> 常用于头文件里声明全局变量/函数，在某一 `.c` 里写一次定义。若只有 extern 声明、全程序找不到定义，链接阶段报 `undefined reference`。

### register：建议放寄存器

<Term def="提示编译器把变量放在 CPU 寄存器以加速访问；C11 起 register 变量取地址 &x 受限，现代编译器常自动优化，关键字已少见。">register</Term> 对局部变量使用，语义接近 auto，只是多一个「尽量别放栈上」的提示。初学阶段知道即可，不必刻意使用。

## 链接：谁能跨文件看见这个名字

<Term def="标识符能否在多个源文件间共享同一份实体。内部链接仅本翻译单元；外部链接全程序唯一。">链接（linkage）</Term> 回答：两个 `.c` 文件能不能指到**同一个**全局变量或函数。

<LinkageDiagram />

规则速记：

- 函数内变量：**无链接**（只在块内）
- 文件内 `static` 全局/函数：**内部链接**——其他 `.c` 看不见，各文件可各有一份同名 static 而不冲突
- 普通全局定义（无 static）：**外部链接**——全程序一份，用 extern 声明引用

头文件惯用法：`extern int foo;` 放 `.h`；`int foo = 0;` 只在一个 `.c` 里定义，避免重复定义链接错误。

## 进程内存布局：代码、静态、堆、栈

程序跑起来后，操作系统给它一块虚拟地址空间。C 程序员最常打交道的是下面几个区域：

<MemoryRegionsDiagram />

| 区域 | 典型内容 | 谁分配/释放 |
|------|----------|-------------|
| 代码段 | 机器指令 | 加载时固定 |
| .data | 有初值的全局/static | 程序整个生命期 |
| .bss | 未初始化全局/static（零初始化） | 程序整个生命期 |
| 堆 | `malloc` 得到的块 | 程序员 `free` |
| 栈 | 局部 auto、参数、返回地址 | 进入/离开函数自动 |

<Term def={'运行时向堆申请指定字节数，成功返回 void* 指针，失败返回 NULL；需包含 stdlib.h 头文件。'}>malloc</Term> 从堆划内存，大小在运行时决定；对应释放是 <Term def="把 malloc/calloc/realloc 得到的指针归还堆；释放后不得再解引用该指针。">free</Term>。

## malloc / free 生命周期

动态数组、运行时才知道长度的字符串，往往离不开堆。标准用法是「申请 → 使用 → 释放」三步，漏掉最后一步就泄漏。

<MallocFreeDiagram />

<Stepper>
  <Step title="第一步：malloc 申请">
    <MallocFreeDiagram step={1} />
    `int *p = malloc(n * sizeof *p);` 向堆要 `n` 个 int 的空间。务必检查 `p == NULL`。
  </Step>
  <Step title="第二步：通过指针使用">
    <MallocFreeDiagram step={2} />
    读写 `p[i]` 或 `*p`。指针类型应匹配：`malloc` 返回 `void*`，赋给 `int*` 前在 C 里会自动转换。
  </Step>
  <Step title="第三步：free 释放">
    <MallocFreeDiagram step={3} />
    `free(p);` 把块还给堆。之后不要把 `p` 当有效指针用（可置 `p = NULL` 防误用）。
  </Step>
  <Step title="第四步：忘记 free → 泄漏">
    <MallocFreeDiagram step={4} />
    丢失唯一指针却不 free，堆块永远无法回收。短程序可能无感，服务器/嵌入式长跑会耗尽内存。
  </Step>
</Stepper>

最小完整示例：

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int n = 5, i;
    int *arr = malloc(n * sizeof *arr);
    if (arr == NULL) {
        fprintf(stderr, "malloc failed\n");
        return 1;
    }
    for (i = 0; i < n; i++)
        arr[i] = i * 10;
    for (i = 0; i < n; i++)
        printf("%d ", arr[i]);
    printf("\n");
    free(arr);
    arr = NULL;
    return 0;
}
```

`sizeof *arr` 比 `sizeof(int)` 更安全：若以后改指针类型，字节数自动跟着变。

## 代码详解：static 计数与跨文件共享

### 场景一：static 局部变量做计数器

```c
#include <stdio.h>

void visit(void) {
    static int count = 0;
    count++;
    printf("visit #%d\n", count);
}

int main(void) {
    visit();  /* visit #1 */
    visit();  /* visit #2 */
    visit();  /* visit #3 */
    return 0;
}
```

`count` 存在静态区，只初始化一次，三次调用递增。

### 场景二：文件私有 helper（内部链接）

```c
/* utils.c */
static int helper(int x) { return x * 2; }

int public_api(int x) { return helper(x) + 1; }
```

`helper` 带文件作用域 static，其他 `.c` 即使写 `extern int helper(int);` 也链接不到——封装实现细节。

### 场景三：动态读入未知长度

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int n, i, *data;
    printf("多少个整数？");
    if (scanf("%d", &n) != 1 || n <= 0)
        return 1;
    data = malloc(n * sizeof *data);
    if (data == NULL)
        return 1;
    for (i = 0; i < n; i++)
        scanf("%d", data + i);
    /* ... 使用 data ... */
    free(data);
    return 0;
}
```

编译期不知道 `n`，栈上 `int data[n]`（VLA，C99）可替代，但堆分配更通用，且大小几乎不受栈限制。

## 容易踩的坑

<Callout type="trap">
  **现象**：函数返回局部数组指针，调用方一用就崩溃或乱码。
  **原因**：`int ar[100]; return ar;` 返回的是**栈上**数组地址，函数结束栈帧销毁，指针悬空。
  **修法**：返回动态分配内存（调用方负责 free），或由调用方传入缓冲区；不要返回指向 auto 局部变量的指针。
</Callout>

<Callout type="trap">
  **现象**：`free(p);` 之后仍 `printf("%d", *p);`，有时还能打印「旧值」，有时直接崩。
  **原因**：**use-after-free** 是未定义行为。堆管理器可能把块交给下次 malloc，旧值不可靠。
  **修法**：`free(p); p = NULL;` 释放后置空；需要时再 malloc 新块。
</Callout>

<Callout type="trap">
  **现象**：两个 `.c` 都写 `int shared = 1;`，链接报 `multiple definition`。
  **原因**：全局定义默认**外部链接**，全程序只能有一份定义。
  **修法**：一个 `.c` 定义，其余文件和头文件用 `extern int shared;` 声明。
</Callout>

<Callout type="trap">
  **现象**：循环里反复 `malloc(1000)` 却不 free，程序越跑越慢、最终 OOM。
  **原因**：每次泄漏 1000 字节，堆可用块耗尽——典型**内存泄漏**。
  **修法**：成对 malloc/free；长生命周期数据在不用时释放；工具如 Valgrind 查泄漏。
</Callout>

## 小结

- 存储类决定作用域、存储位置、生命周期：auto 块内栈；static 静态区且可跨调用保留；extern 声明外部定义；register 少用
- 链接：文件 static → 内部链接；普通全局 → 外部链接，全程序一份
- 内存模型：代码 + .data + .bss + 堆（malloc）+ 栈（auto）；堆栈相向增长
- malloc/free 必须成对；检查 NULL；free 后勿解引用；勿返回指向栈局部的指针

<Exercises>

**练习 1（问答型）** 块内 `static int x = 0;` 与块内 `int x = 0;` 每次进入块时，`x` 的值分别会怎样？

<Answer>

**static**：只初始化一次，之后每次进入块**保留上次的值**。**auto**：每次进入块重新分配，`x = 0` 每次执行（若写初始化），未初始化则内容不确定。

</Answer>

**练习 2（改代码型）** 下面程序泄漏了内存，指出问题并改正。

```c
#include <stdlib.h>
void process(int n) {
    int *buf = malloc(n * sizeof(int));
    if (buf == NULL) return;
    /* 使用 buf ... */
    return;   /* 忘记释放 */
}
```

<Answer>

在 `return` 前加 **`free(buf);`**。若有多处 return，每条路径都要 free，或用单一出口：

```c
void process(int n) {
    int *buf = malloc(n * sizeof(int));
    if (buf == NULL) return;
    /* 使用 buf ... */
    free(buf);
}
```

</Answer>

**练习 3（综合实现型）** 写程序：用 `malloc` 创建长度为 10 的 int 数组，填入 0..9，打印后释放。必须检查 malloc 是否失败。

<Answer>

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int i, *arr = malloc(10 * sizeof *arr);
    if (arr == NULL) {
        fprintf(stderr, "out of memory\n");
        return 1;
    }
    for (i = 0; i < 10; i++)
        arr[i] = i;
    for (i = 0; i < 10; i++)
        printf("%d ", arr[i]);
    printf("\n");
    free(arr);
    return 0;
}
```

</Answer>

## 名词解释

<Glossary>

<GlossaryItem term="存储类（storage class）">
  auto、static、extern、register 等关键字，描述变量存储位置、默认初值、作用域与链接。详见 StorageClassDiagram。
</GlossaryItem>

<GlossaryItem term="auto 变量">
  块内局部变量默认类别；栈上分配，块结束销毁；未初始化内容不确定。
</GlossaryItem>

<GlossaryItem term="static 变量">
  静态存储期：程序启动到退出；块内 static 仅初始化一次；文件 static 内部链接。
</GlossaryItem>

<GlossaryItem term="extern">
  声明标识符在其他翻译单元有定义；不重复分配存储（纯声明时）。
</GlossaryItem>

<GlossaryItem term="register">
  提示编译器把变量放在 CPU 寄存器以加速访问；C11 起 register 变量取地址受限，现代编译器常自动优化，关键字已少见。
</GlossaryItem>

<GlossaryItem term="链接（linkage）">
  标识符能否跨 .c 文件共享。internal 仅本文件；external 全程序唯一。详见 LinkageDiagram。
</GlossaryItem>

<GlossaryItem term="内部链接">
  static 文件作用域变量/函数；符号不导出给其他 .o，各文件可各有同名 static。
</GlossaryItem>

<GlossaryItem term="外部链接">
  普通全局定义；全程序一份，其他文件用 extern 引用。
</GlossaryItem>

<GlossaryItem term="栈（stack）">
  存放 auto 局部变量、参数、返回地址；函数调用时向下增长，返回时自动回收。
</GlossaryItem>

<GlossaryItem term="堆（heap）">
  malloc/calloc/realloc 分配的内存区；手动 free，生命周期可跨越多个函数。
</GlossaryItem>

<GlossaryItem term="BSS 段">
  未显式初始化的全局/static 变量；启动时零填充。详见 MemoryRegionsDiagram。
</GlossaryItem>

<GlossaryItem term="malloc">
  堆上分配字节；返回 void* 或 NULL。见 MallocFreeDiagram。
</GlossaryItem>

<GlossaryItem term="free">
  释放 malloc 族得到的指针；释放后指针变悬空，不得解引用。
</GlossaryItem>

<GlossaryItem term="内存泄漏">
  堆块无法再被程序访问却未 free，可用堆内存持续减少。见 MallocFreeDiagram step 4。
</GlossaryItem>

</Glossary>
