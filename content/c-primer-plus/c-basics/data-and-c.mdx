## 为什么 C 语言里要分这么多种"数字盒子"？

想象你在一个零件仓库里：螺丝钉要放在小格子里，螺母要放在中号抽屉，发动机要放在大货架上。如果所有零件都往同一个万能麻袋里塞——小螺丝淹没在角落里找不到、大发动机根本塞不进。

在 C 语言里，**数据也有自己的"规格"**。一个字符只要一个小格子（1 字节），一个普通整数要四个格子（4 字节），一个带小数的"高精度数"要八个格子（8 字节）。C 语言不替你做主——它把不同的盒子摆在你面前，由你来告诉它："这个数字装 4 字节的盒子就够"还是"装 8 字节的盒子才精确"。

**如果所有数据都往同一种通用盒子里塞呢？** 电脑就帮不了你检查错误——省下的空间白浪费、该存的小数精确度悄悄丢失、输出的时候格式还跟你想象的不一样。这一章就来认识 C 里这些「盒子的种类」，学会怎么挑、怎么装、怎么取出来看得清清楚楚。



## C 的数据类型家族

在 C 语言中，<Term def="编程语言里给数据规定的类别，决定了一个数据占多大内存、能存什么范围的值、能对它做哪些运算。C 要求每个变量在使用前必须声明类型。">类型（type）</Term> 是程序最基本的规矩。任何一个变量在声明的那一刻就定死了类型——编译器根据类型来分配内存空间、校验运算是否合法、在 printf 输出时决定该怎么解释那段二进制数据。

类型在 C 里扮演三个角色：**① 占用多少内存**（sizeof 告诉你）、**② 能存什么范围的值**（int 存整数，float 存小数）、**③ 怎么解释内存里的二进制位**（同样一段 4 字节数据，当 int 解读和当 float 解读，值完全不同）。

### 基本数据类型一览——C 里有哪几种盒子

C 语言提供了以下 <Term def="语言自带的基础类型，不需要 include 任何头文件就能直接用，包括整型、浮点型、字符型。">基本数据类型（basic data type）</Term>：

- **int 家族**：`short`、`int`、`long`、`long long`——存整数，区别在于字节数和取值范围
- **char 家族**：`char`——存字符或小整数，占 1 字节
- **float/double 家族**：`float`、`double`、`long double`——存带小数的数，区别在于精度和范围

下面这张对照表帮你直观感受各类型在内存里的"个头"和"胃口"：

<Callout type="tip">
  `int` 是整数运算的不二之选——32 位机器上一个 int 刚好 4 字节，能装下 −21 亿到 +21 亿。
  `double` 是浮点数运算的默认选择——精度比 float 高得多（15 位 vs 6 位有效数字），现代 CPU 对 double 运算已经不比 float 慢。
  选型口诀：整数用 int、小数用 double、字符用 char。
</Callout>

类型决定了**三件事**，在写程序时时刻都要记住：
1. **内存大小**：`char` 只占 1 字节，`int` 占 4 字节，`double` 占 8 字节——不同的盒子，不同的容量
2. **取值范围**：`unsigned char` 只能存 0~255，`signed char` 存 −128~127——同样的 1 字节，能装的数不一样大
3. **运算行为**：整数除法和浮点除法规则不同——`3 / 2` 得 1，`3.0 / 2.0` 才得 1.5

### 整型的修饰符——控制大小和符号

C 语言提供了四个 <Term def="加在基本类型名前面的关键字，用来改变类型的大小范围或符号属性。例如 short int（缩小）、long int（拉长）、unsigned int（把负数区间挪到正数）。">修饰符（modifier）</Term> 用来调整整型的大小和符号：

| 修饰符 | 作用 | 示例 |
|---|---|---|
| `short` | 缩小——占用的字节变少，范围也变小 | `short int s;` 或 `short s;`（int 可省略） |
| `long` | 拉长——占用的字节增加，范围变大 | `long int l;` 或 `long l;` |
| `long long` | 超级拉长——至少 8 字节 | `long long ll;` |
| `unsigned` | 去掉负数——原来给负数的范围全部加给正数 | `unsigned int u;` |

修饰符可以组合：`unsigned short`、`unsigned long long` 等都是合法的——你可以同时控制大小和符号。

下面这张图直观对比了不同位宽下 signed 和 unsigned 的取值范围：

<Callout type="warn">
  **混用 signed 和 unsigned 是 C 语言里最常见的 bug 来源之一。** 当 `unsigned int u = 0; u = u - 1;` 时，u 不会变成 −1——它会「回绕（wrap-around）」到 4294967295（32-bit unsigned 的最大值）。同理，`unsigned` 和 `signed` 在同一个表达式里出现时，C 会把 signed 隐式转成 unsigned 再算，结果常常不是你想要的。
</Callout>

在实际编程中，**默认用 `int`** 就能满足绝大多数整数需求。`unsigned` 主要用在「这个量绝不可能是负数」的场合——数组下标、`sizeof` 的返回值、位运算、计数等等。

### 浮点类型——存小数靠它们

整数搞不定的任务交给 <Term def="能存带小数点的数字的类型，包括 float（单精度）、double（双精度）、long double（扩展精度）。浮点数用科学计数法在内存中近似存储——有舍入误差。">浮点类型（floating-point type）</Term>。C 提供了三种精度档次：

- **`float`**（4 字节）：约 6 位有效数字，够用但精度有限
- **`double`**（8 字节）：约 15 位有效数字——最常用的浮点类型
- **`long double`**（8/10/16 字节，依平台）：更高精度，科学计算用

**浮点数有一个根本特性必须记住：它们是近似存储的。** 就像十进制里 `1/3 = 0.33333...` 写不完，二进制里很多十进制小数也写不完——`0.1` 在二进制里就是一个无限循环小数。所以浮点运算会有舍入误差。下面这段代码展示了 float 和 double 在精度上的实际差别：

```c
#include <stdio.h>
int main(void) {
    float  f = 1.0f / 3.0f;        // float 除法
    double d = 1.0  / 3.0;         // double 除法

    printf("float:  %.20f\n", f);   // 显示 20 位小数——6 位后开始"胡编"
    printf("double: %.20f\n", d);   // 显示 20 位小数——15 位后才开始"胡编"

    // float:  0.33333334326744079590
    // double: 0.33333333333333331483

    return 0;
}
```

<Callout type="tip">
  C 语言中直接写的浮点数字面值，默认是 `double` 类型。要显式写一个 `float` 字面值，在数字后面加 `f` 或 `F`：
  `3.14` 是 double，`3.14f` 才是 float。`1.0 / 3.0` 是 double 除法，`1.0f / 3.0f` 才是 float 除法。
</Callout>

### 字符类型与转义序列

<Term def="存单个字符的类型，占 1 字节。char 在 C 中本质上是小整数（ASCII 码），可以与 int 互相转换。">char</Term> 存单个字符——在键盘上能敲出来的每个符号，在计算机内部都对应一个整数编码，这套编码叫 <Term def="American Standard Code for Information Interchange，美国信息交换标准代码。规定每个常用字符对应 0~127 中的一个整数。例如 'A'=65，'a'=97，'0'=48。">ASCII 码</Term>。

所以 `char` 在 C 里本质就是一个"穿字符外衣的整数"。`'A'` 存进 char 变量，实际存的是整数 65（ASCII 码）。你可以用 `%c` 按字符输出，也可以用 `%d` 按整数输出：

```c
#include <stdio.h>
int main(void) {
    char ch = 'A';             // 等价于 char ch = 65;

    printf("字符: %c\n", ch);   // 输出 A
    printf("整数: %d\n", ch);   // 输出 65
    return 0;
}
```

有些字符没法直接敲出来——比如换行、制表、双引号本身。这时就需要用到 <Term def="以反斜杠 \\ 开头、后跟一个或几个字符的组合，表示一个特殊含义。如 \\n 表示换行、\\t 表示制表。反斜杠把普通字母「转」成了控制功能，故称转义。">转义序列（escape sequence）</Term>：

<EscapeSequenceDiagram />

转义序列在日常编程中频繁出现——`printf("Hello\nWorld");` 会在 "Hello" 和 "World" 之间换行。`printf("He said \"Hi\"");` 才能在双引号里再放双引号。

### 常量——"不变的值"怎么写

程序中有些值在整个运行过程中都不会改变——π ≈ 3.14159、一周有 7 天、数组的最大长度等等。C 提供了三种方式来声明 <Term def="程序运行期间值不能改变的量。C 里有三种方式声明常量：#define 宏常量、const 常变量、enum 枚举常量。">常量（constant）</Term>：

| 方式 | 写法 | 特点 |
|---|---|---|
| `#define` | `#define PI 3.14159` | 预处理器文本替换——编译前就把所有 `PI` 换成 `3.14159`，不占用内存 |
| `const` | `const int WEEKS = 7;` | 常变量——跟普通变量一样有类型、占内存，但值不能改 |
| `enum` | `enum &#123; RED, GREEN, BLUE &#125;;` | 枚举常量——给一组相关的整数常量起名字，默认从 0 开始递增 |

`#define` 是 C 语言的传统做法——简单粗暴，但**没有类型检查**，容易出问题。`const` 是更现代、更安全的方式——编译器知道它的类型，能帮你检查错误。

```c
#include <stdio.h>
#define PI 3.14159           // 宏常量：纯文本替换，没有类型
const int DAYS = 7;          // 常变量：有类型（int），只读

int main(void) {
    // DAYS = 8;            // ❌ 编译错误！const 变量不能修改

    enum { SUN, MON, TUE, WED, THU, FRI, SAT };
    // SUN=0, MON=1, TUE=2, ..., SAT=6

    printf("圆周率: %f\n", PI);       // PI 被替换成 3.14159
    printf("一周有 %d 天\n", DAYS);
    printf("周三的枚举值是 %d\n", WED);  // 输出 3
    return 0;
}
```

### printf() 格式占位符——输出时告诉电脑"怎么看这段数据"

<Term def="C 标准库中最常用的输出函数，通过格式占位符来决定如何将后续参数的值转换成字符输出。%d 输出整数、%f 输出浮点数、%c 输出字符。">printf()</Term> 的格式占位符跟你选择的类型是**一一绑定**的——用错占位符，输出不是你想要的结果。下面是最常用的几种：

| 类型 | printf 占位符 | 示例 |
|---|---|---|
| `int` | `%d`（有符号）、`%u`（无符号） | `printf("%d", 42);` → `42` |
| `short` | `%hd` | `printf("%hd", s);` |
| `long` | `%ld` | `printf("%ld", l);` |
| `long long` | `%lld` | `printf("%lld", ll);` |
| `char` | `%c`（字符）、`%d`（整数） | `printf("%c", 'A');` → `A` |
| `float` | `%f` | `printf("%f", 3.14f);` → `3.140000` |
| `double` | `%f`（或 `%lf`，但 printf 中等价） | `printf("%f", 3.14);` → `3.140000` |
| 指针/地址 | `%p` | `printf("%p", &x);` |

<Callout type="warn">
  `printf` 不会帮你检查格式占位符和参数类型是否匹配——你写 `printf("%d", 3.14)` 不会报错，但输出会是乱七八糟的数字。**这是 C 初学者最常见的坑之一：类型和占位符对不上，编译不报错，运行结果莫名其妙。**
</Callout>





## 动手：认识不同数据类型的"个头"差异

下面用一个 Stepper 分步演示 `char` / `short` / `int` / `double` 四种类型在内存中占用的空间，以及 `sizeof` 运算符如何帮你确认当前平台上每种类型真实的大小。

> 猜一猜：`sizeof(int)` 在你的电脑上是 4 吗？把 `double` 的 8 字节想象成 8 个并排的格子，跟 `char` 的 1 个格子比——同样存一个数，`double` 占的空间是 `char` 的 8 倍！通过下面 Stepper 一步步看。

<Stepper>
  <Step title="① char——1 个字节的小格子">
    <DataTypeMemoryLayout step={1} />
    `char` 只占 1 字节——刚好装下一个 ASCII 字符（或 −128~127 的整数）。`sizeof(char)` 在 C 标准里固定为 1。这是 C 语言中最小的单位，其他所有类型的 `sizeof` 值都是 `sizeof(char)` 的整数倍。
  </Step>
  <Step title="② short——2 个字节的中号格子">
    <DataTypeMemoryLayout step={2} />
    `short`（或 `short int`）占 2 字节——是 char 的两倍。它能存的整数范围也大得多：−32,768 ~ 32,767。如果你要存的数据不会超过这个范围，用 `short` 比 `int` 省一半内存。
  </Step>
  <Step title="③ int——4 个字节的标准格子">
    <DataTypeMemoryLayout step={3} />
    <IntegerRangeDiagram />
    `int` 占 4 字节（绝大多数现代平台上），是 C 语言中最常用的整型。它能存 −21 亿 ~ +21 亿的整数——对大多数日常场景来说足够用了。上图展示了 32-bit 的 signed/unsigned int 的取值范围差异：unsigned 把负数的区间全部"搬"到了正数高端。
  </Step>
  <Step title="④ double——8 个字节的大块头">
    <DataTypeMemoryLayout step={4} />
    `double` 占 8 字节——是 int 的两倍、char 的八倍。这 8 字节的庞大空间给了它约 15 位有效数字的精度——地球到太阳的距离（约 1.5×10¹¹ 米）用 `float` 存只能精确到约 10 万米，而 `double` 能精确到约 0.1 毫米。
  </Step>
</Stepper>

<Callout type="tip">
  `sizeof` 是 C 语言的关键字（不是函数！），能告诉你任何一个类型或变量在当前平台上占多少字节。
  `sizeof(int)` 返回 `int` 的字节数，`sizeof(x)` 返回变量 x 的字节数，`sizeof(char)` 永远等于 1。
  用它来确认类型大小，别凭记忆猜——不同平台上 `long` 可能是 4 字节（Windows）也可能是 8 字节（Linux/macOS）。
</Callout>



## 代码速查：类型声明与 printf 的写法

下面是在你需要时快速对照的 C 类型声明与输出代码：

```c
#include <stdio.h>
int main(void) {
    // ===== 整型声明 =====
    short s = 32767;            // short 2 字节
    int i = 42;                 // int 4 字节
    long l = 123456789L;        // long，后缀 L
    unsigned int u = 4000000000U; // unsigned，后缀 U

    // ===== 浮点声明 =====
    float  f = 3.14f;           // float，必须后缀 f！
    double d = 3.14159265358979; // double（默认）

    // ===== 字符声明 =====
    char ch   = 'A';            // char
    char code = 65;             // 等价于 'A'

    // ===== printf 输出 =====
    printf("int:    %d\n", i);          // 42
    printf("float:  %f\n", f);          // 3.140000
    printf("double: %.15f\n", d);       // 15 位小数全显示
    printf("char:   %c = %d\n", ch, ch);// A = 65
    return 0;
}
```

`float` 的字面值必须写 `f` 后缀——这是最常见的忘记点。`3.14` 是 `double` 类型的字面值，你如果把它赋给 `float` 变量，编译器会悄悄做一个 double→float 的转换，虽然一般没大问题，但养成好习惯：`float` 字面值带 `f`、`long` 字面值带 `L`。



## 容易踩的坑

<Callout type="trap">
  **现象**：`int x = 3.14; printf("%d", x);` 输出 3，但 `printf("%d", 3.14)` 输出的是乱七八糟的大整数。
  **原因**：第一条 `int x = 3.14` 编译器会把 3.14 截断成 3 再存入 x——这是把 double 赋值给 int 时的隐式截断，虽然丢信息但不报错。第二条直接 `printf("%d", 3.14)` 是格式占位符和参数类型不匹配——%d 期望 int（4 字节），实际给的 3.14 是 double（8 字节），printf 解读前 4 个字节当 int，读到的是 double 二进制表示的前半截，当然不是 3。
  **修法**：赋值时如果要截断，用显式转换 `int x = (int)3.14;` 表明你知道在做什么。printf 的占位符必须跟参数类型严格对应——`printf("%f", 3.14);` 才是对的。
</Callout>

<Callout type="trap">
  **现象**：`int a = 3, b = 2; float result = a / b; printf("%f", result);` 输出 1.000000，不是 1.500000。
  **原因**：这是 **整数除法** 的陷阱——两个 int 做 `/` 运算，C 执行的是整数除法（结果截断为整数），得到 1 之后才把 1 转换成 float 赋给 result。整个过程里 1.5 从没出现过。
  **修法**：把至少一个操作数转成浮点数——`float result = (float)a / b;` 或 `float result = 3.0 / 2;`。一旦有一方是浮点数，整条运算就按浮点规则来。
</Callout>

<Callout type="trap">
  **现象**：`printf("圆周率≈%f\n", 3.14159);` 输出 `3.141590`——小数点后默认 6 位，想多显示几位或控制位数不知道怎么调。
  **原因**：`%f` 默认输出 6 位小数。`float` 的有效数字约 6 位——超过 6 位的数字是随机填充的垃圾。`double` 约 15 位——可以用 `%.15f` 多显示几位。
  **修法**：`printf("%.2f", val)` 保留 2 位小数，`printf("%.10f", val)` 保留 10 位小数。格式是 `%.Nf`，N 是要保留的小数位数。浮点数用 `double`（不要用 `float`）存，用 `%.15f` 显示才靠谱。
</Callout>



## 小结

- C 语言要求每个变量在声明时必须指定类型——类型决定了内存大小、取值范围和 printf 的格式占位符
- 基本类型分三类：整型（short/int/long/long long + unsigned 修饰）、浮点型（float/double/long double）、字符型（char）
- 整型修饰符 short/long/unsigned 让你精确控制变量的大小和符号——signed 和 unsigned 混用时务必要小心隐式转换
- printf 的格式占位符（%d/%f/%c/%u）必须与参数类型一一匹配——编译器不帮你检查，用错只有运行时才知道
- 用 `sizeof` 关键字确认当前平台的类型大小——`long` 在 Windows 上是 4 字节、Linux/macOS 上是 8 字节，别默记

五处问题：
① `int total = pi;`——double 字面值赋给 int，小数部分悄悄截断，total 变成 3
② `unsigned int count = -1;`——给 unsigned 赋负数，count 变成 4294967295（回绕），不是 −1
③ `printf("%d", pi);`——占位符 %d 期望 int，实际 pi 是 float，输出乱码
④ `printf("%d", count);`——count 是 unsigned int，应用 %u 而不是 %d
⑤ `printf("%f", total);`——占位符 %f 期望 double，实际 total 是 int，输出乱码

修正后：

```c
#include <stdio.h>
int main(void) {
    double pi = 3.14159265358979;        // 用 double 的完整精度
    int total = (int)pi;                 // 显式截断，你知道会丢小数
    int count = -1;                      // 负数不能用 unsigned
    printf("pi = %.10f\n", pi);          // %f 配对 double，%.10 显示 10 位
    printf("count = %d\n", count);       // %d 配对 int
    printf("total = %d\n", total);       // %d 配对 int
    return 0;
}
```

</Answer>

**练习 2（问答型）** 读下面这段程序，写出 `printf` 每一行的输出结果，并解释为什么。

```c
#include <stdio.h>
int main(void) {
    int a = 5, b = 2;
    float c = a / b;                    // ①
    float d = (float)a / b;             // ②
    int e = 2.9;                        // ③
    printf("c = %f\n", c);              // ④ 输出？
    printf("d = %f\n", d);              // ⑤ 输出？
    printf("e = %d\n", e);              // ⑥ 输出？

    unsigned int u = 0;
    u = u - 1;
    printf("u = %u\n", u);              // ⑦ 输出？
    return 0;
}
```

<Answer>

④ 输出 `c = 1.000000`——`a / b` 是整数除法，得 1，然后才转成 float 赋给 c
⑤ 输出 `d = 2.500000`——`(float)a` 把 a 先转成浮点数，除法变成浮点除法，得 2.5
⑥ 输出 `e = 2`——`2.9` 是 double 字面值，赋给 int 时截断小数部分，变成 2
⑦ 输出 `u = 4294967295`（32-bit）——unsigned int 的 0 减 1 发生回绕，变成最大值 2³²−1

核心教训：整数除法和浮点除法的结果完全不同；赋值截断丢失信息但编译器不报错；unsigned 减法回绕是经典 bug 来源。

</Answer>

**练习 3（综合型）** 写一段程序，声明 `char`、`short`、`int`、`long long`、`float`、`double` 六个变量并分别用 `sizeof` 输出它们在你机器上的字节数。然后把 `char` 的值分别当字符（%c）和整数（%d）输出——验证同一位数据在不同解读方式下的差异。

<Answer>

```c
#include <stdio.h>
int main(void) {
    char   ch   = 'A';
    short  s    = 100;
    int    i    = 1000;
    long long ll = 1000000LL;
    float  f    = 3.14f;
    double d    = 3.14159265358979;

    printf("sizeof(char)      = %zu\n", sizeof(ch));
    printf("sizeof(short)     = %zu\n", sizeof(s));
    printf("sizeof(int)       = %zu\n", sizeof(i));
    printf("sizeof(long long) = %zu\n", sizeof(ll));
    printf("sizeof(float)     = %zu\n", sizeof(f));
    printf("sizeof(double)    = %zu\n", sizeof(d));

    // 同一个 char，两种解读
    printf("\nchar as char: %c\n", ch);   // 输出 A
    printf("char as int:  %d\n", ch);    // 输出 65（ASCII 码）

    return 0;
}
```

`sizeof` 的返回值类型是 `size_t`，用 `%zu` 输出。不同平台上 `long` 和 `long double` 的 `sizeof` 值可能不同——这正是本章要讲的核心：类型大小取决于编译器和你运行的平台，别死记数值。

</Answer>

<Glossary>

<GlossaryItem term="类型（type）">
  数据的一张"规格标签"。它告诉编译器：这个变量占几字节内存、能存什么范围的值、能用什么格式输出。C 是静态类型语言——类型在编译时就确定了，此后不能变。详见本章「C 的数据类型家族」一节。
</GlossaryItem>

<GlossaryItem term="基本数据类型（basic data type）">
  C 语言内置的最基础的数据种类，不需要 #include 任何头文件就能直接用。包括整型家族（short/int/long/long long）、浮点家族（float/double/long double）、字符型（char）。详见本章「基本数据类型一览」一节和 CTypeSizeDiagram 图示。
</GlossaryItem>

<GlossaryItem term="修饰符（modifier）">
  加在整型名字前面的关键字，用来调整类型的大小和符号。short 缩小范围却省内存，long 拉大范围但占用更多空间，unsigned 去掉负数、把原来给负数的区间全部加到正数那边。修饰符可以组合使用，如 unsigned long long。详见本章「整型的修饰符」一节和 IntegerRangeDiagram 图示。
</GlossaryItem>

<GlossaryItem term="浮点类型（floating-point type）">
  能存带小数点的数字的类型。float 是单精度（约 6 位有效数字），double 是双精度（约 15 位有效数字），long double 是扩展精度。浮点数本质上是近似存储——有舍入误差，就像十进制里 1/3 永远写不完一样，二进制里很多小数也写不完。详见本章「浮点类型」一节。
</GlossaryItem>

<GlossaryItem term="char">
  C 语言存单个字符的类型，占 1 字节。在计算机内部，每个字符都对应一个 ASCII 码整数——'A' 就是 65，'a' 就是 97。所以 char 本质上是一个「穿字符外衣的小整数」——用 %c 输出就是字符，用 %d 输出就是 ASCII 码值。详见本章「字符类型与转义序列」一节。
</GlossaryItem>

<GlossaryItem term="ASCII 码">
  计算机里表示字母、数字、标点符号的一套标准编码。每个常用字符对应 0~127 中的一个整数。比如 'A'=65，'a'=97，'0'=48，空格=32。C 的 char 类型底层就是靠这套编码来区分和存储字符的。详见本章「字符类型与转义序列」一节。
</GlossaryItem>

<GlossaryItem term="转义序列（escape sequence）">
  以反斜杠 \ 开头、后跟一个或几个字符的组合，用来表示没法直接敲出来的特殊字符。\n 表示换行（把光标移到下一行开头），\t 表示制表（跳格对齐），\\ 表示反斜杠本身，\" 表示双引号。反斜杠就是「转义符」——它把后面那个普通字母的意思给转了。详见本章「字符类型与转义序列」一节和 EscapeSequenceDiagram 图示。
</GlossaryItem>

<GlossaryItem term="常量（constant）">
  程序运行期间值不能改变的量。C 有三种声明常量的方式：#define 是预处理器文本替换（没类型，简单粗暴），const 是有类型的只读变量（现代推荐），enum 给一组相关整数起名字（适合红灯/绿灯/蓝灯这种取值有限的场景）。详见本章「常量」一节。
</GlossaryItem>

<GlossaryItem term="printf()">
  C 标准库里最常用的输出函数。通过格式占位符（%d、%f、%c 等）告诉电脑「把后面的参数按什么格式解释和输出」。类型和占位符必须一一对应——用错不报错，但输出的是乱码。这是 C 初学者最常见的坑之一。详见本章「printf() 格式占位符」一节。
</GlossaryItem>

</Glossary>
