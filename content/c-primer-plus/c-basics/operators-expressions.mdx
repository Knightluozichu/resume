## 为什么 C 的运算像一条"加工流水线"？

想象一条巧克力工厂的流水线：原料从传送带左边进来，经过混合、加热、冷却、切割、包装五道工序，右边出来的就是成品。每道工序的输入是上一道的输出，工序之间有固定的先后顺序——你先称量再混合，而不是反过来。

在 C 语言里，**表达式就是这条传送带，运算符就是上面的加工机器**。你写 `a + b * c`，C 不是从左到右一个个算——它先让 `*` 机器（乘法）工作，把 `b * c` 的中间结果传给 `+` 机器（加法），最后才出结果。运算符的优先级和结合律就是这条传送带上各机器的"开动顺序表"。

**如果没有这套规则呢？** `3 + 4 * 5` 的结果可能是 35（从左算）也可能是 23（先乘后加）——同一行代码在不同机器上跑出不同答案。更可怕的是，`++i` 和 `i++` 如果混在一起用，你会拿到意料之外的中间值。这一章就教你掌握 C 运算流水线上每一台机器的脾气——让每个表达式都按你预想的方式出结果。

### 算术运算符——整数的"地板除法"

<Term def="完成加减乘除和求余运算的符号：+（加/正号）、-（减/负号）、*（乘）、/（除）、%（求余/模运算）。">算术运算符（arithmetic operator）</Term> 看起来跟你小学数学课上的符号一样，但 C 里有两条关键差异：

1. **整数除法是"地板除法"——结果往零截断。** `5 / 2` 在 C 里等于 2，不是 2.5。小数部分被直接丢弃（不是四舍五入）。`-5 / 2` 的结果是 -2（C99 起规定往零截断）。如果你想要小数结果，至少一个操作数要是浮点数：`5.0 / 2.0` 或 `5.0 / 2` 都会得到 2.5。

2. **`%` 求余运算符只能用于整数。** `10 % 3` 的结果是 1——10 除以 3 商 3 余 1。`%` 很好记——"百分号求余，结果是除法剩下的那点不能整除的部分"。

```c
#include <stdio.h>
int main(void) {
    printf("5 / 2   = %d\n", 5 / 2);    // 2（整数除法，截断）
    printf("5.0 / 2 = %.1f\n", 5.0 / 2); // 2.5（浮点除法）
    printf("-5 / 2  = %d\n", -5 / 2);    // -2（往零截断）
    printf("10 %% 3 = %d\n", 10 % 3);    // 1（求余）
    printf("10 %% 2 = %d\n", 10 % 2);    // 0（整除，无余数）
    return 0;
}
```

<Callout type="tip">
  `%` 求余的结果符号与**被除数（左边）**同号：`-10 % 3` = -1，`10 % -3` = 1（C99 起规定）。记住这个规则就不会被负数的余数搞糊涂。
</Callout>

### 关系与逻辑运算符——"是真是假"的判断器

<Term def={'用于比较两个值大小关系的运算符：小于、小于等于、大于、大于等于、等于、不等于。结果是一个整数值：关系成立为 1、不成立为 0。'}>关系运算符（relational operator）</Term> 和 <Term def="对真/假条件进行组合的运算符：&&（逻辑与，两边都真才真）、||（逻辑或，任一边真就真）、!（逻辑非，真变假、假变真）。">逻辑运算符（logical operator）</Term> 的作用是"判断条件的真假"——它们在 `if` 和 `while` 的条件里是主力。

C 语言没有专门的"布尔类型"（C99 之后有了 `_Bool`，但本质仍然是整数）。**关系表达式的结果是整数：成立返回 1，不成立返回 0。** 反过来，任何非零值在 C 里都是"真"，只有 0 是"假"。

```c
#include <stdio.h>
int main(void) {
    int a = 5, b = 3;

    printf("a > b   = %d\n", a > b);    // 1（5 大于 3，真）
    printf("a < b   = %d\n", a < b);    // 0（5 小于 3，假）
    printf("a == 5  = %d\n", a == 5);   // 1（a 等于 5，真）
    printf("a != 5  = %d\n", a != 5);   // 0（a 等于 5，不"不等于"）

    // 逻辑运算符：组合多个条件
    printf("a>0 && b>0  = %d\n", a > 0 && b > 0);  // 1（两边都真）
    printf("a>0 && b<0  = %d\n", a > 0 && b < 0);  // 0（一边假就全假）
    printf("a<0 || b>0  = %d\n", a < 0 || b > 0);  // 1（任一边真就真）
    printf("!(a > 0)    = %d\n", !(a > 0));         // 0（真变假）
    return 0;
}
```

<Callout type="trap">
  `==` 是"等于比较"，`=` 是"赋值"——**用 `=` 代替 `==` 在条件里是老 C 程序员的经典翻车现场。** 写成 `if (a = 5)` 不是判断 a 是否等于 5，而是把 5 赋给 a、然后把非零值 5 当"真"——条件永远成立。编译器通常会警告，但不要依赖警告来救你。
</Callout>

### 自增自减运算符——C 语言最"狡猾"的符号

<Term def="一元运算符：++i（前缀递增，先加后用）、i++（后缀递增，先用后加）、--i（前缀递减）、i--（后缀递减）。两者对变量最终值的修改相同，但表达式的返回值不同——这是初学者最容易搞混的点。">自增运算符（++）和自减运算符（--）</Term> 可能是 C 语言里最让人困惑的两个符号。它们都只做一件事——让变量加 1 或减 1——但**前缀和后缀的区别在于表达式返回的是新值还是旧值**。

<IncrementTimelineDiagram />

用代码感受一下这个差异：

```c
#include <stdio.h>
int main(void) {
    int i = 5, j = 5;

    // ++i：先加后用——表达式值 = 新值 6
    printf("++i: i=%d, 表达式值=%d\n", i, ++i);
    // 输出：++i: i=6, 表达式值=6

    // i++：先用后加——表达式值 = 旧值 5
    // ⚠ printf 参数求值顺序未定义——不建议在同一 printf 里用 j 和 j++
    // 更好写法是分开：
    int old_j = j;
    printf("j++: 表达式值=%d, j=%d\n", old_j, j + 1);
    // 输出：j++: 表达式值=5, j=6

    return 0;
}
```

<Callout type="trap">
  **同一条语句里对同一个变量多次使用自增自减是未定义行为。** 比如 `i = i++ + ++i;` 在不同的编译器上可能得到不同的结果——C 标准不规定子表达式的求值顺序。**规则很简单：一条语句里，同一个变量最多自增/自减一次。**
</Callout>

### 赋值与复合赋值运算符

`=` 是简单的赋值运算符——把右边的值拷贝到左边变量里。但 <Term def="把算术/位运算和赋值合并成一个运算符的简写形式：+=、-=、*=、/=、%= 等。a += 3 等价于 a = a + 3，但更简洁、有时能生成更高效的代码。">复合赋值运算符（compound assignment operator）</Term> 能同时做运算和赋值，把代码写得更紧凑。

```c
#include <stdio.h>
int main(void) {
    int n = 10;

    n += 5;    // 等价于 n = n + 5;  → n 变成 15
    n -= 3;    // 等价于 n = n - 3;  → n 变成 12
    n *= 2;    // 等价于 n = n * 2;  → n 变成 24
    n /= 4;    // 等价于 n = n / 4;  → n 变成 6
    n %= 4;    // 等价于 n = n % 4;  → n 变成 2

    printf("n = %d\n", n);  // 输出 2
    return 0;
}
```

复合赋值运算符读起来也很自然：`n += 5` 读作"n 增加 5"。

### 逗号运算符——让你在一行里做多件事

<Term def="优先级最低的二元运算符，写法和逗号一样：表达式1, 表达式2。先求值左边的表达式、丢弃结果，再求值右边的表达式、把它作为整个逗号表达式的结果。主要用于 for 循环里放多个初始化/更新语句。">逗号运算符（comma operator）</Term> `,` 优先级在 C 所有运算符中最低。它先求值左边的表达式（丢弃其结果），再求值右边，**整个逗号表达式的结果就是最右边表达式的结果**。

```c
int x, y;
x = (y = 3, y + 1);  // 先执行 y=3（y 变成 3），再计算 y+1=4
                      // x 的值是 4，y 的值是 3
```

最常见的用处是在 `for` 循环里同时初始化或更新两个变量（这是下两章的内容，先留个印象）：

```c
for (i = 0, j = 10; i < j; i++, j--) { /* ... */ }
```

## 表达式与类型转换——当 int 遇上 float

当一个表达式里混着不同类型的操作数时——比如 `3.14 * 5`（float 乘 int）——C 必须决定"按哪种类型来算"。这就是 <Term def="C 语言在表达式求值时自动将操作数从较低精度类型转为较高精度类型的一组规则。目的是避免数据丢失：先做整型提升（char/short → int），再做寻常算术转换（类型低的转成类型高的）。">类型转换（type conversion）</Term>。

C 的类型转换分两阶段：

1. **整型提升：** `char` 和 `short` 在表达式里首先被自动提升为 `int`（如果 `int` 表示范围不够，则升为 `unsigned int`）。所以 `char c = 'A'; c + 1` 是按 int 来算的，结果是 66。

2. **寻常算术转换：** 提升后如果还有不同类型，C 按一个固定的层次链把"级别低的"转成"级别高的"：

<TypePromotionDiagram />

转换规则的核心原则是"不丢数据"：`5 / 2.0` ——`2.0` 是 double，`5` 被转换为 double 后做浮点除法，结果 2.5。

```c
#include <stdio.h>
int main(void) {
    char ch = 'A';          // 'A' 的 ASCII 码是 65
    int  n  = 5;
    double d = 2.0;

    printf("ch + 1 = %d\n", ch + 1);        // 66（char → int 提升，加 1）
    printf("'A' + 1 = %c\n", 'A' + 1);      // 'B'（用 %c 打印 int 为字符）

    printf("n / 3  = %d\n", n / 3);         // 1（整数除法，截断）
    printf("n / d  = %.2f\n", n / d);       // 2.50（int → double，浮点除法）
    printf("n / 3.0 = %.2f\n", n / 3.0);    // 1.67（int → double，浮点除法）

    return 0;
}
```

### 强制类型转换——你说了算

当你需要显式地把一种类型转换成另一种类型（而不是靠 C 自动帮你做），用 <Term def="一元运算符 (类型名)表达式，它显式地把表达式的值转换为指定类型。与隐式转换不同，cast 是你手动控制的——你清楚自己在做什么、接受可能的精度损失。">强制类型转换（cast）</Term>：

```c
int a = 5, b = 2;
double result;

result = a / b;              // result = 2.0（先整数除法得 2，再赋值时转为 double）
result = (double)a / b;      // result = 2.5（a 先被 cast 成 double，浮点除法）
result = (double)(a / b);    // result = 2.0（先整数除法得 2，再 cast 成 double）
```

强制类型转换的写法是 `(目标类型)表达式`。**cast 的优先级很高**——`(double)a / b` 把 a 转为 double 再做除法（对），而 `(double)(a / b)` 先做整数除法再把结果转 double（错！你已经把小数丢掉了再转 double 也白搭）。

<Callout type="trap">
  赋值时的类型转换是"窄化"——从高级别类型往低级别截断。`int n = 3.99;` 的结果不是 4，而是 3（小数部分直接被丢弃，不是四舍五入）。`float f = 3.1415926535;` 会丢失 double 的精度——`double` 有约 15 位有效数字，`float` 只有约 6 位。
</Callout>

## 把运算符和 while 循环串起来

有了运算符基础，就可以用 `while` 循环来反复执行一段代码了。`while` 是 C 里最基本的结构之一。

<WhileLoopFlowDiagram />

<Term def="C 语言的入口条件循环：while (条件) { 循环体 }。先检查条件——条件为真则执行循环体，执行完后回到条件检查；条件为假则跳过整个循环体、往下执行。循环体可能一次都不执行。">while 循环</Term> 的语法很简单：

```c
while (条件表达式) {
    // 循环体：条件为真时反复执行
    // 每次执行完回到条件检查
}
```

循环体如果只有一条语句，花括号 `&#123;&#125;` 可以省略，但**强烈建议总是写花括号**——它能防止后续添加代码时忘了加花括号而产生的 bug。

<Stepper>

    变量 `i` 先被初始化为 0。这一步只是把"计数器"归零——`while` 循环本身还没开始执行。`i = 0` 这行写在循环外面，只执行一次。确保循环前条件变量有明确的值是写出正确循环的第一步。
  </Step>
  <Step title="第二步：检查条件，执行循环体">
    <WhileLoopFlowDiagram step={2} />
    `while (i < 5)` 检查 `i < 5` 是否为真。`0 < 5` 成立——进入循环体，执行 `printf("%d\n", i)` 输出 0。循环体只做"用一下 i 当前的值"这件事——更新 i 的任务在第三步。while 的核心规则：**先判后做，条件不成立时一次都不做。**
  </Step>
  <Step title="第三步：更新条件变量（自增）">
    <WhileLoopFlowDiagram step={3} />
    循环体末尾 `i++` 把 `i` 从 0 变成了 1。自增运算符在这里的作用就是**逐步推动循环向终止条件靠近**——如果忘了写自增，循环条件永远成立，程序就"死循环"了。注意：循环体里每次只加 1，不会一次跳到 5。
  </Step>
  <Step title="第四步：回到条件判断——循环继续或退出">
    <WhileLoopFlowDiagram step={4} />
    执行完 `i++` 后，流程回到 `while (i < 5)` 重新判断。现在 `i = 1`，仍然 `1 < 5`——再来一轮。这个"检查 → 执行 → 更新 → 回到检查"的循环一直持续到 `i` 变成 5——此时 `5 < 5` 不成立（相等不满足小于），循环退出。**循环退出时 i = 5，不是 4。** 这是 while 最常见的边界陷阱——循环条件是 `i < 5` 时，循环体里 `i` 的最大值是 4，但退出循环后 `i` 的值是 5。把 `i < 5` 改成 `i <= 5`，输出就会多一个 5。
  </Step>
</Stepper>

下面是一段完整可运行的 while 循环程序——打印 1 到 10 及其平方值：

```c
#include <stdio.h>
int main(void) {
    int n = 1;

    printf("数字\t平方\n");
    printf("----\t----\n");

    while (n < 11) {
        printf("%d\t%d\n", n, n * n);
        n++;  // 每次循环把 n 加 1——忘了这行就死循环！
    }

    return 0;
}
```

输出：
```
数字  平方
----  ----
1     1
2     4
3     9
4     16
5     25
6     36
7     49
8     64
9     81
10    100
```

注意 `while (n < 11)` 这个写法——当 n = 10 时满足条件、进入循环；n 变成 11 时 `11 < 11` 不成立、循环结束。`n <= 10` 和 `n < 11` 在这个场景下效果一样，但 `n < 11` 明确告诉你"这条循环跑到 10 为止"。







## 代码详解：四个核心场景逐段拆解

### 场景一：自增自减的完整示例——前缀 vs 后缀的全景

```c
#include <stdio.h>
int main(void) {
    int a = 5, b = 5;
    int result;

    // 前缀递增：先加后用
    result = ++a;
    printf("++a: a=%d, result=%d\n", a, result);  // a=6, result=6

    // 后缀递增：先用后加
    result = b++;
    printf("b++: result=%d, b=%d\n", result, b);  // result=5, b=6

    // 单独使用时效果相同
    int x = 3, y = 3;
    ++x;
    y++;
    printf("单独使用: x=%d, y=%d\n", x, y);        // x=4, y=4

    return 0;
}
```

关键区别只在**表达式有赋值或函数调用**时才体现——把 `++a` 的结果赋给 `result`，`result` 得到新值 6；把 `b++` 的结果赋给 `result`，`result` 得到旧值 5。**如果单独写一行 `++i;` 或 `i++;`，两者对 i 的效果完全一样——都让 i 加 1。** 这个区别让小白很慌，但搞清楚"表达式值"和"变量值"是两个概念之后就简单了。

### 场景二：复合赋值——用 while 循环做累加

```c
#include <stdio.h>
int main(void) {
    int sum = 0;
    int i = 1;

    while (i <= 10) {
        sum += i;  // 等价于 sum = sum + i
        i++;
    }

    printf("1+2+...+10 = %d\n", sum);  // 输出 55
    return 0;
}
```

这段代码把复合赋值和 while 循环结合起来——`sum += i` 每次把当前 i 的值累加到 sum 上，`i++` 推动循环前进。循环体只有两行，却完成了"从 1 加到 10"的计算——**运算符和循环配合得当，几行代码就能做很多事。**

### 场景三：类型转换陷阱——赋值窄化与精度丢失

```c
#include <stdio.h>
int main(void) {
    double pi = 3.1415926535;
    float  f  = pi;          // double → float，精度丢失
    int    n  = pi;          // double → int，小数截断

    printf("pi (double) = %.10f\n", pi);  // 3.1415926535
    printf("f  (float)  = %.10f\n", f);   // 3.1415927410（丢了精度！）
    printf("n  (int)    = %d\n", n);      // 3（小数全丢）

    // 除法中的类型差异
    printf("5/2   = %d\n", 5/2);          // 2（整数除法）
    printf("5/2.0 = %.1f\n", 5/2.0);      // 2.5（5转double，浮点除法）
    printf("5.0/2 = %.1f\n", 5.0/2);      // 2.5（2转double，浮点除法）

    return 0;
}
```

`float` 只有约 6-7 位有效数字——把 `double` 的 10 位有效数字全部塞进 `float`，后面的就被丢掉了。`int` 赋值截断更粗暴——`3.99` 也照样变成 3。理解了这些"数据在类型之间转移时可能被截断/丢失精度"，你就能避免因为"数学上是对的但计算机上错了"而抓狂。

### 场景四：逗号运算符与 cast 的组合用法

```c
#include <stdio.h>
int main(void) {
    int    i = 5;
    double d = 3.14159;
    int    result;

    // 逗号运算符：两个表达式顺序执行，结果是右边
    // i, i*2 的结果是 10（i*2 的值）
    printf("(i, i*2) = %d\n", (i, i * 2));  // 10

    // cast 运算符：显式转换
    result = (int)d;                         // double → int，截断
    printf("(int)%.5f = %d\n", d, result);   // 3

    // 逗号和 cast 组合
    int x;
    x = (i++, (int)(d + 0.6));              // i 先变 6，再算 (int)3.74 = 3
    printf("x = %d, i = %d\n", x, i);       // x=3, i=6

    return 0;
}
```

`(i++, (int)(d + 0.6))` 这个表达式里：先用逗号让 `i++` 把 i 从 5 变成 6（丢弃旧值），再算 `d + 0.6 = 3.74`、cast 成 `(int)` 得 3——整个表达式的值是 3。逗号运算符在实际中用得不多，但在 for 循环的初始化和更新段里几乎必不可少。



## 容易踩的坑

<Callout type="trap">
  **现象**：`int a = 5; int b = a + a++;` 打印 b 的值有时是 10，有时是 11——每次编译或跑在不同编译器上结果不一样。
  **原因**：C 标准不规定同一表达式内多个子表达式的求值顺序（`a+a++` 里先算左边 a 还是先算右边 a++？标准留白）。不同的编译器按不同的顺序求值，结果自然不同——这是**未定义行为（undefined behavior）**。
  **修法**：同一条语句里同一个变量最多自增/自减一次。把 `b = a + a++;` 拆成两行：`b = a + a; a++;`——求值顺序由你显式控制，行为在所有编译器上一致。
</Callout>

<Callout type="trap">
  **现象**：`int i = 0; while (i < 10); &#123; printf("%d ", i); i++; &#125;` 程序"卡死"不动，CPU 风扇狂转。
  **原因**：while 条件后面多了个分号 `;`！`while (i < 10);` 被 C 解析成"一个空的循环体"——只要 i < 10 成立就不停地执行空语句（什么都不做），i 永远不变，条件永远成立。下面那行带花括号的 `&#123; printf... &#125;` 根本不是循环体——它只是 while 结束后的一段普通代码，永远执行不到。
  **修法**：去掉 `while (i < 10)` 后面的分号——写成 `while (i < 10) &#123; ... &#125;`。花括号前不放分号是铁律。
</Callout>

<Callout type="trap">
  **现象**：`float f = 1 / 2; printf("%f", f);` 输出 0.000000，而不是 0.5。
  **原因**：`1 / 2` 的两个操作数都是 `int`——C 先做整数除法得 0，再把结果 0 转为 float 赋给 f。你期待的是浮点除法，但 C 只看到两个整数，"为时已晚"——截断发生了。
  **修法**：让至少一个操作数是浮点数——`1.0 / 2`、`1 / 2.0`、`(float)1 / 2` 都会做浮点除法。习惯写浮点字面量时加小数点（`2.0` 而不是 `2`），就能在"看起来一样"的表达式里明确拆分意图。
</Callout>

<Callout type="trap">
  **现象**：`int a = 5, b = 3; if (a = b) printf("相等");` 永远打印"相等"，即使你改了 a 和 b 的值。
  **原因**：`=` 是赋值，不是比较！`a = b` 把 b 的值 3 赋给 a，赋值表达式的结果是 3（非零，即"真"）。条件永远成立——不管 a 和 b 原来是什么值，经过 `a = b` 后条件一定为真。
  **修法**：判断相等用 `==`——`if (a == b)`。一个自检习惯：在写"相等判断"时心里默念"双等号"，写完看一眼有没有写成单等号。
</Callout>



## 小结

- C 的五大类运算符各有明确的语义：算术（含截断除法和求余）、关系与逻辑（结果是整数 1/0）、自增自减（前缀新值、后缀旧值）、复合赋值（运算+赋值缩写）、逗号（顺序执行，取最右）
- `++i` 返回新值（先加后用），`i++` 返回旧值（先用后加）——单独使用时效果相同，有赋值或传参时表达式值不同
- 混合类型表达式按整型提升 + 寻常算术转换自动升级；赋值时反向截断（窄化，可能丢数据）；cast 显式控制转换
- while 循环遵循"检查条件 → 执行循环体 → 更新条件 → 回到检查"四步；条件不成立时循环体一次都不执行
- 最易踩的坑：同一变量多次自增自减（未定义行为）、while() 后多了分号（空循环体死循环）、`=` 写成 `==`（赋值当比较）

**练习 1（改代码型）** 下面这段程序有四处错误或隐患。找出每一处并修正，写出完整的改正后代码。

```c
#include <stdio.h>
int main(void) {
    int n = 1;
    while (n <= 10); {
        printf("%d^2 = %d\n", n, n * n);
    }
    printf("n 最终值 = %d\n", n);
    return 0;
}
```

<Answer>

两处问题：
① `while (n <= 10);` —— 多了一个分号！这个分号成了"空循环体"，n 永远不变，程序死循环。去掉分号。
② 循环体内缺少 `n++` —— 没有更新条件变量，即使去掉分号也会死循环。在循环体末尾加 `n++`。

修正后：

```c
#include <stdio.h>
int main(void) {
    int n = 1;
    while (n <= 10) {
        printf("%d^2 = %d\n", n, n * n);
        n++;
    }
    printf("n 最终值 = %d\n", n);
    return 0;
}
```

</Answer>

**练习 2（问答型）** 读下面的程序，写出每一行 printf 的输出结果（共 5 行），并逐一解释为什么。

```c
#include <stdio.h>
int main(void) {
    int i = 3, j = 3;

    printf("① 5/2 = %d\n", 5 / 2);
    printf("② 5.0/2 = %.1f\n", 5.0 / 2);
    printf("③ 10%%3 = %d\n", 10 % 3);
    printf("④ ++i = %d, i = %d\n", ++i, i);
    printf("⑤ j++ = %d, j = %d\n", j++, j);
    return 0;
}
```

<Answer>

① 输出 `5/2 = 2`——两个 int 做整数除法，结果截断为 2。
② 输出 `5.0/2 = 2.5`——5.0 是 double，2 被提升为 double，做浮点除法。
③ 输出 `10%3 = 1`——10 除以 3 商 3 余 1。
④ 输出 `++i = 4, i = 4`——前缀递增：先加后用，i 变成 4，表达式返回新值 4。
⑤ **这一行有未定义行为！** printf 的参数求值顺序未定义——在不同的编译器上，可能先算 `j++`（j 变 4，表达式返回 3），也可能先取 `j`（还是 3）再算 `j++`。输出可能为 `j++ = 3, j = 4` 或 `j++ = 3, j = 3`。这不是"你能通过推理得到的确定答案"——正确做法是拆成两行：`int old = j++; printf("⑤ j++ = %d, j = %d\n", old, j);`。

</Answer>

**练习 3（综合实现型）** 写一段程序：用 while 循环打印 1 到 100 之间所有能被 7 整除的数字，每行输出格式为 "7 × n = 结果"（如 "7 × 1 = 7"）。要求必须使用复合赋值运算符 `+=` 来更新循环变量，循环变量初值为 7，每次加 7。

<Answer>

```c
#include <stdio.h>
int main(void) {
    int n = 7;

    while (n <= 100) {
        printf("7 × %d = %d\n", n / 7, n);
        n += 7;  // 每次跳过 7 个——只取 7 的倍数
    }

    return 0;
}
```

> 输出从 `7 × 1 = 7` 到 `7 × 14 = 98`，共 14 行。

循环变量 n 从 7 开始，每次 `n += 7` 直接跳到下一个 7 的倍数，天然只打印被 7 整除的数。你也不一定非要从 1 到 100 逐次检查——直接把循环变量设成"你要检查的那些值"，循环体就简洁得多。

</Answer>

## 名词解释

<Glossary>

<GlossaryItem term="算术运算符（arithmetic operator）">
  完成加减乘除和求余运算的符号：`+`（加/正号）、`-`（减/负号）、`*`（乘）、`/`（除）、`%`（求余/模运算）。它们和小学数学符号长得一样，但 C 里整数除法会截断（丢掉小数部分）、`%` 求余只能用于整数。详见本章「算术运算符」一节和 COperatorTable 速查表。
</GlossaryItem>

<GlossaryItem term="关系运算符（relational operator）">
  用于比较两个值大小关系的运算符：`<`（小于）、`<=`（小于等于）、`>`（大于）、`>=`（大于等于）、`==`（等于）、`!=`（不等于）。关系表达式的结果是整数：关系成立返回 1，不成立返回 0。`==` 和 `=` 是两回事——一个是"比大小"、一个是"把值塞进去"。详见本章「关系与逻辑运算符」一节。
</GlossaryItem>

<GlossaryItem term="逻辑运算符（logical operator）">
  对真/假条件进行组合的运算符：`&&`（逻辑与，两边都真才真）、`||`（逻辑或，任一边真就真）、`!`（逻辑非，真变假、假变真）。C 里没有单独的"布尔类型"——非零就是真、0 就是假，逻辑运算符的结果也是整数 1 或 0。详见本章「关系与逻辑运算符」一节。
</GlossaryItem>

<GlossaryItem term="自增运算符（++）和自减运算符（--）">
  只让变量加 1 或减 1 的一元运算符。前缀（++i）：先加，表达式的值是新值。后缀（i++）：先返回旧值，再加。两者对变量的最终影响完全相同——单独写 `i++;` 和 `++i;` 效果一样。只有在被赋值或当函数参数时，"表达式值"的差异才体现出来。详见本章「自增自减运算符」一节和 IncrementTimelineDiagram Stepper 图示。
</GlossaryItem>

<GlossaryItem term="复合赋值运算符（compound assignment operator）">
  把算术运算和赋值合并成一个运算符：`+=`、`-=`、`*=`、`/=`、`%=`。`a += 3` 等价于 `a = a + 3`，但更简洁、读起来也更自然（"a 增加 3"）。详见本章「赋值与复合赋值运算符」一节。
</GlossaryItem>

<GlossaryItem term="逗号运算符（comma operator）">
  优先级最低的二元运算符：表达式1, 表达式2。从左到右顺序求值——先算左边、丢弃结果，再算右边、以右边结果为整个表达式的结果。最常见用法是在 for 循环的初始化和更新段里同时做多件事（如 `for (i=0, j=10; ...; i++, j--)`）。详见本章「逗号运算符」一节。
</GlossaryItem>

<GlossaryItem term="类型转换（type conversion）">
  C 语言在表达式求值时自动将不同精度的操作数统一为一组规则：先做整型提升（char/short → int），再做寻常算术转换（按 char→short→int→unsigned→long→float→double→long double 的层次链向上统一）。赋值时反过来——从高级别截断成低级别，可能丢失数据。详见本章「表达式与类型转换」一节和 TypePromotionDiagram 图示。
</GlossaryItem>

<GlossaryItem term="强制类型转换（cast）">
  一元运算符 `(目标类型)表达式`，显式地把表达式的值转换成指定类型。cast 优先级很高——`(double)a / b` 是把 a 先转成 double 再做除法，`(double)(a / b)` 是先做整数除法再把结果转 double（已经丢掉的小数再转也回不来）。详见本章「强制类型转换」一节。
</GlossaryItem>

<GlossaryItem term="while 循环">
  C 语言的入口条件循环结构：`while (条件) &#123; 循环体 &#125;`。先检查条件→条件为真则执行循环体→回到条件检查→条件为假则退出。循环体可能一次都不执行（如果条件一开始就不成立）。条件变量必须在循环体内被更新——忘了更新就会死循环。详见本章「while 循环」一节和 WhileLoopFlowDiagram Stepper 图示。
</GlossaryItem>

</Glossary>
