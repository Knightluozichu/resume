## 为什么程序需要「反复执行」？

想象一条巧克力工厂的流水线：同一道工序要对传送带上的每一块巧克力重复操作——称重、包膜、贴标签。工人不会只做一块就下班，而是「只要还有货，就继续干」。程序里的很多任务也一样：打印 1 到 100、累加用户输入的分数、遍历矩阵每个格子——都需要把同一段代码跑很多遍。

**如果没有循环呢？** 你得把 `printf` 复制粘贴一百遍，改一百次数字。用户输入个数不确定时，你甚至不知道要粘贴多少遍。循环就是 C 语言给你的「流水线开关」：写一次逻辑，让机器按条件反复执行，直到该停的时候停。

这一章教你三种循环开关——`for`、`while`、`do-while`——以及两种经典用法：数着跑（计数循环）和看着停（哨兵值循环）。

## while 回顾：先判后做的入口条件循环

上一章你已经见过 <Term def="先检查条件、条件为真才执行循环体、条件为假则跳过的入口条件循环。循环体可能一次都不执行。">while 循环</Term>。它的规则很简单：**先问「还要不要继续？」，答「要」才干活。**

<WhileLoopFlowDiagram />

当输入数据的个数事先不知道时，常用 <Term def="用一个特殊值（哨兵）标记数据结束；循环每次先读入数据，再判断是否为哨兵。不是哨兵就处理并继续读入，是哨兵则退出。典型哨兵如 -1、0、空行。">哨兵值（sentinel value）</Term> 驱动循环——你不需要提前知道有多少个数字，只要约定「读到 -1 就表示结束」：

<SentinelLoopDiagram />

```c
#include <stdio.h>
int main(void) {
    int item;
    int sum = 0;

    printf("输入整数（-1 结束）：\n");
    scanf("%d", &item);           // 先读第一个

    while (item != -1) {         // -1 是哨兵
        sum += item;
        scanf("%d", &item);       // 处理完再读下一个
    }

    printf("总和 = %d\n", sum);
    return 0;
}
```

这种模式叫「先读入、再判断、再处理、再读入」——第一个数在循环外读，循环内每次处理完再读下一个。也有人把读入放在循环体开头，逻辑等价，只是写法不同。

## for 循环：把三要素写在一行

<Term def="C 语言的计数循环结构：for(初始化; 条件; 更新) { 循环体 }。初始化只执行一次，然后重复 条件判断→循环体→更新→回到条件，直到条件为假。">for 循环</Term> 把循环最需要关心的三件事——**从哪开始、什么时候停、每轮怎么推进**——压缩进括号里：

```c
for (初始化; 条件; 更新) {
    // 循环体
}
```

<ForLoopFlowDiagram />

三要素各司其职：

- **初始化（init）**：设置 <Term def="在循环中用来计数或推动循环向终止条件靠近的变量，如 i、n、row。必须在循环体内被更新，否则可能死循环。">循环控制变量（loop control variable）</Term>，只执行一次
- **条件（cond）**：每轮开始前检查，为真才进入循环体
- **更新（update）**：每轮循环体结束后执行，通常用 `i++` 推动控制变量

> 猜一猜：下面 `for` 循环结束后，`i` 的值是多少？通过 Stepper 一步步揭晓。

<Stepper>
  <Step title="第一步：初始化（只执行一次）">
    <ForLoopFlowDiagram step={1} />
    `int i = 0` 在循环开始前执行一次——把计数器归零。这一步不属于「重复」的部分，for 的语法把它写在括号最左边，强调「循环从什么状态出发」。
  </Step>
  <Step title="第二步：检查条件">
    <ForLoopFlowDiagram step={2} />
    判断 `i < 5`。`0 < 5` 为真——进入循环体。如果这里为假，循环体一次都不执行，直接跳到 for 后面的语句。for 和 while 在「先判后做」这一点上是一样的。
  </Step>
  <Step title="第三步：执行循环体">
    <ForLoopFlowDiagram step={3} />
    执行花括号里的代码，比如 `printf("%d\n", i)` 输出当前的 i。循环体里一般不改控制变量的步进逻辑——那是第四步的事。
  </Step>
  <Step title="第四步：更新并回到条件">
    <ForLoopFlowDiagram step={4} />
    执行 `i++`，i 从 0 变成 1，然后流程回到第二步重新判断。当 i 变成 5 时，`5 < 5` 为假，循环结束。**退出后 i = 5**——这是 `<` 边界最常见的陷阱。
  </Step>
</Stepper>

典型计数循环——打印 1 到 10 的平方：

```c
#include <stdio.h>
int main(void) {
    int i;

    for (i = 1; i <= 10; i++) {
        printf("%d^2 = %d\n", i, i * i);
    }

    return 0;
}
```

`i = 1` 起步，`i <= 10` 控制最后一轮是 10，`i++` 每轮加 1。三个要素写在同一行，一眼看清循环的「起止步长」——这就是 for 比 while 更适合**次数已知**场景的原因。

逗号运算符在 for 里很常见——同时初始化或更新两个变量：

```c
for (i = 0, j = 10; i < j; i++, j--) {
    printf("i=%d j=%d\n", i, j);
}
```

## do-while：至少执行一次的出口条件循环

<Term def="先执行循环体、再检查条件的出口条件循环：do { 循环体 } while (条件); 分号不能少。保证循环体至少运行一次，适合菜单、输入校验等必须先做一次的场景。">do-while 循环</Term> 和 while 的最大区别：**先干活，再问要不要继续。**

<DoWhileVsWhileDiagram />

语法注意末尾的分号——`while (条件);` 这里的分号是 do-while 语法的一部分，不能省：

```c
do {
    // 循环体——至少执行一次
} while (条件);   // 分号必须有！
```

菜单程序是经典用例——总得先显示选项，再让用户选：

```c
#include <stdio.h>
int main(void) {
    int choice;

    do {
        printf("1. 新建  2. 打开  0. 退出\n");
        printf("请选择：");
        scanf("%d", &choice);
        /* 根据 choice 处理 ... */
    } while (choice != 0);

    return 0;
}
```

即使用户第一次就输入 0，菜单也会显示一次——这就是 do-while 和 while 的分水岭。

## 嵌套循环：外圈换行、内圈扫列

<Term def="在一个循环的循环体内再写另一个循环。外循环每走一轮，内循环完整执行一遍。总执行次数 ≈ 外层次数 × 内层次数。常用于二维数组、矩阵、图案打印。">嵌套循环（nested loop）</Term> 就是「循环里套循环」——外循环控制行，内循环控制列，合起来遍历整张表：

<NestedLoopDiagram />

打印九九乘法表——外循环控制被乘数、内循环控制乘数：

```c
#include <stdio.h>
int main(void) {
    int i, j;

    for (i = 1; i <= 9; i++) {
        for (j = 1; j <= i; j++) {
            printf("%d×%d=%2d  ", j, i, i * j);
        }
        printf("\n");   // 每行结束换行——外循环的职责
    }

    return 0;
}
```

内循环的上界写成 `j <= i` 而不是固定的 9，所以每行只打印到对角线——三角形乘法表。如果把内循环改成 `j <= 9`，就是完整的 9×9 方阵。

## 代码详解：四种循环场景逐段拆解

### 场景一：for 计数循环——累加 1 到 N

```c
#include <stdio.h>
int main(void) {
    int n, sum = 0;

    printf("N = ");
    scanf("%d", &n);

    for (int i = 1; i <= n; i++) {
        sum += i;
    }

    printf("1+...+%d = %d\n", n, sum);
    return 0;
}
```

C99 起可以在 for 的初始化里声明变量（`int i`）——i 的作用域只在 for 循环内，出了循环就不能再用 i。`sum += i` 每轮把当前 i 累加进去，循环结束后 sum 就是等差数列求和。

### 场景二：while 哨兵值——统计正数个数

```c
#include <stdio.h>
int main(void) {
    int x, count = 0;

    printf("输入整数（0 结束）：\n");
    scanf("%d", &x);

    while (x != 0) {
        if (x > 0)
            count++;
        scanf("%d", &x);
    }

    printf("正数个数：%d\n", count);
    return 0;
}
```

哨兵这里是 0。循环条件 `x != 0` 和「0 表示结束」的约定必须一致——换哨兵值时，判断条件也要一起改。

### 场景三：do-while 输入校验——至少问一次

```c
#include <stdio.h>
int main(void) {
    int age;

    do {
        printf("请输入年龄（1-120）：");
        scanf("%d", &age);
    } while (age < 1 || age > 120);

    printf("年龄 %d 已记录\n", age);
    return 0;
}
```

如果用户第一次就输入合法年龄，循环也只跑一轮就退出。如果输入非法，会反复提示——用 while 也能写，但得把提示和读入拆到循环外，结构不如 do-while 直观。

### 场景四：嵌套循环——遍历 3×3 矩阵求和

```c
#include <stdio.h>
int main(void) {
    int matrix[3][3] = {
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9}
    };
    int row, col, total = 0;

    for (row = 0; row < 3; row++) {
        for (col = 0; col < 3; col++) {
            total += matrix[row][col];
        }
    }

    printf("矩阵元素总和 = %d\n", total);  // 45
    return 0;
}
```

外循环 `row` 从 0 到 2，内循环 `col` 从 0 到 2——每个 (row, col) 恰好访问一次，共 9 次，与 NestedLoopDiagram 里的访问顺序一致。

## 容易踩的坑

<Callout type="trap">
  **现象**：`for (i = 0; i < 10; i++); { printf("%d\n", i); }` 循环体一次都不执行，或者只执行固定次数、与 i 无关。
  **原因**：for 括号后面多了分号 `;`——和 while 的分号陷阱一样，这个分号被当成「空循环体」。后面带花括号的代码块不属于 for，只是 for 结束后的普通语句。
  **修法**：去掉 for 后面的分号：`for (i = 0; i < 10; i++) { printf("%d\n", i); }`。所有循环关键字后面，花括号前不放分号。
</Callout>

<Callout type="trap">
  **现象**：`for (i = 0; i < 10; ) { printf("%d ", i); }` 程序卡死，风扇狂转。
  **原因**：更新部分被留空——i 永远是 0，条件 `i < 10` 永远为真，死循环。有人以为「循环体里写 i++ 就行」，但如果循环体里有 `continue` 跳过 i++，还是会挂。
  **修法**：把推进逻辑放在 for 的更新段（`i++`）——这是 for 的设计意图。如果更新只能写在循环体里，改用 while 更清晰。
</Callout>

<Callout type="trap">
  **现象**：嵌套循环打印矩形图案时，每行字符数不对，或三角形缺了一角。
  **原因**：内外循环边界搞混——外循环控制「行数」，内循环控制「每行几个」；把 `j <= i` 写成 `j < i` 会少打一列，把内外循环变量混用（内循环用外循环的变量当边界）会越界或重复。
  **修法**：画 NestedLoopDiagram 那样的访问顺序图，先标清「外循环走一步，内循环跑几圈」。乘法表用 `j <= i` 得三角，用 `j <= 9` 得方阵——边界差一个等号，图案完全不同。
</Callout>

<Callout type="trap">
  **现象**：哨兵值循环把第一个有效数据当成了结束标记，或者最后一个数据没被处理。
  **原因**：「先读再判」和「先判再读」的顺序搞反。若在循环外没先读入，while 条件用的是未初始化的垃圾值；若循环内处理完忘记再读入，会用同一个值死循环。
  **修法**：遵循 SentinelLoopDiagram 的四步：① 循环外读第一个 ② 判断是否为哨兵 ③ 不是则处理 ④ 再读下一个。哨兵值要选「正常数据不会出现」的数（如 -1、0），并在提示里告诉用户。
</Callout>

## 小结

- 三种循环：while（先判后做，可能零次）、for（三要素计数，次数已知时最清晰）、do-while（先做后判，至少一次）
- 哨兵值模式适合输入个数未知的场景：约定结束标记，先读入再判断
- 嵌套循环：外层次数 × 内层次数 = 总迭代次数；外循环管行/层，内循环管列/项
- for 的 init/cond/update 分工明确；退出时控制变量往往比最后一轮循环体的值大 1（当条件是 `<` 时）
- 铁律：循环关键字后不加多余分号；循环体永远写花括号；控制变量必须有明确的更新路径

**练习 1（改代码型）** 下面程序想打印 1 到 5，但有两处错误。找出并修正。

```c
#include <stdio.h>
int main(void) {
    int i;
    for (i = 1; i < 5; i++);
        printf("%d ", i);
    return 0;
}
```

<Answer>

两处问题：
① `i < 5` 写成了 `i < 5` 应该是 `i <= 5` 才能包含 5——当前只打印到 4。
② `for (...);` 后面多了分号，printf 不在循环体内，只执行一次。

修正后：

```c
#include <stdio.h>
int main(void) {
    int i;
    for (i = 1; i <= 5; i++) {
        printf("%d ", i);
    }
    return 0;
}
```

</Answer>

**练习 2（问答型）** 读程序，写出输出。解释 do-while 为什么至少执行一次。

```c
#include <stdio.h>
int main(void) {
    int n = 10;
    do {
        printf("%d ", n);
        n++;
    } while (n < 10);
    return 0;
}
```

<Answer>

输出：`10 `（一个 10 加空格）。

do-while 先执行循环体再判断条件。第一轮：打印 n=10，n 变成 11；判断 `11 < 10` 为假，退出。虽然条件一开始就不满足，循环体仍然执行了一次——这就是 do-while 与 while 的本质区别。

</Answer>

**练习 3（综合实现型）** 用嵌套 for 循环打印如下 4 行图案（每行字符数递增）：

```
*
**
***
****
```

要求：外循环控制行数（1 到 4），内循环控制每行星号个数。不能使用字符串常量数组。

<Answer>

```c
#include <stdio.h>
int main(void) {
    int row, col;

    for (row = 1; row <= 4; row++) {
        for (col = 1; col <= row; col++) {
            printf("*");
        }
        printf("\n");
    }

    return 0;
}
```

外循环 row 从 1 到 4；内循环 col 从 1 到 row——第 row 行打印 row 个星号。每行结束用 `printf("\n")` 换行。这与乘法表三角形是同一套嵌套边界模式。

</Answer>

## 名词解释

<Glossary>

<GlossaryItem term="while 循环">
  先检查条件、条件为真才执行循环体的入口条件循环。条件一开始就不成立时，循环体一次都不执行。适合「可能零次」的重复任务。详见本章「while 回顾」一节和 WhileLoopFlowDiagram。
</GlossaryItem>

<GlossaryItem term="哨兵值（sentinel value）">
  标记数据结束的特殊值（如 -1、0）。循环每次先读入数据，再判断是否为哨兵——不是就处理并继续读入，是就退出。适合事先不知道输入个数的场景。详见 SentinelLoopDiagram 和「while 回顾」一节。
</GlossaryItem>

<GlossaryItem term="for 循环">
  把初始化、条件、更新写进括号的计数循环：`for(init; cond; update) { ... }`。初始化只执行一次，然后重复「判断→循环体→更新」。次数已知时比 while 更紧凑。详见 ForLoopFlowDiagram 和 Stepper 分步图示。
</GlossaryItem>

<GlossaryItem term="循环控制变量（loop control variable）">
  推动循环向终止条件靠近的变量（如 i、n）。必须在每轮循环中被更新（通常在 for 的 update 段或循环体末尾），否则条件永远不变就会死循环。忘了更新是最常见的死循环原因。
</GlossaryItem>

<GlossaryItem term="do-while 循环">
  先执行循环体、再检查条件的出口条件循环。语法末尾的分号不能少。保证循环体至少运行一次——适合菜单显示、输入校验等必须先做一轮的场景。详见 DoWhileVsWhileDiagram。
</GlossaryItem>

<GlossaryItem term="嵌套循环（nested loop）">
  循环体内再写循环。外循环每走一步，内循环完整跑一圈。总次数约等于外层次数乘内层次数。常用于二维数组、矩阵、图案打印。详见 NestedLoopDiagram 和「嵌套循环」一节。
</GlossaryItem>

</Glossary>
