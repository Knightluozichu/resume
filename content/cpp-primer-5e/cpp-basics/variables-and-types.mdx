## 为什么 C++ 中要给每个数据贴一张"标签"？

想象你面前有一排不同大小的盒子：火柴盒、鞋盒、收纳箱、搬家纸箱。你要放的是一颗纽扣——扔进搬家纸箱也行，但太浪费；要放的是一床被子——硬塞火柴盒肯定不行。

在 C++ 里，**类型就像盒子规格**：每种数据类型决定了它能装多大的值、能装什么样的值。你把一个整数放进 `int` 这个盒子里，它干干净净占 4 字节；把一个带小数的数字放进 `float` 这个盒子，它就按浮点规则存储。**没有类型的语言？** 那就相当于你只有一种「万能麻袋」——什么都往里面塞，编译器帮不了你检查错误，内存要么浪费要么溢出。

这一章就来认识 C++ 里这些「盒子」的种类、贴标签的规矩，以及怎么让数据在该出现的地方出现、在该消失的时候消失。

## 类型——给内存里的每个格子划定规矩

在 C++ 中，<Term def="编程语言里给数据规定的类别，决定了一个数据占多大内存、能存什么范围的值、能对它做哪些运算。C++ 是强类型语言——每个变量的类型在编译时就定下来了。">类型（type）</Term> 是程序最基本的「规矩」。任何一个变量在被使用之前，它的类型必须是确定的——编译器要在**编译阶段**就搞清楚：这个变量得分配几字节的内存、它的值能有多大范围、能对它做加减乘除还是不能。

类型不是运行时才起作用的概念——它从编译器看到你代码的第一刻起就参与工作。所以它处在程序编写的**第一关口**：在你敲完代码、点编译的那一刻，类型系统就开始逐行检查你的变量声明是否合法、运算是否合理。

### 基本算术类型——C++ 里有哪几种盒子

C++ 内置了以下几种 <Term def="语言自带的基础类型，如 int、float、char 等，不需要 include 任何头文件就能直接用。与之相对的是标准库提供的高级类型（如 string、vector）。">算术类型（arithmetic type）</Term>：

- **整型**：`short`（2 字节）、`int`（4 字节）、`long`（4 或 8 字节）、`long long`（8 字节）——存整数
- **浮点型**：`float`（4 字节，约 7 位有效数字）、`double`（8 字节，约 15 位有效数字）、`long double`（更大）
- **字符型**：`char`（1 字节）、`wchar_t`（宽字符，2 或 4 字节）
- **布尔型**：`bool`（1 字节）——只能存 `true` 或 `false`

下面这张对照表帮你直观感受各类型在内存里的"个头"：

<TypeSizeDiagram />

<Callout type="tip">
  `int` 是 C++ 最常用的整型——32 位机器上一位大整数天生就爱它。
  `double` 是浮点数运算的默认选择（比 float 精度高很多，现代 CPU 对 double 计算已经很快）。
  选型口诀：整数用 int、小数用 double、字符用 char、真/假用 bool。
</Callout>

类型决定了**三件事**：
1. **内存大小**：`int` 占 4 字节，`short` 只占 2 字节——就像选盒子大小
2. **取值范围**：`unsigned char` 只能存 0~255，`signed char` 是 -128~127
3. **能做什么运算**：`float` 能算小数除法但可能有舍入误差，`bool` 不能做加减

### 带符号与无符号——同一段二进制，两种解读

当你写 `int`，它默认就是 **signed int**——能存正数、负数、零。如果你前面加一个 `unsigned`，比如 `unsigned int`，那它就**只能存 ≥0 的整数**。

两者的核心差别在于最高位的解读方式。C++ 使用 <Term def="计算机表示有符号整数的一种编码方式，根据 IEEE 标准。在补码系统中，最高位为 1 表示负数，值为 0→所有位全 0，-1→所有位全 1（11111111）。">补码（two's complement）</Term> 来表示有符号整数——在补码里，最高位为 1 就代表负数。下面这张图直观对比了同一个 8-bit 位模式在两种解读下的不同含义：

<SignedUnsignedDiagram />

<Callout type="warn">
  混用 signed 和 unsigned 是 C++ 里最常见的 bug 来源之一。
  当一个表达式中同时出现 signed 和 unsigned 时，signed 会被**隐式转换**成 unsigned——结果可能完全不是你想的那样。
  例如 `unsigned u = 10; int i = -42; std::cout << i + i << " vs " << u + i;`，第一个输出 -84，第二个却会是一个巨大的正数。
</Callout>

无符号类型常用来表示那些**不可能为负数**的量：数组下标、大小（`sizeof` 的返回值）、位掩码等。但在一般算术中，**默认用 signed int** 即可。

### 字面值常量——写在代码里的"裸值"

你写在程序里的那些直接写出来的值——`42`、`3.14`、`'a'`、`"Hello"`——叫作 <Term def="写在代码里的原始值本身，如 42、3.14、'a'、'Hello'。编译器从它的书写形式就能推断出它的类型。">字面值常量（literal）</Term>。编译器从字面值的**写法**就能推断出它的类型，不同的书写形式对应不同的类型：

<LiteralTypesDiagram />

几条记忆关键点：

- 整数 `42` 默认是 `int`；加 `u` 后缀变 `unsigned`、加 `L` 变 `long`、加 `LL` 变 `long long`
- 浮点数 `3.14` 默认是 `double`；加 `f` 后缀才能变成 `float`
- 字符用单引号 `'a'`、字符串用双引号 `"a"`（末尾暗藏一个 `\0` 结束符）
- 相邻字符串会被自动拼接：`"Hello" " World"` 等价于 `"Hello World"`

### 变量——给内存格子起个名字

<Term def="程序中一个有名字的、有类型的、可以存值并在生命周期内值可以被改变的内存空间。变量必须先声明（告诉编译器名字和类型）才能使用。">变量（variable）</Term> 就是给一块内存格子起的名字，加上一个类型标牌。在 C++ 里，变量**必须先声明再使用**。

几个容易混淆的概念必须分清：

| 概念 | 含义 | 例子 |
|---|---|---|
| <Term def="告诉编译器「这个名字、这个类型」存在，但不一定分配空间。声明可以出现多次。">声明（declaration）</Term> | 告诉编译器名字和类型 | `extern int x;`（仅声明，不在本文件定义） |
| <Term def="在声明的同时分配了内存空间、给了初始值（或不给）。在一个程序里，每个变量只能有一处定义。">定义（definition）</Term> | 分配内存空间 | `int x = 0;`（声明 + 定义） |
| <Term def="变量定义时给它第一个值。C++ 允许多种初始化方式：传统 = 号、花括号列表初始化（推荐）、括号初始化。">初始化（initialization）</Term> | 创建时给第一个值 | `int x = 0;` 或 `int x{0};`（推荐！） |
| <Term def="变量已经存在、已经初始化过的前提下，把一个新的值覆盖进去。不要跟初始化混为一谈——初始化是诞生时的事，赋值是之后的事。">赋值（assignment）</Term> | 把已有变量换个新值 | `x = 5;`（不是初始化，是赋值） |

C++11 引入的 <Term def="用花括号 {} 包围初始值的写法，如 int x{0}。它额外做了安全检查：不允许隐式截断的类型转换（如 double→int 花括号包围会报错或警告）。">列表初始化（list initialization）</Term> 是推荐的初始化方式——用花括号 `{}`：

```cpp
int x = 0;    // 老写法（拷贝初始化）
int x{0};     // 列表初始化——更安全！会拒绝截断的窄化转换
double d{3.14};      // ✅
int i{d};            // ❌ 警告或报错：double → int 会丢失信息！
int i = d;           // ⚠️ 不会报错，悄悄截断为 3（危险！）
```

`{}` 花括号不会允许你"悄悄地"把 `double` 截成 `int`——这是列表初始化比传统 `=` 写法更安全的核心原因。

### 作用域——变量该在哪儿"出生"、在哪儿"死掉"

<Term def="变量在程序中的可见范围——从它的声明点开始，到它所在那对花括号 { } 结束为止。出了这个范围，变量就被销毁，再也不能访问它。">作用域（scope）</Term> 决定了变量在哪里可以看见、在哪里已经不存在了。C++ 的作用域由花括号 `{}` 划定边界。

<ScopeNestingDiagram />

**嵌套作用域的核心规则**：

1. **外层可见内不会**：外层不能访问内层声明的变量——`innerOnly` 出了内层花括号就销毁了
2. **内层可见外层**：内层可以访问外层的变量（前提是同名变量没有被遮蔽）
3. **同名遮蔽（shadowing）**：内层声明了和外部同名的变量时，**内部用内层的，外层的在内部暂时看不见**

```cpp
#include <iostream>
int global = 42;                      // 全局作用域
int main() {
    int local = 10;                  // main 函数作用域
    {
        int local = 20;             // 遮蔽了外层的 local！
        int innerOnly = 30;         // 只在这个块里存在
        std::cout << local;         // 输出 20（内层的）
        std::cout << global;        // 输出 42（全局的，能看见）
        std::cout << innerOnly;     // 输出 30（本块定义的）
    }
    std::cout << local;             // 输出 10（外层又可见了！）
    // std::cout << innerOnly;      // ❌ 编译错误！innerOnly 不存在了
    return 0;
}
```

<Callout type="tip">
  每个变量从它的**声明点**出生，到它所在的**最近花括号 `}`** 结束就死亡。这就是 C++ 的 RAII（资源获取即初始化）思想的根基——生命周期 = 作用域。离开作用域的那一刻，变量自动被销毁，占用的内存自动还给系统。
</Callout>

## 动手：跟踪一个变量从出生到死亡

下面用一个最简程序，分步追踪变量的**出生（定义）→ 使用（运算/输出）→ 遮蔽（内层同名）→ 死亡（离开作用域）**的完整生命周期。

> 猜一猜：在内层代码块里把 `local` 改成 20 并输出，出了内层 `local` 又变成 10。变量到底是"被改回来了"还是"根本没被改"？通过下面 Stepper 一步步看解释。

<Stepper>
  <Step title="① 全局变量出生">
    <TypeSizeDiagram />
    程序启动后，最先创建全局作用域里的变量。`int global = 42;` 在程序整个运行期间都活着——它在 `main` 之前就已经分配好内存、赋予了初始值 42。全局变量存在「静态存储区」，生命周期 = 整个程序的运行时间。
  </Step>
  <Step title="② main 函数：local 出生">
    <VariableLifecycleDiagram step={2} />
    程序执行到 `int main()` 花括号里，`int local = 10;` 创建了一个局部变量。它分配在**栈（stack）**上——这块 4 字节内存的名字就是 `local`，存的值是 10。此时 `global=42` 仍然可见，`local=10` 也进入了 main 函数的作用域。
  </Step>
  <Step title="③ 进入内层块：同名遮蔽">
    <ScopeNestingDiagram />
    当程序遇到 `{` 内层花括号，一个**新的作用域**打开。`int local = 20;` 在这片新领地里又声明了一个同名变量——此时 C++ 的规则是：**用最近作用域的。** 所以 `std::cout << local;` 输出 20，不是 10。外层的 `local=10` 并没有被改掉，只是暂时被**遮蔽（shadowed）**了。`int innerOnly = 30;` 也在内层出生了。
  </Step>
  <Step title="④ 离开内层块：变量死亡 + 外层复活">
    <VariableLifecycleDiagram step={4} />
    碰到 `}`——内层作用域结束。`innerOnly` 和 内层 `local` 在出界的那一刻被销毁（栈顶上弹出），占用的内存还给系统。外层的 `local=10` 重新变成「当前最近的 local」，所以 `std::cout << local;` 输出 10。**外层的 local 从没被改过——它只是被内层同名变量暂时挡住了。**
  </Step>
  <Step title="⑤ main 结束：全局收尾">
    <VariableLifecycleDiagram step={5} />
    main 函数的 `}` 到来——`local` 离开作用域、销毁。最后程序退出，全局变量 `global` 也自动清理。整个生命周期闭环：出生（定义）→ 存在（生存期）→ 死亡（离开作用域），每个变量都遵循这个节拍。
  </Step>
</Stepper>

<Callout type="tip">
  **作用域 = 生命周期**。C++ 的这个铁律让你不用手动释放局部变量——出了花括号它自己就没了。
  这种"谁出生谁负责、到期自动销毁"的机制，就是 C++ RAII（资源获取即初始化）思想的根基。
</Callout>

## 代码速查：声明与初始化的写法

C++ 变量声明和初始化有几种不同写法，下面是在你需要时快速对照的关键例子：

```cpp
// 定义 + 初始化
int a = 0;          // 拷贝初始化（C 风格写法）
int b{0};           // 列表初始化（C++11，安全！拒绝窄化转换）
int c(0);           // 直接初始化
int d;              // 仅定义，未初始化（值是垃圾！不推荐）

// 不同后缀决定字面值类型
auto x = 3.14;      // x 是 double
auto y = 3.14f;     // y 是 float
auto z = 42u;       // z 是 unsigned int

// 列表初始化的安全检测
double pi = 3.14159;
// int bad{pi};     // ❌ 报错：窄化转换
int good = pi;      // ⚠️ 通过但截断成 3
```

`auto` 关键字能让编译器**自动推导**变量类型（从初始值推断），在你不关心具体类型或类型名很长时很方便——但记住推导规则：`auto` 会**去掉顶层 const 和引用**。

## 容易踩的坑

<Callout type="trap">
  **现象**：给 `unsigned` 变量赋一个负数，`std::cout` 输出一个巨大的正数。
  **原因**：负数在计算机里是补码存储，比如 -1 的 32-bit 补码是全 1（0xFFFFFFFF），当 unsigned 解读就是 4294967295。
  **修法**：绝对不要给 unsigned 变量赋负数。如果逻辑上某个量可能为负数（如差值），就用 signed 类型。
</Callout>

<Callout type="trap">
  **现象**：声明了一个变量没给它初始值，读出来是个随机垃圾数，每次运行都不一样。
  **原因**：C++ 不会自动帮你把变量初始化成 0——栈上分配的内存里可能残留上一次使用时的值。
  **修法**：养成习惯——**声明变量的同时就初始化它**。用 `int x{0};` 或 `int x = 0;`，别写 `int x;` 之后才赋值。
</Callout>

<Callout type="trap">
  **现象**：内层块里声明了和外层同名的变量，以为"修改了外层的变量"，结果出外层后发现外层值根本没变。
  **原因**：内层 `int x = 20;` 不是修改外层的 x，而是声明了一个**全新的同名变量**，遮蔽了外层的。出了内层这个新变量销毁，外层的原值复现。
  **修法**：避免滥用同名变量遮蔽。如果确实需要在内层直接修改外层变量，不重复声明即可——`x = 20;`（没有 `int` 前缀）就是在赋值，不是声明新变量。
</Callout>

## 小结

- C++ 是强类型语言——每个变量在编译时就确定了类型、内存大小、取值范围和允许的运算
- 基本算术类型有整型（short/int/long/long long）、浮点（float/double/long double）、字符（char/wchar_t）、布尔（bool）
- 有符号和无符号类型的差别在于最高位的解读方式——混用会导致隐式转换和意料之外的结果
- 列表初始化 `int x{0};` 比传统 `=` 写法更安全——它会拒绝窄化转换
- 作用域由花括号 `{}` 划定——变量从声明点活到所在块 `}` 结束；内层同名变量会遮蔽外层但不会修改它

1. `unsigned int x = -1;` ——给无符号类型赋负数，x 变成巨大的正数（4294967295）
2. `int y;` 后直接 `cout << y;`——y 未初始化，输出是随机垃圾值
3. `int rounded{pi};`——列表初始化会拒绝 double→int 的窄化转换，编译报错

修正后：

```cpp
#include <iostream>
int main() {
    int x = -1;                              // 负数用 signed
    int y{0};                                // 声明即初始化
    std::cout << y << std::endl;             // 输出 0
    double pi = 3.14159;
    int rounded = static_cast<int>(pi);      // 显式转换，表明你知道会截断
    std::cout << rounded << std::endl;       // 输出 3
    return 0;
}
```

**练习 2（问答型）** 读下面这段程序，写出每一行 `cout` 的输出结果，并解释原因：

```cpp
#include <iostream>
int main() {
    int x = 10;
    {
        int x = 100;
        std::cout << x << std::endl;     // ① 输出？
    }
    std::cout << x << std::endl;         // ② 输出？
    {
        x = 50;
        std::cout << x << std::endl;     // ③ 输出？
    }
    std::cout << x << std::endl;         // ④ 输出？
    return 0;
}
```

<Answer>

① 输出 100——内层 `int x = 100;` 声明新变量，遮蔽外层
② 输出 10——内层 x 已销毁，外层 x 恢复可见
③ 输出 50——`x = 50;` 没有 `int` 前缀，不是新声明而是**赋值**，修改的就是外层 x
④ 输出 50——外层 x 已经被③改成了 50

关键区分：`int x = 100;`（有 int）= 声明新变量、遮蔽外层；`x = 50;`（没有 int）= 给当前可见的 x 赋值。

</Answer>

**练习 3（综合型）** 把 `char` 类型变量分别赋值为 65、'A'、'\x41'，用 `cout` 输出。它们输出一样吗？为什么？再试试 `int i = 'A'; std::cout << i;` 输出多少？解释这几种写法之间的关系。

<Answer>

```cpp
#include <iostream>
int main() {
    char c1 = 65;
    char c2 = 'A';
    char c3 = '\x41';
    std::cout << c1 << c2 << c3 << std::endl;   // 输出 AAA
    int i = 'A';
    std::cout << i << std::endl;                 // 输出 65
    return 0;
}
```

三种写法都创建了一个值为 65（十六进制 0x41）的 char 变量。
- `65` 是十进制整数赋给 char
- `'A'` 是字符字面值——编译器查 ASCII 表得知 'A' = 65
- `'\x41'` 是十六进制转义——0x41 = 65 = 'A'
它们输出**都是 A**，因为 cout 对 char 类型按字符输出。而 `int i = 'A';` 后 cout 对 int 类型按整数输出，显示 65。**同一个值（65），不同的类型导致不同的输出形式**——这恰好说明了类型的核心作用：决定了数据如何被解释和显示。
</Answer>

<Glossary>

<GlossaryItem term="类型（type）">
  数据的一张"规格标签"。它告诉编译器：这个数据占多大内存（几字节）、能存什么范围的值（比如 0~255 还是 -128~127）、能对它做哪些运算（比如整数能加减、字符串能拼接）。C++ 是强类型语言——每个变量在编译时就必须确定类型，编译器会帮你检查类型是否匹配。详见本章「类型」一节。
</GlossaryItem>

<GlossaryItem term="算术类型（arithmetic type）">
  C++ 内置的基础数字类型，包括整型（int/short/long/long long）、浮点型（float/double/long double）、字符型（char/wchar_t）、布尔型（bool）。它们不需要 include 任何头文件就能直接用。详见本章「基本算术类型」一节和 TypeSizeDiagram 图示。
</GlossaryItem>

<GlossaryItem term="补码（two's complement）">
  计算机里表示有符号整数的方式。在补码里，最高位是"符号位"——为 1 表示负数、为 0 表示 ≥0。0 是所有位全 0，-1 是所有位全 1（11111111），-128 是 10000000。unsigned 把同一个位模式当纯正数解读，所以同样的 8-bit 位模式 11111111，unsigned 看是 255，signed（补码）看是 -1。详见本章「带符号与无符号」一节和 SignedUnsignedDiagram 图示。
</GlossaryItem>

<GlossaryItem term="字面值常量（literal）">
  直接写在代码里的"裸值"——比如 42、3.14、'a'、"Hello"。编译器从写法就能推断类型：没有小数点的整数是 int，带小数点的数是 double，单引号括起来的是 char，双引号括起来的是字符串。后缀可以改变默认类型：42u 是 unsigned int，3.14f 是 float。详见本章「字面值常量」一节和 LiteralTypesDiagram 图示。
</GlossaryItem>

<GlossaryItem term="变量（variable）">
  程序中一个有名字、有类型、可以存值的内存空间。变量必须先声明（告诉编译器"有这么个东西、它是什么类型"）然后才能使用。创建变量时给它第一个值叫"初始化"，之后给它换新值叫"赋值"——两者不一样。出了它所在的花括号，变量就自动被销毁。详见本章「变量」一节。
</GlossaryItem>

<GlossaryItem term="声明（declaration）">
  告诉编译器「这个名字、这个类型存在」，但不一定分配内存空间（如 `extern int x;`）。声明可以多次出现——只要不冲突。而"定义"一定是声明的一种，同时分配了内存。
</GlossaryItem>

<GlossaryItem term="定义（definition）">
  声明 + 分配内存空间。`int x = 0;` 既声明了变量 x 的类型和名字，又给它分配了 4 字节内存并写入初始值。在一个程序里，每个变量只能有一处定义（否则链接器报重定义错误）。
</GlossaryItem>

<GlossaryItem term="初始化（initialization）">
  变量出生时（定义时）给它第一个值。不是之后改值（那叫赋值）。C++ 有多种初始化写法：传统 `int x = 0;`（拷贝初始化）、推荐的 `int x{0};`（列表初始化——更安全，会拒绝窄化转换）、`int x(0);`（直接初始化）。如果不初始化就使用，变量里是上一段程序留下的随机"垃圾值"。详见本章「变量」一节。
</GlossaryItem>

<GlossaryItem term="赋值（assignment）">
  变量已经存在、已经有初始值之后，把一个新的值覆盖进去的操作。`x = 5;` 是赋值（没有类型名在前）。不要和初始化混淆——初始化是变量出生那一刻的事，赋值是之后随时可以做的。
</GlossaryItem>

<GlossaryItem term="列表初始化（list initialization）">
  C++11 引入的初始化写法：用花括号 `{}` 包围初始值，如 `int x{0};`。它比传统 `=` 写法更安全——会自动拒绝窄化转换（比如 `int x{3.14}` 编译器会报错，因为 double→int 会丢失小数部分）。详见本章「变量」一节。
</GlossaryItem>

<GlossaryItem term="作用域（scope）">
  变量在程序里的"可见范围"——从它声明的地方开始，到包含它的那对花括号 `}` 结束。出了作用域，变量自动销毁、内存还给系统。嵌套作用域的核心规则：内层可以看见外层的变量，但同名声明会遮蔽外层；出了内层，内层变量消失、外层重新可见。详见本章「作用域」一节和 ScopeNestingDiagram 图示。
</GlossaryItem>

</Glossary>
