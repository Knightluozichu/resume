下面几段代码块用来验证 shiki 高亮（github-dark）对多语言可用。GLSL 顶点着色器：

```glsl
#version 330 core
layout (location = 0) in vec3 aPos;
void main() {
  gl_Position = vec4(aPos, 1.0);
}
```

C++ 侧创建 VAO：

```cpp
unsigned int VAO;
glGenVertexArrays(1, &VAO);
glBindVertexArray(VAO);
```

WebGL2-TS 侧拿到上下文：

```ts
const gl = canvas.getContext("webgl2");
if (!gl) throw new Error("WebGL2 not supported");
```

数学方面，裁剪空间到屏幕的透视除法是逐分量除以 $w$ 分量；投影矩阵把视锥映射到
NDC 立方体，可写成下面这个独立公式块：

$$
P = \begin{bmatrix}
\frac{1}{a\tan(\theta/2)} & 0 & 0 & 0 \\
0 & \frac{1}{\tan(\theta/2)} & 0 & 0 \\
0 & 0 & -\frac{f+n}{f-n} & -\frac{2fn}{f-n} \\
0 & 0 & -1 & 0
\end{bmatrix}
$$

到这里，如果你能在浏览器里看到这页标题、正文、彩色代码块与公式，说明管线已经通了。

<Exercises>

**练习 1（改代码型）** 把顶点数据改成画一个矩形（两个三角形），观察并解释现象。

<Answer>

用两个三角形拼一个矩形：提供 4 个顶点 + 一个索引缓冲（EBO/`ELEMENT_ARRAY_BUFFER`），用 `glDrawElements` 绘制。

</Answer>

**练习 2（问答型）** 为什么需要先绑定 VAO 再配置顶点属性？

<Answer>

VAO 记录顶点属性指针配置与对应的 VBO 绑定；先绑 VAO，后续 `glVertexAttribPointer` 的状态才会被它捕获，绘制时只需绑回 VAO。

</Answer>

</Exercises>
