## 给流水线加一道「省料」工序

前几章那条流水线已经学会比远近、按镂空模板放行。但它还在干一件傻事：一个实心盒子摆在你面前，你最多只能看到它朝向你的两三个面，可流水线**连背对你那几个面也老老实实地铺色块、算颜色**——这些面最后总会被前面的面整个挡住，等于白算。

这一章给流水线加一道**省料**工序：在真正动手铺色块之前，先扫一眼每个面**是朝着你、还是背对着你**，把背对你的那些面**直接扔掉、根本不铺**。盒子是实心封闭的，背面反正被正面严严实实挡住，看不见——不画它，画面一模一样，却省掉了将近一半的活。

没有它会怎样？你的盒子照样能正确显示（有深度测试兜着），但流水线白白多算了背面那一半色块，性能凭空浪费。而怎么让流水线一眼认出「这个面是朝你还是背你」？秘密藏在三角形三个顶点的**绕行方向**里——这章就来拆这个。

## 面剔除：背对你的面，干脆不画

这道省料工序就是 <Term def="渲染管线在光栅化之前的一道优化工序：GPU 检查每个三角形是朝向镜头还是背对镜头，把背对镜头、观察者根本看不见的面直接丢弃，不再为它们做光栅化和片段着色。对封闭物体（如立方体）能省下约一半的片段开销，画面却完全不变。默认关闭，需手动启用。">面剔除</Term>（face culling）：GPU 检查每个三角形是**朝向镜头**还是**背对镜头**，把背对镜头、你根本看不见的面**在光栅化之前就丢掉**，不再为它们铺色块、跑片段着色器。

它在管线里的位置是 A 型必须答得出的：面剔除发生在**光栅化之前**——三角形顶点经过顶点着色器、被投影到屏幕之后，但还没被拆成一个个片段送进片段着色器之前。正因为卡在这个位置，被剔掉的面**连片段都不会生成**，省下的就是这些片段的着色开销。下面这张图先让你看清「画与不画」的差别：

注意：面剔除**默认是关着的**，要手动开。还有一个关键前提——它只对**封闭物体**（像立方体、球这种把内部裹严实的形状）才安全省料，因为只有封闭物体的背面才**总是被正面挡住**、注定看不见。这个前提到 §7 会反过来咬你一口。

## 环绕顺序：顶点绕一圈是逆时针还是顺时针

那 GPU 凭什么一眼认出「这个面朝你还是背你」？靠的是三角形三个顶点的 <Term def="一个三角形的三个顶点，按它们被定义的先后（v0→v1→v2）连起来走一圈，要么是逆时针（CCW，counter-clockwise）、要么是顺时针（CW，clockwise）——这个绕行方向就叫环绕顺序。它由你定义顶点的顺序决定，是 GPU 判定正面 / 背面的唯一依据。">环绕顺序</Term>（winding order）——把三个顶点按定义的先后（`v0 → v1 → v2`）连起来走一圈，这一圈要么是**逆时针**（CCW，counter-clockwise）、要么是**顺时针**（CW，clockwise）。

这个方向完全由你**定义顶点的顺序**决定。同样三个位置的点，你先写左下还是先写右下，绕出来的方向就相反。一般约定建模时所有三角形都按**逆时针**定义顶点。下面这张图把「逆时针」和「顺时针」两种绕法并排画出来：

<WindingOrderDiagram />

记住一个反直觉但关键的事实（也是整章的题眼）：**同一个三角形，从正面看是逆时针、从它背面看就变成顺时针**——就像你正面看墙上一个顺时针画的钟，绕到墙背后透过去看，它就成了逆时针。GPU 利用的正是这一点。

## 正面与背面：投影到屏幕后看到的环绕方向

有了环绕顺序，区分正背面就水到渠成了。一个三角形投影到屏幕后，GPU 看它三个顶点**在屏幕上**的环绕方向：方向和你约定的一致（默认逆时针），它就是 <Term def="投影到屏幕后，三个顶点的环绕方向和你约定的方向（默认逆时针 CCW）一致的三角形面。它朝向镜头、是你看得见的那一面，面剔除默认保留它、照常画出来。">正面</Term>（front face，朝向你、保留）；方向反过来（变成顺时针），它就是 <Term def="投影到屏幕后，三个顶点的环绕方向和你约定的方向相反的三角形面。它背对镜头、被正面挡住看不见，面剔除默认就是把它丢弃。同一个面从正面看是逆时针、绕到背后看会反转成顺时针，正是据此判为背面。">背面</Term>（back face，背对你、默认被剔）。

为什么这招管用？正是上一节那个事实：封闭物体上，朝向你的面投影后保持约定的逆时针（正面），而背对你的面是「从背后看」的，环绕自动反转成顺时针（背面）。于是 GPU 不需要任何额外信息，单看屏幕上的环绕方向，就能把朝你的和背你的分得清清楚楚。这个判定过程，下一节单步走一遍就彻底清楚了。

## glCullFace 与 glFrontFace：剔谁、谁算正面

光会判定还不够，你得告诉 GPU **剔哪一面**、以及**哪种环绕算正面**，这是两个独立的开关。先看决定剔掉哪一面的 <Term def="设置面剔除『剔掉哪一面』的函数，原型 glCullFace(mode)：GL_BACK 只剔背面（默认，最常用）、GL_FRONT 只剔正面、GL_FRONT_AND_BACK 正背都剔（这个面就什么都不画了）。它只在面剔除已启用（glEnable(GL_CULL_FACE)）时才生效。">`glCullFace`</Term>，默认是 `GL_BACK`（剔背面，也是绝大多数情况要的）；再看决定哪种环绕算正面的 <Term def="设置『哪种环绕方向算正面』的函数，原型 glFrontFace(mode)：GL_CCW 表示逆时针环绕为正面（默认）、GL_CW 表示顺时针环绕为正面。它必须和你顶点数据实际的环绕约定一致——设反了，正面会被当成背面剔掉、物体整个消失或像内外翻。">`glFrontFace`</Term>，默认是 `GL_CCW`（逆时针为正面）。这两件套配上 `glEnable(GL_CULL_FACE)` 这个总开关，就是面剔除的全部 API。

`glCullFace` 可填的值就三个：

| `glCullFace(...)`   | 剔掉哪一面                           |
| ------------------- | ------------------------------------ |
| `GL_BACK`           | 只剔**背面**（**默认**，最常用）     |
| `GL_FRONT`          | 只剔**正面**（反过来，留背面）       |
| `GL_FRONT_AND_BACK` | 正面背面**都剔**（这个面什么都不画） |

划重点：`glFrontFace` 的约定**必须和你顶点数据实际的环绕方向一致**。要是你的顶点其实是顺时针定义的，却把 `glFrontFace` 留成默认的 `GL_CCW`，GPU 就会把你的正面全判成背面剔掉——物体直接消失或像内外翻。这正是 §7 第一个坑。

## 对比一下：剔除关 vs 剔除开

> 猜一猜：把下面这个滑块从左拖到右，左边是「面剔除关着」的立方体、右边是「面剔除开了」的。关着的时候，立方体看起来会更**干净**还是更**乱**？那些多出来的、本不该看见的，是哪些面？拖动看看，再回想 §5——它们正是「背对你、本该被剔掉」的背面。

## 代码对照：开启面剔除这三行

面剔除的代码短到不能再短——开总开关、（按需）指定剔哪面和哪种环绕算正面，剩下 GPU 全自动干。

<CodeTabs>
  <Tab label="C++ / OpenGL 3.3">

```cpp
// 初始化时开一次：默认是关的，不开就不会剔任何面
glEnable(GL_CULL_FACE);

// 下面两行其实都是默认值，写出来只为表明意图：
glCullFace(GL_BACK);    // 剔背面（默认就是 GL_BACK）
glFrontFace(GL_CCW);    // 逆时针环绕算正面（默认就是 GL_CCW）
```

  </Tab>
  <Tab label="TypeScript / WebGL2">

```ts
// 初始化时开一次：默认是关的，不开就不会剔任何面
gl.enable(gl.CULL_FACE);

// 下面两行其实都是默认值，写出来只为表明意图：
gl.cullFace(gl.BACK); // 剔背面（默认就是 BACK）
gl.frontFace(gl.CCW); // 逆时针环绕算正面（默认就是 CCW）
```

  </Tab>
</CodeTabs>

逻辑两端一字对应：`glEnable(GL_CULL_FACE)` 打开总开关；`glCullFace` 选剔哪面、`glFrontFace` 选哪种环绕算正面，**这两个都用默认值就够了**——只要你的顶点确实是逆时针定义的，开个总开关立刻就能省料。需要看物体内部时（比如想从内部看一个房间），把 `glCullFace` 设成 `GL_FRONT` 改剔正面，或干脆 `glDisable(GL_CULL_FACE)` 关掉即可。注意这里的 API 差异和那个最容易栽的前提：

<Callout type="warn">
  API 差异：桌面是全局函数 + 大写常量 `glEnable(GL_CULL_FACE)`；WebGL2
  是上下文方法 + 上下文常量 `gl.enable(gl.CULL_FACE)`，`gl.cullFace` /
  `gl.frontFace` 同理。更要命的是 **面剔除默认是关的**——不调 `glEnable`
  它一点不起作用、也不报错。还有一条铁律：**`glFrontFace`
  的环绕约定必须和你顶点数据实际的绕向一致**。顶点是逆时针就保持默认
  `GL_CCW`；要是数据其实是顺时针的，得改成
  `GL_CW`，否则正面会被当背面剔掉、物体整个消失。
</Callout>

## 容易踩的坑

<Callout type="trap">
  **现象**：开了面剔除后，物体**整个消失**了，或者看起来像**内外翻**——只剩本该看不见的内壁、朝你的面全没了。**原因**：环绕约定对不上——要么顶点数据其实是**顺时针**定义的，`glFrontFace`
  却还是默认 `GL_CCW`；要么不小心把 `glCullFace` 设成了
  `GL_FRONT`（剔正面）。两种都导致**正面被当背面剔掉**。**修法**：让
  `glFrontFace` 和顶点真实绕向一致（顺时针数据用 `GL_CW`），并确认 `glCullFace`
  是 `GL_BACK`。
</Callout>

<Callout type="trap">
  **现象**：给一片**草**、一张**纸片**、一面**布**（这类只有一层、两面都要看见的网格）开了面剔除，结果它从某些角度看**缺了一半、出现破洞**或干脆消失。**原因**：面剔除假设「背面看不见」，可单面/开放网格**没有把内部裹住**，它的「背面」其实是要给你看的另一面——被剔掉就破了。**修法**：对这类单面或双面要看的物体，**别开面剔除**（`glDisable(GL_CULL_FACE)`），或单独为它们临时关掉、画完再开回来。
</Callout>

<Callout type="trap">
  **现象**：照着教程设了 `glCullFace` /
  `glFrontFace`，可性能一点没省、背面照样在画。**原因**：忘了
  `glEnable(GL_CULL_FACE)` 这个**总开关**——它默认是**关**的，光设 `glCullFace` /
  `glFrontFace` 而不启用，面剔除根本没开工，还不报错。**修法**：初始化时务必
  `glEnable(GL_CULL_FACE)` 开一次；想临时不剔，用 `glDisable(GL_CULL_FACE)` 关。
</Callout>

## 小结

- **面剔除**在**光栅化之前**把背对镜头、看不见的面直接丢弃——对**封闭物体**白省约一半片段开销，画面不变
- GPU 靠**环绕顺序**判正背：顶点 `v0 → v1 → v2` 走一圈，**逆时针 CCW** 还是 **顺时针 CW**，由你定义顶点的顺序决定
- 投影到屏幕后，朝你的面保持约定的逆时针 = **正面**（保留）；背你的面环绕反转成顺时针 = **背面**（默认被剔）
- 三件套：`glEnable(GL_CULL_FACE)`（默认关，要开）/ `glCullFace`（剔谁，默认 `GL_BACK`）/ `glFrontFace`（哪种环绕算正面，默认 `GL_CCW`，**必须和顶点绕向一致**）
- **单面 / 开放网格**（草、纸片、布、敞口模型）和**双面要看的物体**别开面剔除，否则会缺一半、破洞

<Exercises>

**练习 1（排错题：物体消失 / 破洞）** 某人做了两件事都翻车了：① 给一个立方体开了面剔除，立方体却**整个消失**了；② 给一片随风摆的**草**（一张单面纸片网格）开了同样的设置，草从背面看**整个没了**。这两个现象分别是什么原因？各自怎么改？

<Answer>

两个翻车原因不同：

① **立方体消失**：这是环绕约定对不上——它的顶点数据其实是**顺时针**定义的，而 `glFrontFace` 还是默认 `GL_CCW`，于是 GPU 把它朝你的**正面全判成了背面剔掉**，只剩本该看不见的内壁，看起来就是「消失 / 内外翻」。修法：把 `glFrontFace(GL_CW)` 改成和顶点真实绕向一致（或反过来调整顶点数据），确保 `glCullFace` 是 `GL_BACK`。

② **草从背面没了**：草是**单面网格**，两面都要看见，但面剔除会把它「背对你」的那一面当背面剔掉——绕到草背后看，正好剔的就是你要看的那面，于是整片草没了。修法：草这类单面 / 双面要看的网格**根本不该开面剔除**，给它单独 `glDisable(GL_CULL_FACE)`，画完再开回来。

关键区别：①是「该开却设错了约定」，②是「这类物体压根不该开」。

</Answer>

**练习 2（问答题）** 有人说「面剔除靠的是比较深度，把更远的面扔掉」。这个说法对吗？面剔除到底是在管线的哪一步、靠什么信息认出该剔的面？它和深度测试是同一回事吗？

<Answer>

不对。面剔除**不比较深度**，靠的是三角形三个顶点投影到屏幕后的**环绕方向**（逆时针还是顺时针）来判正背面，把背面剔掉。它发生在**光栅化之前**（顶点投影完、还没拆成片段时），被剔的面连片段都不生成。

它和**深度测试**是两回事：深度测试在**片段着色器之后**、靠**比深度值**逐片段决定遮挡，处理的是任意物体之间谁挡谁；面剔除在**光栅化之前**、靠**环绕方向**整面丢弃单个封闭物体自己背对镜头的面，是一种省开销的优化。两者各管一段、互不替代——面剔除省下背面的片段，深度测试再处理剩下这些面之间的遮挡。

</Answer>

**练习 3（改设置 / 观察题）** 在 §5 那个「剔除关 vs 开」对比里，默认是 `glCullFace(GL_BACK)`（剔背面）。如果把它改成 `glCullFace(GL_FRONT)`（改剔正面），对着一个封闭立方体看，画面会变成什么样？这种「剔正面」在什么场景下反而有用？

<Answer>

改成 `glCullFace(GL_FRONT)` 后，朝你的**正面全被剔掉**，留下的是**背对你的背面**——于是你会**透过物体看到它的内壁**，就像把立方体「翻过来从里面看」。对一个普通实心立方体来说这通常是穿帮的。

但「剔正面」在**从物体内部观察**的场景下反而正好有用：比如你要做一个**房间 / 天空盒**，把摄像机放在一个大盒子内部，你想看到的恰恰是盒子的**内壁**（背对盒子外侧、朝向房间内的那些面）。这时剔掉朝外的正面、保留朝内的背面，正好把房间内壁干净地显示出来，又省掉了朝外那些你看不到的面。

</Answer>

<Glossary>

<GlossaryItem term="面剔除">
  渲染管线在**光栅化之前**的一道优化工序：GPU
  检查每个三角形是朝向镜头还是背对镜头，把背对镜头、你根本看不见的面**直接丢弃**，不再为它们铺色块、跑片段着色器。对封闭物体（如立方体）能省下约一半片段开销，画面却完全不变。默认**关闭**，要手动
  `glEnable(GL_CULL_FACE)` 开。详见本章「面剔除」一节。
</GlossaryItem>

<GlossaryItem term="环绕顺序">
  一个三角形的三个顶点，按定义的先后（`v0 → v1 →
  v2`）连起来走一圈，要么是**逆时针**（CCW，counter-clockwise）、要么是**顺时针**（CW，clockwise）——这个绕行方向就叫环绕顺序。它由你写顶点的顺序决定，是
  GPU 判定正面 / 背面的唯一依据。详见本章「环绕顺序」一节。
</GlossaryItem>

<GlossaryItem term="正面">
  投影到屏幕后，三个顶点的环绕方向和你约定的方向（默认逆时针
  CCW）**一致**的三角形面。它朝向镜头、是你看得见的那一面，面剔除默认**保留**它、照常画出来。详见本章「正面与背面」一节。
</GlossaryItem>

<GlossaryItem term="背面">
  投影到屏幕后，三个顶点的环绕方向和你约定的方向**相反**的三角形面。它背对镜头、被正面挡住看不见，面剔除默认就是把它**丢弃**。同一个面从正面看是逆时针、绕到背后看会反转成顺时针，正是据此判为背面。详见本章「正面与背面」一节。
</GlossaryItem>

<GlossaryItem term="glCullFace">
  设置面剔除「**剔掉哪一面**」的函数，原型 `glCullFace(mode)`：`GL_BACK`
  只剔背面（**默认**，最常用）、`GL_FRONT` 只剔正面、`GL_FRONT_AND_BACK`
  正背都剔（这个面就什么都不画了）。它只在面剔除已启用 `glEnable(GL_CULL_FACE)`
  时才生效。详见本章「glCullFace 与 glFrontFace」一节。
</GlossaryItem>

<GlossaryItem term="glFrontFace">
  设置「**哪种环绕方向算正面**」的函数，原型 `glFrontFace(mode)`：`GL_CCW`
  表示逆时针环绕为正面（**默认**）、`GL_CW`
  表示顺时针环绕为正面。它**必须和你顶点数据实际的环绕约定一致**——设反了，正面会被当背面剔掉、物体整个消失或像内外翻。详见本章「glCullFace
  与 glFrontFace」一节。
</GlossaryItem>

</Glossary>
